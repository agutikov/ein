## S3.1.2 Tasks

### Task T3.1.2.1 — Sort declarations

For each `(type T)` in the IR ontology, emit:

```smt2
(declare-datatypes ((T 0)) (((T_instance_1) (T_instance_2) ...)))
```

For `:positional` types (P1.4 S1.4.2), also emit per-instance
position variables:

```smt2
(declare-fun pos_T_instance_1 () Int)
(assert (and (<= 1 pos_T_instance_1) (<= pos_T_instance_1 5)))
```

### Task T3.1.2.2 — Relation translation

For each `(rel R src dst)` fact: depends on the relation's
declared shape.

- *functional* (`R : A → B`): emit a function symbol `R : A → B`
  and assert `(= (R src) dst)`.
- *symmetric* / *transitive*: emit relation-level axioms before
  facts.

### Task T3.1.2.3 — Constraint translation

Map each M1 constraint to SMT-LIB:

| M1 form            | SMT-LIB                                                         |
|--------------------|-----------------------------------------------------------------|
| `all-different`    | `(distinct …)`                                                  |
| `at-most-one`      | unpacked: pairwise `(=>` exists `=>` other-doesnt-exist`)`        |
| `exactly-n`        | `(= (+ (ite p1 1 0) … (ite pn 1 0)) n)`                          |
| spatial `right-of` | `(> pos_a pos_b)`                                               |
| spatial `next-to`  | `(= 1 (abs (- pos_a pos_b)))`                                   |

### Task T3.1.2.4 — Query translation

`(query :mode solve :goal G)` → assert nothing extra; on `sat`,
extract model values for `G`'s free variables.

# S3.3.1 — Solve / gaps / contradictions on SMT

**Phase:** P3.3 (three task classes on SMT).
**Estimate:** 5-7 days.
**Depends on:** P3.1, P3.2.
**Resolves:** [Q28](../../open_questions.md#q28--model-extraction-granularity).

## Context

Three modes, three SMT idioms. All three operate on the same
`(translate ir) -> smt2_text` plumbing from P3.1.

## Tasks

### Task T3.3.1.1 — Solve mode

`src/ein_bot/smt/solve.py`. `check-sat`; on `sat`, `get-value` for
the query's goal variables; convert back to IR atoms (instance
names, positions).

### Task T3.3.1.2 — Gaps mode (model enumeration)

`src/ein_bot/smt/gaps.py`. Iteratively:

1. Solve once → model M₁.
2. Add `(assert (or (≠ goal₁ M₁(goal₁)) (≠ goal₂ M₁(goal₂)) …))`.
3. Solve again → M₂.
4. For each goal variable, record whether M₂ disagrees with M₁.
5. Repeat to cap K (default 8 — enumerating all models on a small
   puzzle is fine; cap protects against blow-up).
6. Variables with agreement across all enumerated models →
   *forced*; variables with disagreement → *free* (gap).

Output: a dict `{goal_var: {forced: value} | {free: [v1, v2, ...]}}`.

### Task T3.3.1.3 — Contradictions mode (unsat core / MUS)

`src/ein_bot/smt/contradictions.py`. With Z3:

```
(set-option :produce-unsat-cores true)
;; ... assertions named with :named labels
(check-sat)
(get-unsat-core)
```

Translator emits `:named` labels on every fact assertion
(`(! (rel lives-in Norwegian House_1) :named fact-condition-10)`)
so the core comes back as a set of fact labels. MUS refinement:
iteratively drop assertions and re-check until a minimal core is
left.

### Task T3.3.1.4 — IR ↔ SMT name mapping

`src/ein_bot/smt/names.py`. Bidirectional mapping between IR
atoms / facts and the SMT-LIB `:named` labels. Required by P3.4 to
lift unsat cores back to IR provenance.

### Task T3.3.1.5 — Tests

`tests/smt/test_task_classes.py`:

- Solve mode → canonical Zebra answer.
- Gaps mode on Zebra-minus-15 → at least one free goal variable.
- Contradictions mode on `zebra-bad.ein` → a 2-3 fact unsat core.
- All-models enumeration on a deliberately ambiguous toy → ≥ 2
  models.

## Acceptance

- All three modes operational on Zebra + the three variants.
- Each mode returns the *same answer* as M1's pure-graph engine on
  the same input.
- A divergence (graph vs SMT) raises a loud assertion in CI — it's
  almost certainly a translator bug.

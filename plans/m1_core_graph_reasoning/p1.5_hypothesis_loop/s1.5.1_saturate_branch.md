# S1.5.1 — Saturate-then-branch driver

**Phase:** P1.5 (hypothesis loop).
**Estimate:** 4-5 days.
**Depends on:** P1.2 `g.fork()`, P1.3 saturator, P1.4 verify.
**Resolves question:** [M1 Q19 — hypothesis branching strategy](../../open_questions.md#q19--hypothesis-branching-strategy).

## Context

Working answer from [M1 Q19](../../open_questions.md#q19--hypothesis-branching-strategy):
**lazy branching**. The driver:

1. Saturates with all propagation rules (P1.3 S1.3.3).
2. If `solved()` → return Solution.
3. If `verify` fires a contradiction → return Contradiction with
   the unsat-core (P1.4 + P1.2 provenance).
4. Else (saturator stalled, no contradiction): generate hypotheses
   for the most-constrained still-unbound node ("first-fail"
   heuristic), `g.fork()` per candidate, recurse.

This mirrors the PoC's algorithm sketch but defers branching as
long as possible — closer to a human walkthrough than the PoC's
eager hypothesis generation.

## Tasks

### Task T1.5.1.1 — `solve` driver

`src/ein_bot/engine/solve.py`:

```python
@dataclass(frozen=True, slots=True)
class Solution:    graph: Graph; trace: list[Firing]
class Ambiguity:   branches: list[Solution]
class Contradiction: unsat_core: set[EdgeId]; derivation: DerivationDAG

Verdict = Solution | Ambiguity | Contradiction

def solve(g: Graph, reg: Registry,
          *, mode: Mode = Mode.SOLVE,
          max_depth: int = 6) -> Verdict: ...
```

`Mode = SOLVE | GAPS | CONTRADICTIONS` selects what to return on
quiescence: SOLVE returns the single survivor, GAPS returns all
survivors, CONTRADICTIONS returns the unsat-core whenever a branch
collapses (instead of pruning silently).

### Task T1.5.1.2 — Hypothesis generation (`hgen`)

`src/ein_bot/engine/hgen.py`. Following [idea 05 §Generation](../../../docs/ideas/05-zebra-puzzle-graph-reasoner.md#generation-hgen):

1. Pick the *most-constrained* unsolved node — the one with the
   smallest live domain (`dom(?a)`). Ties broken by node id.
2. For each value `v ∈ dom(?a)`, produce a candidate fact
   `(= ?a v)` with `kind='hypothesis'` provenance.
3. Eliminate candidates that immediately violate a constraint
   without saturation (`verify_incremental`).

Output: a list of `(g.fork(), hypothesis_edge)` pairs to try.

### Task T1.5.1.3 — Branch test + backjump

For each branch:

- Apply hypothesis as an edge in the reasoning layer with
  `Provenance(kind='hypothesis', branch=branch_id)`.
- Run `saturate`.
- On `Violation`: backjump (drop the branch). Record the unsat-core
  in CONTRADICTIONS mode.
- On clean quiescence with `solved()`: keep as survivor.
- On clean quiescence with still-unsolved: recurse (one level
  deeper hypothesis), per S1.5.2.

### Task T1.5.1.4 — Quiescence predicate (`solved`)

Per-mode plug-in:

- SOLVE: every node has a singleton domain.
- GAPS: every *queried* node has a singleton domain (others can stay
  free).
- CONTRADICTIONS: never solved (always run to exhaustion).

The query (P1.1's `(query :goal …)`) carries the goal nodes. The
engine reads that and configures `solved`.

### Task T1.5.1.5 — Tests

`tests/engine/test_solve_driver.py`:

- A trivial 1-variable puzzle solves in zero branches.
- A 2-variable puzzle with one constraint solves in ≤ 1 branch.
- A deliberately-contradictory input returns Contradiction with the
  expected unsat-core.
- Lazy-vs-eager assertion: a fully-propagable puzzle never branches.

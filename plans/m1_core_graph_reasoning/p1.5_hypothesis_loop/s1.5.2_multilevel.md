# S1.5.2 — Multilevel branching + search tree

**Phase:** P1.5 (hypothesis loop).
**Estimate:** 3-4 days.
**Depends on:** S1.5.1.

## Context

[Idea 05 §Multilevel hypothesis](../../../docs/ideas/05-zebra-puzzle-graph-reasoner.md#multilevel-hypothesis):
when a hypothesis survives but leaves the puzzle still
unsolved, branch *again*. The structure is a search tree:

- nodes = hypotheses;
- arcs = "leads-to";
- leaves = either Solution (kept), Contradiction (pruned), or
  open (not yet explored).

The tree is a *first-class artefact* — rendered as DOT in P1.6
S1.6.3, walked by the trace planner, and shown in the markdown
trace.

## Tasks

### Task T1.5.2.1 — `SearchTree` value

`src/ein_bot/engine/search_tree.py`:

```python
@dataclass(frozen=True, slots=True)
class SearchNode:
    id:         BranchId
    parent:     BranchId | None
    hypothesis: EdgeId | None            # the (= ?a v) we assumed at this node
    children:   tuple[BranchId, ...]
    verdict:    Literal['solution', 'contradiction', 'open']
    contradiction_core: tuple[EdgeId, ...] | None = None

@dataclass(frozen=True, slots=True)
class SearchTree:
    root:  BranchId
    nodes: dict[BranchId, SearchNode]
```

Immutable tree built bottom-up by `solve()`.

### Task T1.5.2.2 — Recursion + max-depth guard

`solve` already recurses; this task formalises:

- `max_depth` budget (default 6). The Zebra walkthrough needs ≤ 3;
  6 leaves headroom for harder puzzles.
- On exceeded depth → return `Ambiguity` carrying the survivors at
  the cutoff (caller decides whether to deepen).
- Detect "doesn't matter which" branches (S1.5.3) and clip them.

### Task T1.5.2.3 — Pruning + dedup

If two sibling branches produce identical post-saturation reasoning
layers (modulo the hypothesised edge), they're indistinguishable and
can be collapsed. Compare hashes of the reasoning-layer edge set.

### Task T1.5.2.4 — Ambiguity packaging

`Ambiguity(branches: list[Solution])` exposes:

- `.equal_on(goal_node)` — true iff all branches agree on the
  value of `goal_node` (used by GAPS mode to identify free
  variables);
- `.diverges_on(goal_node)` — list of distinct values.

P1.7 uses these to power the GAPS task class output: "the colour
of house 1 is free — could be Yellow or Ivory."

### Task T1.5.2.5 — Tests

`tests/engine/test_multilevel.py`:

- A 3-variable puzzle requiring 2 hypothesis levels.
- A puzzle requiring 3 levels with one mid-level contradiction
  retraction.
- Dedup test: two sibling branches with the same reasoning closure
  collapse.
- GAPS mode: removing one Zebra clue gives at least one diverging
  goal node.

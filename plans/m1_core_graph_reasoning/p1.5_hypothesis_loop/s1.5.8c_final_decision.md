## 3 — The kernel delta in detail

### 3.1 — `(not P)` becomes uniform fact-match

**Today** (`ein.py/src/ein/inference/compile.py:210`):
```python
if head_name == "not":
    return [NegativeGuard(sub_steps=tuple(sub_steps))]
```

The matcher treats `(not P)` in `:match` as NAF — "P is not
derivable" — *independently of whether `(not P)` is stored as
a fact*. This conflicts with `:assert (not P)`, which stores
the negative as an ordinary fact.

**Decision**: delete the special-case arm. `(not P)` in
`:match` becomes an ordinary fact pattern with head `not` and
nested arg. The matcher matches it against stored `(not …)`
facts (which `type-exclusivity` writes throughout zebra2's
elimination work).

**Migration**: zero rules. A grep of `examples/zebra2.ein` and
`examples/branching/*.ein` finds 56 `:match` lines and zero
that use `(not P)` as NAF. No churn.

### 3.2 — `NegativeGuard` → `AbsentGuard`; `(absent P)` head keyword

Same opcode, new name. Register `absent` in `_compile_premise`
as the head that produces `AbsentGuard`. Truth semantics
unchanged from today's `NegativeGuard` — sub-plan tried,
parent continues iff sub-plan yields zero rows.

This is now the *only* way to express NAF. Every NAF in M1
becomes explicit at its use site.

### 3.3 — Grammar: `*` in identifier tail

`grammar.lark` diff:
```diff
- SYMBOL:   /(?!(?:instance|not|and|or|neq|relation|a-priori)\b)[A-Za-z][A-Za-z0-9_-]*/
+ SYMBOL:   /(?!(?:instance|not|and|or|neq|relation|a-priori)\b)[A-Za-z][A-Za-z0-9_*-]*/
- VAR:      /\?[A-Za-z][A-Za-z0-9_-]*/
+ VAR:      /\?[A-Za-z][A-Za-z0-9_*-]*/
```

`*` is purely a character in identifier names; no
multiplicative or Kleene meaning. `is-a*` is to `is-a` as
`Old_Gold` is to `Old` — a distinct identifier sharing a
prefix. No tokenisation ambiguity (S-expression whitespace /
paren delimiters; `INT` / `RANGE` / `STRING` have
non-overlapping leading chars).

### 3.4 — Parser sugar: `(forall ?b (G) (B))`

IR-loader rewrites at load time:

```
(forall ?b (G) (B))   →   (absent (and G (absent B)))
```

Reduction: `∀x. G(x) → B(x)` ≡ `¬∃x. G(x) ∧ ¬B(x)`. The
matcher sees only the desugared form; no new opcode.

Safety: the rewrite preserves `?b` as a local var (sealed
inside the outer `absent`'s sub-plan). The rewrite rejects at
load time if `?b` doesn't appear positively in `G` (same
condition `Scan` already enforces — `?b` must be grounded by
an enumerable guard).

### 3.5 — Parser sugar: `(open P)`

```
(open P)   →   (and (absent P) (absent (not P)))
```

Names the third fact state: "P is neither asserted nor
negated — still hypothetical". With the three pattern shapes
matching the three storage states, the rule layer becomes
3-valued-aware.

| pattern        | semantics                                       |
|----------------|-------------------------------------------------|
| `(P)`          | `P` is asserted (stored positive)               |
| `(not P)`      | `P` is negated (stored `(not P)`)               |
| `(open P)`     | `P` is neither asserted nor negated             |
| `(absent P)`   | `P` is not asserted (could be negated OR open)  |
| `(absent (not P))` | `P` is not negated (could be asserted OR open) |

`(open P)` is the principled way to express "still candidate"
in any rule that wants to *gate on hypothesis state* — most
visibly in hrules that emit guesses only for un-decided slots.

## 4 — The ein stdlib block

The complete 10-rule library S1.5.8 ships. Lives inlined in
`examples/zebra2.ein` until P1.8 imports land; future puzzles
copy-paste or import. zebra2 activates the first 6; the
closure facility (last 4) fires only when a puzzle declares
`(closure …)`.

```lisp
(rules
  ;; ─────────────────────────────────────────────────────────
  ;; § A — Reflection (M1-blocking)
  ;; Two rules per (relation R T1 T2) declaration → two
  ;; (arg-type R i T) facts. Read by typecheck + domain-elim.
  ;; ─────────────────────────────────────────────────────────
  (rule reflect-arg-type-0 ()
    :match  (relation ?R ?T1 ?T2)
    :assert (arg-type ?R 0 ?T1)
    :priority 100
    :why    "{?R} arg 0 is typed {?T1}")

  (rule reflect-arg-type-1 ()
    :match  (relation ?R ?T1 ?T2)
    :assert (arg-type ?R 1 ?T2)
    :priority 100
    :why    "{?R} arg 1 is typed {?T2}")

  ;; ─────────────────────────────────────────────────────────
  ;; § B — Typecheck (M1-blocking)
  ;; Closed-world NAF on direct (is-a …) — requires B1's
  ;; direct-parent-types constraint to be sound. Priority 110:
  ;; well-formedness sub-band within propagate.
  ;; ─────────────────────────────────────────────────────────
  (rule typecheck-arg-0 ()
    :match  (and (relation ?R ?Dom ?Ran)
                 (?R ?a ?b)
                 (absent (is-a ?a ?Dom)))
    :assert (false)
    :priority 110
    :why    "{?R} arg 0 expects {?Dom}; {?a} is not a {?Dom}.")

  (rule typecheck-arg-1 ()
    :match  (and (relation ?R ?Dom ?Ran)
                 (?R ?a ?b)
                 (absent (is-a ?b ?Ran)))
    :assert (false)
    :priority 110
    :why    "{?R} arg 1 expects {?Ran}; {?b} is not a {?Ran}.")

  ;; ─────────────────────────────────────────────────────────
  ;; § C — Domain elimination + all-excluded contradiction
  ;; (M1-blocking). Priority 400 (eliminate-late) ensures all
  ;; type-exclusivity firings (300 band) have written the (not
  ;; (R a v)) facts before these rules consume them.
  ;; ─────────────────────────────────────────────────────────
  (rule domain-elimination ()
    :match  (and (functional  ?R 0 1)
                 (total       ?R 0)
                 (arg-type    ?R 0 ?OT)
                 (arg-type    ?R 1 ?VT)
                 (is-a ?a ?OT)
                 (is-a ?v ?VT)
                 (forall ?v_other
                   (and (is-a ?v_other ?VT) (neq ?v_other ?v))
                   (not (?R ?a ?v_other))))
    :assert (?R ?a ?v)
    :priority 400
    :why    "every {?VT} except {?v} excluded for {?a} — therefore ({?R} {?a} {?v}).")

  (rule no-room-left ()
    :match  (and (functional  ?R 0 1)
                 (total       ?R 0)
                 (arg-type    ?R 0 ?OT)
                 (arg-type    ?R 1 ?VT)
                 (is-a ?a ?OT)
                 (forall ?v
                   (is-a ?v ?VT)
                   (not (?R ?a ?v))))
    :assert (false)
    :priority 110
    :why    "{?a} has every {?VT} excluded — no value left.")

  ;; ─────────────────────────────────────────────────────────
  ;; § D — Closure facility (OPTIONAL; zebra2 doesn't activate
  ;; it). Four rules: subrel does the actual edge lift; the
  ;; three closure-emit-* rules turn one (closure R R*) user
  ;; declaration into the relation-decl + transitive + subrel
  ;; setup. Multi-emission via decomposition (M1 :assert is
  ;; single-fact; (and …) in :assert deferred until 3+ rules
  ;; need it).
  ;; ─────────────────────────────────────────────────────────
  (rule subrel (?Sub ?Super)
    :match  (?Sub ?a ?b)
    :assert (?Super ?a ?b)
    :priority 100
    :why    "{?Sub} ⊆ {?Super}: lift edge")

  (rule closure-decl-relation ()
    :match  (and (closure ?R ?R*) (relation ?R ?a ?b))
    :assert (relation ?R* ?a ?b)
    :priority 100
    :why    "{?R*} inherits {?R}'s signature")

  (rule closure-emit-transitive ()
    :match  (closure ?R ?R*)
    :assert (transitive ?R*)
    :priority 100
    :why    "{?R*} declared transitive")

  (rule closure-emit-subrel ()
    :match  (closure ?R ?R*)
    :assert (subrel ?R ?R*)
    :priority 100
    :why    "{?R} ⊆ {?R*}"))
```

## 5 — Worked example: domain-elimination firing

Setup (post-B1 zebra2):
- `(relation house-color House Color)`
- `(functional house-color 0 1)` + `(total house-color 0)`
- `(is-a House-1 House)`, `(is-a Yellow Color)`, etc.
- type-exclusivity has written: `(not (house-color House-1 Red))`,
  `(not (house-color House-1 Green))`,
  `(not (house-color House-1 Ivory))`,
  `(not (house-color House-1 Blue))`.

Saturation:
1. **Priority 100** — reflection rules fire on
   `(relation house-color House Color)` →
   `(arg-type house-color 0 House)`,
   `(arg-type house-color 1 Color)`. Closure rules see no
   `(closure …)` activator; silent.
2. **Priority 110** — typecheck-arg-0/1 scan every
   `(house-color ?a ?b)` fact. For `(house-color House-1 Yellow)`,
   `absent (is-a House-1 House)` fails (the fact is stored), so
   the rule is silent. ✓ `no-room-left` checks every `?a`'s slot
   set; `(is-a Yellow Color)` is open (no `(not (house-color
   House-1 Yellow))` stored), so the forall fails → silent. ✓
3. **Priority 400** — `domain-elimination`'s outer pattern
   matches `?R = house-color, ?a = House-1, ?v = Yellow,
   ?OT = House, ?VT = Color`. The forall (desugared to
   `absent (and (is-a ?v_other Color) (neq ?v_other Yellow)
   (absent (not (house-color House-1 ?v_other))))`) iterates
   `?v_other ∈ {Red, Green, Ivory, Blue}`; for each, the
   stored neg exists, so `absent (not …)` fails for every
   binding, so the outer `absent` passes. forall passes →
   rule fires → `(house-color House-1 Yellow)` asserted.

Trace text:
> **domain-elimination** — every Color except Yellow
> excluded for House-1 — therefore (house-color House-1 Yellow).

Matches the idea-08 "Therefore the first house is yellow"
line with the 4 stored negatives + the Yellow is-a fact as
provenance premises.

  the closure facility decomposes into 3 always-active rules
  + `subrel`. Add to F4 / P1.8 stdlib-polish themes.
- **Q-S1.5.8c.2 — `:on` / `:to` keyword sugar.** Reader-friendly
  for n-ary; not needed for binary (positional is fine).
  Defer to P1.8 stdlib polish if user feedback wants it.
- **Q-S1.5.8c.3 — `(closure A B)` as parser sugar.** If
  multi-emission `:assert` lands (Q-S1.5.8c.1), the closure
  facility collapses to 1 multi-fact rule and a parser-level
  `(closure …)` declaration form is *also* feasible. Future
  syntax sugar; out of M1.
- **Q-S1.5.8c.4 — n-ary relations.** Grammar accepts
  `(relation R T1 T2 … Tn)` for any arity; M1's rule library
  is binary-only. When variadic match (Q33) lands, the
  reflection rules become one variadic rule and arity gets
  re-introduced. F4 / followups scope.
- **Q-S1.5.8c.5 — `(open P)` and hypgen.** With `(open P)`
  matchable in `:match`, hypothesis generation that's
  currently Python (P1.5 S1.5.4 / S1.5.6b machinery) could
  be expressed as hrules. Not M1-blocking — current hypgen
  works — but worth recording as a refactor candidate. Park
  to P1.9 (hypothesis-loop followups).
- **Q-S1.5.8c.6 — `sibling-exclusive` cross-boundary on
  transitive is-a.** zebra2 still avoids `(transitive is-a)`
  because of the cross-boundary blowup. Not an M1
  problem under B1 (direct-parent-types avoids it). The
  long-term fix — scope `sibling-exclusive` to direct
  parents only — is parked to F4.

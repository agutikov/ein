# S1.5.5 — Closure auto-inference  ·  deferred to P1.8

**Status:** **deferred to P1.8** — 2026-05-22.

Closure auto-inference (`infer-closure-from-functional` + the
`:enable-auto-closure` flag) is no longer a P1.5 stage. It is a
**stdlib-rule** concern and was re-homed to
[P1.8 Theme A — modules, imports + standard library](../p1.8_ein_lang_modules/README.md#standard-library),
together with the other rule-library / language features
discussed 2026-05-22 (`converse`, the `imply` family, general
totality, reflective rule-implication, type/domain matching).

Two corrections from that review travelled with it and are
recorded in P1.8:

- the rule must be **parameter-less** — `(rule infer-closure ()
  …)`; the original `(?R)` sketch here would have compiled to
  zero activator plans and never fired (the engine reads a
  non-empty param list as "needs activator facts");
- `functional ⇒ closed` is too weak — the principled witness is
  **`functional ∧ total ⇒ closed`**; see P1.8 for the
  two-senses-of-"closed" framing.

What stays in P1.5: the **manual** `(closed R)` activator and its
recognition path ([S1.5.4 T1.5.4.1](s1.5.4_hypgen_improvements.md#task-t1541--relation-closure-declarative-closed-r-activator),
already shipped) — only the *automatic inference* of `(closed R)`
deferred. The hypothesis generator reads `(closed R)` the same
way regardless of who wrote the fact.

Pre-deferral content (T1.5.5.1–.3, acceptance, open questions) is
preserved in git history.

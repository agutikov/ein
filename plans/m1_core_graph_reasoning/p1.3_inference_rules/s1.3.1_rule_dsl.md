## Context

The S1.3.0 review reframed S1.3.1 substantially. Key constraints
the rewrite honours:

- **No duplicate Pattern / Rule / Registry classes** — `kb.Pattern`
  (P1.2 S1.2.1) and `kb.Rule` (P1.2 S1.2.1) are the canonical
  data; `kb.rules` is the registry. The matcher *consumes* them.
- **Compile, don't walk** — Q29 resolved to option B with
  per-(rule, activator-binding) compile. Each activator fact
  (e.g. `(transitive co-located)`) binds the rule's parameters
  *before* matching; the compiler bakes concrete relation names
  into the program.
- **Predicate registry, not `:where`** — Q32 resolved to drop
  `:where`; built-in predicates (`eq`, `neq` per Q33) live in
  `inference/predicates.py` and are consulted by both the loader
  (suppress auto-vivification) and the matcher (dispatch).
- **`{?var}` substitution** — Q31 fixed the `:why` template
  grammar.
- **Flat `inference/` module** — Q39 picked option A.
- **Nested-fact patterns** — Q40 Option A; the matcher must handle
  `Fact`-valued args, both at compile time (sub-pattern) and at
  runtime (recursive unification).

The DSL **surface syntax** lives in the IR
(P1.1 S1.1.1 T1.1.1.2); this stage implements the **runtime**
(compile + match + predicates).

## Tasks

### Task T1.3.1.1 — Built-in predicate registry

`src/ein/inference/predicates.py`. The single source of truth
for built-in predicates available inside `:match` clauses.

```python
# inference/predicates.py
Predicate = Callable[[dict[str, Any], tuple[Any, ...]], bool]

_REGISTRY: dict[str, Predicate] = {
    "eq":  lambda b, args: _resolve(b, args[0]) == _resolve(b, args[1]),
    "neq": lambda b, args: _resolve(b, args[0]) != _resolve(b, args[1]),
}

def is_predicate(name: str) -> bool: ...
def get(name: str) -> Predicate | None: ...
def register(name: str, fn: Predicate) -> None: ...   # for followups
```

Q33's M1 contents: `eq` + `neq` **only**. No numeric, set,
cardinality, ordinality, or variadic primitives. `not` is **not**
a predicate — it's a matcher-level structural wrapper (see
T1.3.1.4).

### Task T1.3.1.2 — Loader change: consult registry before auto-vivification

`src/ein/kb/from_ir.py`. When an unknown head appears inside a
`:match` body, check `predicates.is_predicate(head)` first; only
auto-vivify as an open-world `Relation` if absent from the registry.

Prevents phantom `eq` / `neq` relations from appearing in
`kb.relations`.

### Task T1.3.1.3 — Per-(rule, activator) compiler

`src/ein/inference/compile.py`:

```python
@dataclass(frozen=True)
class JoinPlan:
    steps: tuple[PlanStep, ...]
    # PlanStep = Scan | Join | Guard | AssertSpec

@dataclass(frozen=True)
class Scan:
    relation: str            # baked-in by activator
    arg_slots: tuple[str | int | NestedPattern, ...]

@dataclass(frozen=True)
class Join:
    relation: str
    arg_slots: tuple[str | int | NestedPattern, ...]
    shared_vars: frozenset[str]

@dataclass(frozen=True)
class Guard:
    predicate: str           # name in inference/predicates.py
    args: tuple[str | int, ...]

@dataclass(frozen=True)
class NestedPattern:        # Q40 Option A — relational-node patterns
    relation: str
    arg_slots: tuple[str | int | "NestedPattern", ...]

def compile_pattern(
    expr: IRNode, bindings: dict[str, str | int | Fact],
) -> JoinPlan: ...
```

`compile_pattern` walks the `:match` IR expression *once*,
substitutes the activator's bindings into rule parameters
(`?rel` → concrete relation name), and emits a sequence of
opcodes. Nested SForms in arg positions compile to `NestedPattern`
opcodes (sub-patterns that unify against `Fact`-valued args at
runtime).

For each rule in `kb.rules` and each activator fact in
`kb.facts.by_relation[rule.name]`, the engine compiles one program
keyed by `(rule.name, activator.args)`.

### Task T1.3.1.4 — Runtime matcher

`src/ein/inference/match.py`. Executes a `JoinPlan` against a
`KnowledgeBase`. Yields `dict[str, str | int | Fact]` bindings.

Algorithm:

```python
def run(plan: JoinPlan, kb: KnowledgeBase) -> Iterator[dict]:
    yield from _run_steps(plan.steps, {}, kb)

def _run_steps(steps, bindings, kb):
    if not steps: yield bindings; return
    step, *rest = steps
    match step:
        case Scan(relation=r, arg_slots=slots):
            for fact in kb.facts.by_relation.get(r, ()):
                if (new := _bind_args(slots, fact.args, bindings)):
                    yield from _run_steps(rest, new, kb)
        case Join(relation=r, arg_slots=slots, shared_vars=svs):
            for fact in kb.facts.by_relation.get(r, ()):
                if (new := _bind_args(slots, fact.args, bindings,
                                       require=svs)):
                    yield from _run_steps(rest, new, kb)
        case Guard(predicate=p, args=a):
            if predicates.get(p)(bindings, a):
                yield from _run_steps(rest, bindings, kb)
```

Unification (`_bind_args`) is recursive:
- An atomic slot (str/int) unifies with an atomic arg by equality.
- A var slot binds on first hit; on subsequent hits, must match
  current binding.
- A **NestedPattern slot** unifies with a `Fact`-valued arg iff
  their `(relation_name, args)` tuples unify pointwise
  (recursively). This is Q40 Option A's matcher extension.

**`(not P)` wrapper** (negation-as-failure): when the compiler
sees `(not <sub-form>)` as a `:match` premise, it emits a
`NegativeGuard` opcode that runs `_run_steps([sub_plan], bindings,
kb)` and yields `bindings` iff the sub-plan produces **zero**
matches. (Not a predicate-registry entry — it's a structural form.)

### Task T1.3.1.5 — Firing record + assert

`src/ein/inference/firing.py`:

```python
@dataclass(frozen=True)
class Firing:
    rule: str                       # kb.Rule.name
    activator: tuple[str, ...]      # the activator fact's args
    bindings: dict[str, Any]
    derived: Fact                   # the new fact added to reasoning
    premises: tuple[Fact, ...]      # facts consumed by the match
```

The asserter (called by the engine driver) takes the rule's
`:assert` clause + bindings + premises, substitutes bindings into
the assert pattern (producing a `Fact`), and adds it to the KB:

```python
kb.add_fact(
    fact,
    layer=Layer.REASONING,
    provenance=Provenance.from_rule(
        rule=firing.rule,
        premises_raw=tuple((p.relation_name, p.args) for p in firing.premises),
        bindings=tuple(firing.bindings.items()),
    ),
)
```

For an `:assert (not X)` (negative-fact assertion), the resulting
fact has `relation_name="not"` and `args=(<X-fact>,)` — a nested
`Fact` arg per Q40. The contradiction detector (P1.4 collapse, see
P1.5) finds `(X, (not X))` pairs.

### Task T1.3.1.6 — `:why` template substitution

`src/ein/inference/why.py`. Tokenises the `:why` string by
scanning for `{` … `}` delimited sequences with a **leading `?`**
(Q31). Inside the sequence, the body is a var-name; substitute
with the binding. Bare `{x}` without `?` stays literal.

```python
def render_why(template: str, bindings: dict[str, Any]) -> str: ...
```

Example: `"{?rel} is transitive: {?a} →{?rel}→ {?b}"` with
bindings `{rel: 'co-located', a: 'Norwegian', b: 'House-1'}` →
`"co-located is transitive: Norwegian →co-located→ House-1"`.

### Task T1.3.1.7 — Engine driver shim

`src/ein/inference/engine.py`:

```python
class Engine:
    def __init__(self, kb: KnowledgeBase): ...
    def compile_all(self) -> None:
        """Walk kb.rules × activators; cache one JoinPlan per (rule, activator)."""
    def step(self) -> Firing | None: ...
    def saturate(self) -> Iterator[Firing]: ...   # → S1.3.3
```

S1.3.3's saturation loop builds on `step()`. The compile cache
lives on the engine instance; on `kb.fork()`, the fork inherits
the cache by reference (programs are pure functions of rule +
bindings).

### Task T1.3.1.8 — Migration: zebra.ein's `transitive` to positional `(neq …)`

Rewrite zebra.ein's `transitive` rule body from:

```lisp
:match  (and (?rel ?a ?b)
             (?rel ?b ?c)
             :where (neq ?a ?c))
```

to:

```lisp
:match  (and (?rel ?a ?b)
             (?rel ?b ?c)
             (neq ?a ?c))
```

Q32 dropped `:where`; the matcher dispatches `neq` via the
predicate registry. One line of diff.

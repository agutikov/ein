## Goal

Lock the refactor with the user-named acceptance: a single test runner, no
root `pytest.ini`, and PyPy tests that solve all three zebra2 variants
**completely with expected results**, plus a zebra2 PyPy speedup check.

## Tasks

- **T1 — remove root `pytest.ini`.** Relocate any needed pytest config into
  `ein.py/pyproject.toml` `[tool.pytest.ini_options]` (or `ein.py/pytest.ini`)
  so discovery from the package root is preserved. Verify nothing else reads
  the root file.
- **T2 — `run_tests.sh`** (repo root, `chmod +x`, `--help`): runs the full
  suite; prefers `.venv-pypy` when present, falls back to `.venv`/system;
  passes through extra args (e.g. `-k`, `EIN_RUN_SLOW=1`); echoes which
  interpreter it used. Project convention: one committed script, sensible
  default, no flag without help.
- **T3 — 3-variant acceptance test** (`tests/integration/test_zebra_three_classes.py`,
  superseding S1.7.3 T1.7.3.2): under the unified API —
  - `zebra2.ein` → **Unique** (`k=1`); the 25/25 model; goal answer
    "Japanese/zebra, Norwegian/water".
  - `zebra2-minus-15.ein` → **Ambiguity** (`k>1`); residual open set names the
    dropped-(15) dependence.
  - `ein-bugs/zebra2-bad.ein` → **Contradiction** (`k=0`); 2–3 edge core with
    the injected fact + condition (6).
- **T4 — invariant test.** Across the variant suite: no SAT → `Contradiction`,
  no UNSAT → `Solution`.
- **T5 — PyPy speedup check.** A test (or `run_tests.sh --bench` mode /
  committed bench) asserting/reporting zebra2 solves faster under PyPy than
  CPython (baseline ~5 s CPython / ~1.6–1.9 s PyPy from S1.5b.8). Gate the slow
  full-solve under `EIN_RUN_SLOW` if it exceeds the default <30 s budget.

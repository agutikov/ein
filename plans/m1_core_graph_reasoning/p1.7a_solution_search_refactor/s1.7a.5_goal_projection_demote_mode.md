**Phase:** P1.7a. **Estimate:** 1 day. **Status:** planned.
**Depends on:** [S1.7a.3](s1.7a.3_one_search_result_from_k.md),
[S1.7a.4](s1.7a.4_stop_policy.md).

## Goal

Finish demoting "mode": `:goal` becomes a projection over solution node(s),
not a termination driver; `:mode`/`--mode` become (stop preset × projection),
not a search selector. Revert the dead S1.7.3 completeness-gate experiment.

## Tasks

- **T1 — `goal_bindings(model_kb, goal)`.** Extract the matcher body from
  `is_solved` (`verdict.py:116`) into a pure projection that returns bindings;
  delete the mode-aware count/termination logic. `is_solved` as a terminator
  is gone (S1.7a.3 T3 already removed it from the loop).
- **T2 — `:mode` semantics.** In the query block, `:mode` selects a *preset*
  (`solve` → stop_after=1 + goal projection; `gaps` → exhaust + residual-open
  projection; `contradictions` → exhaust + core projection). It does not pick a
  search. Keep `Mode` only as this preset label (or remove and use strings).
- **T3 — revert the experiment.** Per the S1.7a.1 T4 inventory, remove
  `SolverConfig.solve_requires_complete_model` + the monotonic accept-point
  edits + CLI use; **keep** the harmless/correct pieces (`goal_bindings`,
  root unsat-core, `trace/answer.py` if present, `demo/compare_engines.py`).
- **T4 — projection tests.** zebra2 Unique → goal projects
  `{?h_water: House-1, ?h_zebra: House-5}`; with the `nation-loc` extension,
  "Norwegian/Japanese". zebra2-minus-15 Ambiguity → goal answers that vary
  across models are reported as the gap.

## Acceptance

- `:goal` never affects search/termination; only the end projection.
- `solve_requires_complete_model` and its edits are gone; suite green.
- Goal projection correct on zebra2 + minus-15; `ruff` green.

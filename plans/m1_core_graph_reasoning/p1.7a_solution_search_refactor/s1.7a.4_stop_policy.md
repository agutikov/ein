# S1.7a.4 — Stop policy (`stop_after`)

**Phase:** P1.7a. **Estimate:** ½ day. **Status:** **shipped 2026-05-31** — `stop_after: int | None` on `solve()`; `stop_after=1` is the sound fast path (sets `stats.exhausted=False` so a k=1 there reads as "a model", not certified-unique); the acceptance uses `stop_after=2` to prove minus-15 ambiguity cheaply (commit `4297deb`).
**Depends on:** [S1.7a.3](s1.7a.3_one_search_result_from_k.md).
**Design:** [`refactoring_design.md` §4](refactoring_design.md).

## Goal

Make "how many solutions to find before stopping" an explicit, orthogonal
user knob — the thing that "give me the first solution" and "give me 42"
actually are. Couple it to result honesty via `exhausted`.

## Tasks

- **T1 — `stop_after: int | None`** on the unified `search` (None = exhaust).
  After recording a *new* (deduped) solution node, if
  `len(solution_nodes) >= stop_after`, finalise with `exhausted=False`,
  `stopped_by="stop_after"`.
- **T2 — sound "first solution" fast-path.** `stop_after=1` stops on the
  first *complete ∧ consistent* node — replacing today's unsound
  first-goal-match stop. Verify it returns the real model on zebra2, never the
  partial dead-end.
- **T3 — honesty coupling.** `verdict_of` already caps `¬exhausted` runs at
  "uncertified": a `stop_after`-stopped run with `k=1` is **not** reported as
  proven-unique. Document and test this (a `stop_after=1` solve states "a
  solution"; certifying uniqueness requires `stop_after=None`).
- **T4 — budgets stay orthogonal.** `max_set_size`/`max_time`/`max_enterings`
  unchanged; a budget abort sets `stopped_by="budget"`, `exhausted=False`.

## Acceptance

- `stop_after ∈ {1, 2, None}` honoured on a multi-solution fixture
  (e.g. `branching/04` → 2 nodes): `1` stops at 1, `None` finds 2.
- zebra2 `search(stop_after=1)` → the correct 25/25 model, reported as a
  (not-yet-unique-certified) solution; `search(stop_after=None)` → Unique.
- `ruff` green; suite green.

   S1.6.5 shape is the gate** — the frozen checklist

   always; the 9 target rules actually fire under `EIN_RUN_SLOW`). **Do
   not build the per-move YAML harness.**
2. The engine/CLI evolved: `solve` runs `gaps_solve` / `contradictions_solve`
   and emits an **inline-DOT markdown trace**; there is no `query`
   subcommand, no `--diagrams=DIR` flag, and — critically — **no CLI path
   that runs `monotonic_solve` and prints the answer in words**.

So this stage's live remainder is: the three-task-class integration test,
the human-answer surfacing + CLI reconciliation, and the M1-done closer.

### Task T1.7.3.2 — Three-task-class integration test
`tests/integration/test_zebra_three_classes.py`:
- **SOLVE** — `monotonic_solve(zebra2)` → `Solution`; goal bindings
  include the canonical answer (`?h_water=House-1`, `?h_zebra=House-5`).
  (Already pinned by `test_monotonic_skeleton`; re-assert at integration
  level.)
- **GAPS** — `gaps_solve(zebra2-minus-15)` → ≥ 1 diverging goal node; name
  the condition the gap depends on.
- **CONTRADICTIONS** — `contradictions_solve(zebra2-bad)` → a 2–3 edge
  unsat core including the injected fact + the colliding condition.
(Needs the two fixtures from [S1.7.1](s1.7.1_zebra_encoding.md).)

### Task T1.7.3.3 — Answer surfacing + CLI reconciliation
Acceptance wants `solve` to *emit the unique solution in words* ("The
Japanese keeps the zebra; the Norwegian drinks water"). Today nothing does
— `monotonic_solve` returns the solved KB; `bench_monotonic.py` prints
goal bindings; the markdown trace shows a DOT grid.
- Add a CLI **answer path**: a `solve` mode (e.g. `--mode=solve`) that
  runs `monotonic_solve` and renders the goal bindings as an English
  answer line.
- Compose with [S1.7.5](s1.7.5_query_semantics_who_vs_where.md): extend
  zebra2's goal with the `nation-loc` projection so the answer reads
  "Norwegian / Japanese", not "House-1 / House-5".
- Reconcile the README acceptance command spec with the real CLI
  (`query`→`solve --mode`; drop `--diagrams=DIR`, output is inline /
  `--no-diagrams`).

### Task T1.7.3.4 — Performance note
zebra2 solves in ~5 s CPython / ~1.6 s PyPy. The original "< 2 s" target
is **PyPy-only** today. Either relax to "< 2 s PyPy / < 6 s CPython" or
open a profiling task; not a blocker.

### Task T1.7.3.5 — Trace narrative review

English; no raw node ids; hypothesis branches fold; symmetry-break appears
once; final line names the answer. Capture notes as `<!-- review: … -->`.

# S1.7.22 — engine-internal protocol strings (survey)

**Phase:** P1.7 (bootstrapping) — per-name kernel-purity analysis.
**Parent:** [S1.7.7](s1.7.7_kernel_purity_analysis.md) (inventory) — **Tier B**.
**Class:** engine-internal enums-as-strings — **not ein-lang names**.
**Verdict:** **not a kernel-purity question** — these are the engine's own
control vocabulary; for each the only "de-naming" is *string → enum*. One
survey file rather than one-per-name (per-name files would be thin and
non-kernel); split out on request.
**Status:** analysis (survey).

These strings never appear in a `.ein` file and no rule matches them. They
are internal protocol tags. For each: §1 role, §2 removal = impossible
(the engine needs the distinction) / the real question is representation,
§3 not ein-expressible (engine vocabulary), §4 replace string with a typed
value. Grouped by mechanism.

## A. Provenance kinds — `source` / `rule` / `hypothesis` / `rejected`

- **Where:** `back_prop.py:89` (`_SPECULATIVE_KINDS = {"hypothesis",
  "rejected"}`), `back_prop.py:119` (`prov.kind == "rule"`),
  `state_dump.py:76–88` (`source`/`rule`/`hypothesis` render arms).
- **Role:** `Provenance.kind` tags *how a fact got here* — given
  (`source`), rule-derived (`rule`), speculated (`hypothesis`), or
  refuted-speculation (`rejected`). Drives trace rendering, the
  back-prop "is this fact speculative?" test, and DAG walks.
- **De-naming:** a `ProvKind` enum; `_SPECULATIVE_KINDS` becomes
  `kind.is_speculative`. The word `"hypothesis"` here is **distinct** from
  the bookkeeping head ([S1.7.19](s1.7.19_bookkeeping_heads.md)) — same
  string, different mechanism; an enum would disambiguate them.

## B. Lattice / SearchResult node kinds — `alive` / `dead-pre` / `dead-post` / `solution` / `dead`

- **Where:** `monotonic/lattice.py:121,145`; `solver.py` (`result.kind in
  ("dead-pre","dead-post")`, `:717,727,1013,…`); `sanity.py:118,134`;
  `commitment.py:57,90,93,96,162`; `state_dump.py:259,616,680`.
- **Role:** the search-node verdict lattice — a candidate is `alive`,
  `dead-pre` (doomed before saturation, by lookahead/nogood), `dead-post`
  (doomed after saturation), or a `solution`. The promotion invariant
  (every candidate becomes alive-or-dead, never absent) is stated over
  these tags.
- **De-naming:** a `NodeKind`/`Verdict` enum. These are the most
  performance- and invariant-critical internal strings; typing them would
  make the lattice's monotonic promotion statically checkable. Pure
  engine-internal; no ein surface.

## C. Contradiction kinds — `direct` / `pair`

- **Where:** `contradiction.py:86` (`Literal["pair","direct"]`),
  `:162,189`.
- **Role:** distinguishes the two ⊥ shapes — a `(false …)` fact
  (`direct`) vs a same-layer `(X,(not X))` (`pair`); selects the
  unsat-core seed (`witness`). Already a `Literal`, half-typed.
- **De-naming:** promote the `Literal` to an enum. Tied to the
  [`false`](s1.7.13_false.md) / [`not`](s1.7.12_not.md) ⊥ calculus but is
  itself just a discriminator.

## D. Heuristic mode strings

- **apriori order:** `lex` / `score-sum` / `canonicalise`
  (`apriori.py:156,159,200`).
- **hypgen order:** `most-constrained` / `popularity` / `branch-info` /
  `popularity+branch-info` (`hypgen.py:405–440`, `config.py:86–171`).
- **Role:** select the candidate-ordering heuristic — *which* alive
  hypothesis to branch on first. Pure **policy**: affects wall-time and
  trace order, never the solution set (soundness/completeness invariant
  to the choice).
- **De-naming:** a strategy registry (like
  [predicates](s1.7.18_eq_neq.md)) keyed by enum, or pluggable
  comparator objects. These are the *most* legitimately
  string-configured (they are user-facing `config.py` knobs) — an enum +
  registry is the natural form; a puzzle could even supply its own
  ordering as data ([F5](../../followups/f5_rules_as_data.md)).

## E. Misc tags

- **NameRef category** `object` (`hypgen.py:511`) — the entity taxonomy
  (object / type / relation) hypgen filters fillers by; an enum on
  `NameRef`.
- **Node label** `dead-by-nogood` (`nogoods.py:116`) — a SearchNode label
  for a candidate killed by a learned clause without `try_branch`; a
  display string, subsumable into the NodeKind enum (B).
- **Commitment outcomes** `considered` / `fire` (`firing.py:49,171`) —
  firing-log event tags; enum.

## Cross-cutting decision (to record in [S1.7.7 T1.7.7.5](s1.7.7_kernel_purity_analysis.md))

None of A–E is an ein-lang name; **none is a kernel-purity removal**. The
uniform recommendation is **string → typed value** (enum / `Literal` /
strategy object), which buys exhaustiveness checking and disambiguates
collisions (the two `"hypothesis"`es, A vs S1.7.19). This is
[P1.7b](../p1.7b_review_and_refactor/)-flavoured code hygiene, explicitly
**out of scope** for the kernel-language boundary in
[S1.7.7 T1.7.7.5](s1.7.7_kernel_purity_analysis.md) §6 — recorded here so
the inventory is genuinely complete (every hardcoded string accounted
for), not because these names leave the language.

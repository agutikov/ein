# S1.7.1 — Zebra IR + ontology

**Phase:** P1.7 (bootstrapping).
**Estimate:** 2-3 days.
**Depends on:** P1.1 IR + P1.2 graph + P1.4 constraints.

## Context

The canonical 15-condition Zebra puzzle (the version
[Wikipedia](https://en.wikipedia.org/wiki/Zebra_Puzzle)). We encode
it three times to stress different engine paths:

1. `zebra.ein` — the canonical puzzle. Expected: unique solution.
2. `zebra-minus-15.ein` — delete condition (15). Expected:
   ambiguous; GAPS mode reports diverging goals.
3. `zebra-bad.ein` — add a self-contradictory fact. Expected:
   CONTRADICTIONS mode produces an unsat-core including the bad
   fact.

## Tasks

### Task T1.7.1.1 — Ontology

`examples/zebra/ontology.ein`:

```ein
(ontology
  (type House     :positional :bounds (1 5))
  (type Color)
  (type Nationality)
  (type Pet)
  (type Drink)
  (type Cigarette)

  (instance House_1 House) ... (instance House_5 House)
  (instance Red     Color) (instance Green Color) (instance Ivory Color)
  (instance Yellow  Color) (instance Blue  Color)
  (instance Englishman Nationality) (instance Spaniard Nationality)
  (instance Ukrainian  Nationality) (instance Norwegian Nationality)
  (instance Japanese   Nationality)
  ...

  (relation lives-in   Nationality House     :cardinality (1..1))
  (relation has-color  House Color           :cardinality (1..1) :functional)
  (relation drinks     Nationality Drink     :cardinality (1..1))
  (relation smokes     Nationality Cigarette :cardinality (1..1))
  (relation keeps      Nationality Pet       :cardinality (1..1))

  (a-priori inverse-of left-of right-of)
  (a-priori symmetric  next-to)

  (constraint all-different-color   :over (color House_*)        :pairwise-distinct)
  (constraint all-different-nation  :over (lives-in * House)     :pairwise-distinct)
  (constraint all-different-pet     :over (keeps * *)            :pairwise-distinct)
  (constraint all-different-drink   :over (drinks * *)           :pairwise-distinct)
  (constraint all-different-smoke   :over (smokes * *)           :pairwise-distinct)

  (symmetry house-numbering :group S2 :on House
            :note "doesn't matter which way numbered"))
```

### Task T1.7.1.2 — Facts (15 conditions)

`examples/zebra/facts.ein`. Each of the 15 conditions becomes a
fact with `:source "condition (N)"`:

```ein
(facts
  (rel lives-in Englishman (and has-color Red)        :source "condition (2)")
  (rel keeps    Spaniard   Dog                         :source "condition (3)")
  (rel drinks   Ukrainian  Coffee                      :source "condition (4)")
  ; (5) "The green house is immediately to the right of the ivory house."
  (immediately-right-of (the-house-with-color Green) (the-house-with-color Ivory)
                        :source "condition (5)")
  ; (10) "The Norwegian lives in the first house."
  (at-position (the-house-of Norwegian) 1               :source "condition (10)")
  ...)
```

The Lisp-style `(the-house-with-color Green)` is a *description* —
the IR's "the X with property P" idiom, equivalent to a definite
description. Loader resolves to a fresh variable + constraint
linking it to the unique satisfier.

### Task T1.7.1.3 — Query forms

`examples/zebra/query.ein`:

```ein
(query :mode solve
       :goal (and (keeps ?n Zebra) (drinks ?m Water))
       :explain true)
```

The query establishes the *goal* nodes — what the solver must
report at quiescence — and turns on full trace.

### Task T1.7.1.4 — Variants

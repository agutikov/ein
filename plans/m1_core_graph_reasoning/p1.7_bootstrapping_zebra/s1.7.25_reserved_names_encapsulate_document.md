# S1.7.25 — Kernel reserved names: encapsulate, isolate, document

**Phase:** P1.7 (bootstrapping) — kernel-purity **consolidation** (the
"what *stays* in the kernel, made clean + documented" counterpart of
[S1.7.23](s1.7.23_retire_kernel_type_system.md) / [S1.7.24](s1.7.24_dehardcode_symmetric.md),
which remove `is-a`/`T`/`symmetric`).
**Estimate:** ~2-3 days (light encapsulation refactor + two docs).
**Depends on:** [S1.7.7](s1.7.7_kernel_purity_analysis.md) (the inventory)
and its per-name children S1.7.10–.22.
**Status:** **DONE (2026-06-02)** — kept-kernel vocabulary encapsulated +
documented; pure refactor, full fast suite green (1036✓), ruff clean.
**Origin:** user direction 2026-06-01 — fix the M1 kernel boundary: a
defined set of primitives stays, *encapsulated* (the `eq`/`neq`
[`predicates.py`](../../../ein.py/src/ein_bot/inference/predicates.py)
pattern is the model) and *documented*; the rest are reserved engine
strings for M1.

## The M1 decision (user-fixed)

After `is-a`/`T` ([S1.7.23](s1.7.23_retire_kernel_type_system.md)) and
`symmetric` ([S1.7.24](s1.7.24_dehardcode_symmetric.md)) leave, the
kernel's remaining reserved names are fixed for M1 in three buckets:

1. **Kept primitives — encapsulate + isolate + document.** The rule-body /
   ⊥ calculus: **`false` `not` `and` `or` `absent`** plus the predicates
   **`eq` `neq`**. These are genuine inference/rule primitives and stay
   kernel; the *only* requirement is that each one's semantics live in a
   single isolated place — exactly as `eq`/`neq` already do in
   [`predicates.py`](../../../ein.py/src/ein_bot/inference/predicates.py).
   (Analyses: [S1.7.12](s1.7.12_not.md)–[S1.7.17](s1.7.17_absent.md),
   [S1.7.13](s1.7.13_false.md), [S1.7.18](s1.7.18_eq_neq.md).)
2. **Kept mechanism — `closed`.** Auto-inference (`emit_closed`) + the
   hypgen suppression effect **remain in the kernel for M1**, but
   encapsulated, isolated, and documented (analysis:
   [S1.7.10](s1.7.10_closed.md)).
3. **Reserved engine strings (M1).** Bookkeeping carrier heads
   (`hypothesis`, `contradiction-under`), the `hypothesis-relations` query
   keyword, and the task-class entries (`solve` / `gaps` /
   `contradictions`) are **reserved engine strings** — documented, not
   refactored or de-hardcoded this milestone (analyses:
   [S1.7.19](s1.7.19_bookkeeping_heads.md), [S1.7.20](s1.7.20_hypothesis_relations.md),
   [S1.7.21](s1.7.21_task_class_entries.md), [S1.7.22](s1.7.22_engine_internal_strings.md)).

**Skipped here:** `open` / `forall` — they become **P1.8 macros**
([S1.5.9](../p1.8_ein_lang_modules/s1.5.9_ein_lang_macros.md)); no kernel
work this stage (analyses [S1.7.11](s1.7.11_open.md) / [S1.7.16](s1.7.16_forall.md)).

## Tasks

### T1.7.25.1 — Encapsulate the rule-body / ⊥ primitives
Create a single authoritative module — the analogue of `predicates.py`,
e.g. `inference/primitives.py` — that **declares** each structural
primitive (`false` `not` `and` `or` `absent`) with: its name constant,
arity, semantic role (one line), and the canonical site that implements
it. Replace the scattered bare string literals
(`compile.py` `== "and"/"or"/"absent"/"forall"`, `contradiction.py`
`_facts_by_relation["false"]/["not"]`, `lookahead.py`, `match.py`,
`back_prop.py`/`solver.py` `"not"` writes) with references to these
constants, so the kernel's primitive vocabulary is **declared once**, not
implicit in `== "…"` checks. Deep behaviour stays where it is (compiler /
matcher / detector); this isolates the *names + meaning*, mirroring how
`predicates.is_predicate` / `get` centralise `eq`/`neq`. (Full behavioural
consolidation is a larger, optional follow-on.)

### T1.7.25.2 — Encapsulate + isolate `closed`
`inference/closed.py` already isolates `emit_closed` /
`producible_relations`. Finish: hoist the `"closed"` literal to a single
module constant; have hypgen's suppression read it via that constant
(`hypgen.py:328` `_is_closed`); confirm the auto-inference + hypgen-effect
boundary is in one module. **Keep the kernel behaviour** (S1.7.10: it is
load-bearing for hypgen scoping / NAF soundness this milestone). Document
it as a *kept* kernel mechanism with the S1.7.10 rationale; the de-hardcode
question stays parked (post-M1).

### T1.7.25.3 — Centralise + reserve the engine strings
Ensure each reserved engine string is a **named constant in one place**:
bookkeeping heads already are (`canon.BOOKKEEPING_HEADS`); give the
`hypothesis-relations` keyword and the task-class entries (`solve`/`gaps`/
`contradictions`) the same treatment (a constant / enum-ish single
source), without changing behaviour. Mark them **reserved for M1** — no
de-hardcoding (the de-naming routes stay in S1.7.19/.20/.21/.22 for
post-M1).

### T1.7.25.4 — Write `docs/kernel/ir/03-ein-lang/06_reserved_names.md`
The authoritative **surface-language** reserved-words reference (what a
puzzle author may write but not redefine). One table, each row: name ·
arity · semantic · where the engine handles it · status. Cover:
- rule-body / ⊥ primitives `false` `not` `and` `or` `absent` + predicates
  `eq` `neq` (point at `predicates.py` / `primitives.py`);
- `open` / `forall` — flagged *desugaring sugar → P1.8 macros*
  ([S1.5.9](../p1.8_ein_lang_modules/s1.5.9_ein_lang_macros.md));
- `closed` — author-writable `(closed R)`, usually auto-inferred;
- `hypothesis-relations` — the `(query …)` scoping keyword;
- `relation` / `rule` / `hrule` — the declarators (already in §6 below).
Cross-link the existing partial `§6 Reserved relation names` in
[01-ein-graph/03_ein_model.md](../../../docs/kernel/ir/01-ein-graph/03_ein_model.md)
(from the new doc it is `../01-ein-graph/03_ein_model.md`) and make
**this** the authoritative source (§6 becomes the graph-node subset
pointing here, T1.7.25.6).

### T1.7.25.5 — Write `docs/kernel/inference/reserved_engine_strings.md`
The **engine-internal** reserved-strings reference (engine vocabulary, not
author-facing): bookkeeping carrier heads (`hypothesis`,
`contradiction-under`), task-class entries (`solve`/`gaps`/
`contradictions`), and the protocol enums surveyed in
[S1.7.22](s1.7.22_engine_internal_strings.md) (provenance kinds, lattice
node kinds, contradiction kinds, heuristic modes). Each: meaning · where
used · "reserved engine string, M1; de-naming → string-to-enum, post-M1".
Document `closed`'s *engine effect* (auto-inference + hypgen suppression)
here, cross-linking the surface form in T1.7.25.4.

### T1.7.25.6 — Reconcile §6 of `03_ein_model.md`
Update the existing `§6 Reserved relation names` to drop duplication and
point at the two new authoritative docs (keep §6 as the graph-node
declarators `relation`/`rule`/`hrule` + a pointer). Note that `is-a`/`T`
(once [S1.7.23](s1.7.23_retire_kernel_type_system.md) lands) and
`symmetric` (once [S1.7.24](s1.7.24_dehardcode_symmetric.md) lands) are
**no longer reserved** — remove them from §6.

## Acceptance

- Each kept primitive (`false`/`not`/`and`/`or`/`absent`) has its name +
  semantic declared in one module (the `predicates.py`-style registry);
  the scattered `== "…"` literals reference it. `eq`/`neq` already do.
- `closed` is in one module with a single `"closed"` constant; behaviour
  unchanged; documented as kept.
- Bookkeeping heads / `hypothesis-relations` / task classes are single
  constants, documented as reserved M1 engine strings.
- `docs/kernel/ir/03-ein-lang/06_reserved_names.md` (surface) and
  `docs/kernel/inference/reserved_engine_strings.md` (engine) exist and
  are authoritative; `03_ein_model.md` §6 points at them and no longer
  lists the now-removable `is-a` (and never `symmetric`).
- A name is reserved iff it appears in exactly one of the two docs; no
  name is undocumented; the suite + `ruff` stay green (pure refactor).

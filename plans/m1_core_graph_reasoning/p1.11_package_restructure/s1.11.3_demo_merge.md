## Context

`ein.py/demo/bench_saturate.py` and `bench_solve.py` are
runnable only via `python ein.py/demo/X.py`. They contain
useful benchmark infrastructure that belongs *inside* the
package, exposed as CLI subcommands `ein bench solve …` /
`ein bench saturate …`.

## Tasks

### Task T1.11.3.1 — Decide module layout

Two options:

- **(a) Flat under cli/:** `src/ein/cli/bench_solve.py` +
  `bench_saturate.py`. Simple; consistent with one-file-per-
  subcommand from S1.11.4.
- **(b) Sub-package `bench/`:** `src/ein/bench/{solve,saturate}.py`
  with a thin wrapper in `src/ein/cli/bench.py`. Cleaner if
  bench grows past two scripts.

**Recommend (a)** for M1; promote to (b) if bench grows.

### Task T1.11.3.2 — Move + integrate

- **a.** `git mv ein.py/demo/bench_solve.py
  ein.py/src/ein/cli/bench_solve.py` (or sub-package per (b)).
- **b.** Refactor: remove `if __name__ == "__main__":` boiler
  if present; expose a `main(argv)` function.
- **c.** Wire as CLI subcommand: `ein bench solve PATH …`
  invokes `bench_solve.main([…])`.
- **d.** Same for `bench_saturate`.

### Task T1.11.3.3 — Remove the old demo directory

- **a.** `git rm -r ein.py/demo/`.
- **b.** Note in a top-level doc (README / CLAUDE.md / docs)
  that bench commands are now `ein bench …` not `python
  ein.py/demo/…`.

### Task T1.11.3.4 — Acceptance

- `ein bench solve examples/zebra2.ein` works and produces
  the same output as `python ein.py/demo/bench_solve.py
  examples/zebra2.ein` did pre-merge.
- `ein bench saturate examples/saturation/transitive/taxonomy.ein`
  same.
- `python ein.py/demo/...` paths return file-not-found.
- Documentation cross-references updated.

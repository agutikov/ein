# S1.11.4 — CLI split

**Phase:** P1.11 (package + CLI restructure).
**Estimate:** ½-1 day.
**Status:** sequential after S1.11.2 (codebase rename); can run
in parallel with S1.11.3 (demo merge) since both touch the
same `src/ein/cli/` directory — coordinate the layout choice.
**Depends on:** [S1.11.2](s1.11.2_codebase_rename.md).
**Blocks:** [S1.11.6](s1.11.6_acceptance.md).

## Context

`src/ein/cli.py` is a single file collecting all
subcommands. The file is fine today; the M1 feature surface
("solve", "query", "dump", "saturate", maybe "validate")
crowds the file as it lands. Pre-empting the crowding by
splitting into `src/ein/cli/{solve,saturate,query,dump,…}.py`
with a top-level dispatcher in `src/ein/cli/__init__.py`.

## Tasks

### Task T1.11.4.1 — Subcommand inventory

List every subcommand the current `cli.py` exposes. Likely:

- `ein solve <ein-file>` — solve a puzzle.
- `ein saturate <ein-file>` — run saturation only.
- `ein dump <ein-file>` — parse + round-trip dump.
- `ein query <ein-file>` — answer a query.
- `ein bench solve …` / `ein bench saturate …` (from S1.11.3).

Plus possibly: `ein validate`, `ein repl`, `ein trace`. Confirm
against the current `cli.py` before writing new files.

### Task T1.11.4.2 — Split

- **a.** `mkdir src/ein/cli` (or rename `cli.py` to
  `cli/__init__.py`).
- **b.** Per subcommand, create `cli/<name>.py` with a `main(args)`
  entry-point.
- **c.** `cli/__init__.py` becomes the dispatcher: parses the
  top-level argv, dispatches to the right subcommand module.
- **d.** Update `pyproject.toml [project.scripts]`:
  `ein = "ein.cli:main"`.

### Task T1.11.4.3 — Subcommand discovery

Optional polish: instead of hardcoding the dispatch table,
discover subcommands by iterating `cli/*.py` modules. Cuts
boilerplate when adding new subcommands.

- **a.** If using `click` or `typer`: the framework probably
  has a `Group`/`Typer.command` decorator that handles this.
- **b.** If hand-rolled argparse: a small `_register` helper
  collects subcommand modules.

### Task T1.11.4.4 — Acceptance

- `ein solve examples/zebra2.ein` works (smoke).
- `ein --help` lists all subcommands with descriptions.
- Each subcommand's `--help` works.
- No regression in CLI behaviour vs pre-split.

## Open questions

- **Q-S1.11.4.A — Framework choice.** Stick with what the
  current `cli.py` uses (argparse? click?) or upgrade to
  `typer`? Out-of-scope decision; record what's current and
  pick.
- **Q-S1.11.4.B — Subcommand naming.** `ein solve` vs `ein
  query --mode=solve`? The mode flag is what M1 currently uses
  internally; the subcommand form is more discoverable for
  CLI users. Discuss the surface UX.

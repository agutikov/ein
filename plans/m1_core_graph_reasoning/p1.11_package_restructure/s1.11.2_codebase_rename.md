# S1.11.2 — Codebase rename

**Phase:** P1.11 (package + CLI restructure).
**Estimate:** ½ day — pure search-and-replace + test runs.
**Status:** sequential after S1.11.1 (directory rename).
**Depends on:** [S1.11.1](s1.11.1_directory_rename.md) — the
directory has to be renamed first; this stage edits the imports
that the directory rename invalidated.
**Blocks:** [S1.11.3](s1.11.3_demo_merge.md) (demo merge —
imports rename must be clean first).

## Context

Every `from ein_bot.X import Y` and `import ein_bot.X.Y` in
the package source + tests + demos refers to the old name. The
directory rename in S1.11.1 makes these fail; this stage fixes
them in one sweep.

## Tasks

### Task T1.11.2.1 — Source imports

- **a.** `grep -rln "ein_bot" ein.py/src/ein/` (or new path);
  search-and-replace `ein_bot` → `ein` across all hits. Use
  per-file `sed -i` or an IDE refactor.
- **b.** Edit type hints that string-quote module paths (rare;
  TYPE_CHECKING usually doesn't need it).
- **c.** Edit docstrings that reference the package name.

### Task T1.11.2.2 — Test imports

- **a.** `grep -rln "ein_bot" ein.py/tests/`; same rewrite.
- **b.** Test fixture files that hardcode paths (e.g. CLI
  example outputs) — rewrite.

### Task T1.11.2.3 — Demo imports

- **a.** `ein.py/demo/bench_*.py` — rewrite imports.
- **b.** Confirm `python ein.py/demo/bench_solve.py` runs
  under the renamed package (S1.11.3 will then merge these
  into the package proper).

### Task T1.11.2.4 — Acceptance

- `grep -rln "ein_bot" ein.py/` returns no matches (except in
  comment-style historical references — if the user prefers
  to keep some, document them).
- `pytest ein.py/tests/` green.
- `python ein.py/demo/bench_solve.py examples/zebra2.ein`
  works.

## Open questions

- **Q-S1.11.2.A — Historical references.** Some doc files
  reference `ein_bot` as historical context (e.g. CHANGELOG
  entries, design docs). Are those kept verbatim or
  rewritten? Default: keep historical refs in commit
  messages + CHANGELOG, rewrite everything else.

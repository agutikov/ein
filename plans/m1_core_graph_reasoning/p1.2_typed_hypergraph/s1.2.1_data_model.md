## Context

The PoC kept two parallel dicts and called the whole thing `State`.
That gets us through binary `is`/`next_to` relations but cannot
natively express:

- `(allDifferent house_1.color house_2.color ...)` — n-ary;
- `(pos House_2 = pos House_1 + 1)` — relational with arithmetic;
- equality classes (e-graph machinery, deferred).

The data model needs to support **n-ary hyperedges**, **typed
nodes**, and reserve **equality-class IDs** so an e-graph promotion
later is not a rewrite.

## Tasks

### Task T1.2.1.1 — Core data structures

`src/ein_bot/graph/store.py`. Roughly:

```python
NodeId  = NewType('NodeId',  int)
EdgeId  = NewType('EdgeId',  int)
ClassId = NewType('ClassId', int)         # e-class id; equal to self.id until egraph fires
TypeId  = NewType('TypeId',  int)

@dataclass(slots=True)
class Node:
    id:    NodeId
    name:  str                            # interned atom name from the IR
    type:  TypeId
    cls:   ClassId                        # for e-graph; cls==id initially
    layer: Layer                          # ONTOLOGY | FACT | REASONING

@dataclass(slots=True)
class Edge:
    id:        EdgeId
    rel:       str                        # relation name (interned)
    args:      tuple[NodeId, ...]         # length ≥ 2; arity-2 is the common case
    layer:     Layer
    provenance: ProvenanceId | None       # see S1.2.3

class Graph:
    types:     dict[str, TypeId]
    nodes:     dict[NodeId, Node]
    edges:     dict[EdgeId, Edge]
    by_type:   defaultdict[TypeId, set[NodeId]]
    by_rel:    defaultdict[str,    set[EdgeId]]
    by_arg:    defaultdict[NodeId, set[EdgeId]]
    classes:   UnionFind[NodeId]          # deferred-use placeholder
```

`Edge` is the *hyperedge* primitive (arity ≥ 2). The arity-2 case is
a regular relation; arity-3+ is a hyper-relation
(`allDifferent(A, B, C)`, `between(A, B, C)`, …).

### Task T1.2.1.2 — Mutation API

`add_type(name, parent=None) -> TypeId`,
`add_node(name, type, layer) -> NodeId`,
`add_edge(rel, args, layer, provenance=None) -> EdgeId`,
`remove_edge(edge_id)` (for hypothesis rollback in P1.5).

Re-adding the same `(rel, args)` should be idempotent: return the
existing `EdgeId`. Mutation is *transactional* — `with g.txn():`
context manager batches additions, snapshots for rollback, and
commits atomically. Required for the hypothesis-test loop.

### Task T1.2.1.3 — Query API

```python
g.nodes(type=t)                        -> Iterator[Node]
g.edges(rel='lives-in', layer=FACT)    -> Iterator[Edge]
g.adjacent(node_id, rel=None)          -> Iterator[Edge]
g.match(pattern: Pattern)              -> Iterator[Bindings]
```

`match` is the pattern matcher; spec'd here, fleshed out in P1.3
(rule firing reuses it). Should be linear in matching edges for the
common arity-2 case (uses `by_rel` + `by_arg` indexes).

### Task T1.2.1.4 — IR ↔ Graph

`src/ein_bot/graph/from_ir.py` and `to_ir.py`.

- `from_ir(ir_forms) -> Graph` walks the parsed IR (P1.1) and
  populates the graph. `(ontology …)` forms create types +
  ontology-layer nodes/edges; `(facts …)` forms become
  fact-layer nodes/edges with `provenance=Provenance.from_source(loc)`.
- `to_ir(graph, layers=...) -> IRNode` is the inverse — emits a
  `(facts …)` block per layer. Used by the dump CLI and the trace
  renderer.

### Task T1.2.1.5 — Equality-class hooks (placeholder)

Implement `union_find` over `NodeId` but do *not* yet fire equality
propagation. Document the integration point: when an inference rule
asserts `(= ?a ?b)`, call `g.classes.union(a, b)`. Engine doesn't
yet act on the classes; that's reserved for an optional future
phase ([F4](../../../followups/f4_cross_cutting.md)).

## Context

Without provenance the *contradictions* task class
([idea 03](../../../docs/ideas/03-three-task-classes.md)) returns
"UNSAT" — useless. With provenance it can return a minimal
contradictory subset *and* the chain of derivations leading to
the conflict. Same machinery powers the trace generator
([P1.6](../p1.6_rendering_and_trace/README.md)): every "It follows
that …" sentence resolves to one provenance record.

Granularity is fixed at **per-edge**, per the working answer in
[M1 Q18](../../open_questions.md#q18--provenance-granularity). The
full derivation DAG falls out by transitive closure over premise
edges.

## Tasks

### Task T1.2.3.1 — Provenance record type

`src/ein_bot/graph/provenance.py`:

```python
@dataclass(frozen=True, slots=True)
class Provenance:
    id:         ProvenanceId
    kind:       Literal['source', 'rule', 'hypothesis', 'rejected']
    # 'source':     came from the user's input (Loc reference + sentence id)
    source:     Loc | None         = None
    sentence:   str | None         = None        # "condition (10)"
    # 'rule':       derived by a rule firing
    rule:       str | None         = None        # e.g. 'triangle-composition'
    premises:   tuple[EdgeId, ...] = ()           # the edges the rule consumed
    bindings:   dict[str, str]     = field(default_factory=dict)  # ?vars → atom names
    # 'hypothesis': speculative branch
    branch:     BranchId | None    = None
```

Each derived `Edge.provenance` points at one of these. Source-layer
facts get a `kind='source'` record on load (carries the loc + the
quoted sentence the IR's `:source` clause supplied).

### Task T1.2.3.2 — Derivation DAG queries

`Graph.derivation_dag(edge_id) -> DerivationDAG`:

- BFS from `edge_id` through `provenance.premises`.
- Terminates at edges whose provenance is `kind='source'`.
- Returns a `DerivationDAG` value: nodes are edges; arcs go from
  premise to conclusion. Hashable, comparable, printable in DOT
  (P1.6 will format it).

`Graph.unsat_core(conflicting_edges) -> set[EdgeId]`:

- Takes a set of mutually contradictory edges (typically returned
  by the constraint checker when verify() fires).
- Returns the minimal set of *source-layer* edges that, together,
  derive them. Computed as the union of source-layer terminals of
  each `derivation_dag()`.

### Task T1.2.3.3 — DerivationDAG render

A small helper returning DOT. Used by:

- `ein-bot trace explain <edge_id>` CLI;
- the markdown trace renderer in P1.6;
- the contradictions output in P1.5.

Render rules: rule-edges shown as boxes labelled with rule name +
truncated bindings; source-edges shown as ellipses with the source
sentence.

### Task T1.2.3.4 — Provenance round-trip in IR

Extend the IR `(rel …)` form: optional `:provenance` clause.
On dump, source-kind provenance prints `:source "condition (10)"`;
rule-kind provenance prints `:from-rule triangle-1 :using (#3 #7)`.
Lets a trace file be replayed: `parse(trace) → graph → equal under
provenance comparison`.

### Task T1.2.3.5 — Tests

`tests/graph/test_provenance.py`:

- Adding a derived edge with premises produces a recoverable DAG.
- Multi-hop derivation: A → B → C; `derivation_dag(C)` includes A.
- Cycles in derivations are detected (rule misuse). Test that the
  loader rejects them with a clear message.
- `unsat_core` returns the source-only frontier.

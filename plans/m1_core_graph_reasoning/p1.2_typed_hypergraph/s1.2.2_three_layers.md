# S1.2.2 — Three layers as views

**Phase:** P1.2 (typed hypergraph).
**Estimate:** 2-3 days.
**Depends on:** S1.2.1 data model.
**Implements idea:** [05 §The three implicit layers](../../../docs/ideas/05-zebra-puzzle-graph-reasoner.md#the-three-implicit-layers).

## Context

The PoC tangled three semantically distinct graphs together:

| layer | content | typical edge example |
|-------|---------|----------------------|
| Ontology | types, instances, value domains, a-priori inter-type relations | `(Color is-a Attribute)` |
| Fact     | relations the puzzle text actually asserted | `(lives-in Norwegian House_1)` |
| Reasoning | derived edges, hypothesis edges, rejected edges | `(derived (drinks Norwegian Water) :rule triangle-1)` |

Naming them explicitly buys: (a) the DOT renderer can pick *one*
layer to show without redrawing the world; (b) the trace planner
can say "of the 23 reasoning-layer edges, derive these 5 in this
order"; (c) hypothesis branching only ever copies the reasoning
layer.

## Tasks

### Task T1.2.2.1 — `Layer` enum + edge tagging

`src/ein_bot/graph/layer.py`:

```python
class Layer(IntEnum):
    ONTOLOGY  = 1
    FACT      = 2
    REASONING = 3
```

Each `Node` and `Edge` already carries a `layer` field from S1.2.1.
This task only formalises the enum and audits that ingestion (the
IR loader) sets it correctly per top-level form:

- `(ontology …)` → `ONTOLOGY`
- `(facts …)`    → `FACT`
- inference rule firings → `REASONING`

### Task T1.2.2.2 — Layer views

Add the three filtered accessors:

```python
class Graph:
    def ontology(self)  -> GraphView: ...
    def facts(self)     -> GraphView: ...
    def reasoning(self) -> GraphView: ...
    def all_layers(self) -> GraphView: ...
```

`GraphView` is a read-only filter over the parent `Graph`. Pattern
matching (`match()`) on a view restricts to the view's layer.

### Task T1.2.2.3 — Hypothesis branch = reasoning-layer copy

Add `g.fork() -> Graph` that:

- Shares the **ontology** and **fact** layers by reference (they're
  immutable post-load).
- Deep-copies the **reasoning** layer.
- Inherits provenance for shared edges; new edges get fresh ids.

This is the primitive [P1.5](../p1.5_hypothesis_loop/README.md)
builds on. Its existence in P1.2 (not P1.5) is so the data model is
forced to support it from day one.

### Task T1.2.2.4 — Layer-aware tests

`tests/graph/test_layers.py`. Verify:

- An ontology edge never appears in `g.facts()`.
- A fact edge never appears in `g.reasoning()`.
- `g.fork().reasoning()` mutations don't leak back to the parent.
- The `match()` pattern matcher honours layer filters.

## Acceptance

- All three views return the correct subset.
- `g.fork()` is O(|reasoning|), not O(|all|).
- `pytest tests/graph/test_layers.py` ≥ 12 tests, green.

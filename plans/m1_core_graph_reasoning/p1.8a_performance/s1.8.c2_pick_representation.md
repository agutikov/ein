# S1.8.C2 — Pick a representation

**Phase:** P1.8 (improvements), Theme C (negative-fact volume).
**Estimate:** 1 day for the decision; the implementation is C3.
**Status:** parked; sequential after S1.8.C1.
**Depends on:** [S1.8.C1](s1.8.c1_measure_negative_volume.md).
**Blocks:** [S1.8.C3](s1.8.c3_refactor_saturator.md).

## Context

The four options enumerated in [P1.8 README Theme C](README.md#theme-c--negative-fact-volume-reduction):

### Option 1 — Lazy materialisation

Derive `(not X)` *on demand* when the contradiction detector
asks rather than during saturation. Trade fact-volume for
detector-time; the detector becomes a pull-style matcher.

- **Win:** if most negatives are never consumed.
- **Loss:** detector path slows (per-query work).

### Option 2 — Layer-aware filtering

A `(not X)` derived from type-exclusivity is mechanically true
given the puzzle's type declarations — it doesn't need to live
in REASONING at all if no consumer reads it. Move to a virtual
layer.

- **Win:** if negatives are ontology-derivable and rarely
  consumed.
- **Loss:** layer management complexity.

### Option 3 — Goal-driven pruning

Combined with F7 §C (rule-set sufficiency), suppress
type-exclusivity firings whose conclusions are demonstrably
unconsumed by any subsequent rule *and* unconsumed by the
query.

- **Win:** big if the goal-set is small.
- **Loss:** needs static analysis of rule-set + query.

### Option 4 — Compressed representation

A *single* `(not (co-located ? ?_T))` fact with `?_T` denoting
"any pair distinct under T", expanded lazily. Needs compound-
node-kind support (Q26).

- **Win:** factor of N volume reduction for symmetric N×N
  cases.
- **Loss:** needs Q26 (compound node kinds) to land first.

## Tasks

### Task T1.8.C2.1 — Cross-reference C1 data with options

Based on the measurements:

- If high unused-fraction → lean Option 1 (lazy).
- If most unused are ontology-layer → lean Option 2 (virtual).
- If most consumed-only-by-detector → lean Option 1 anyway.
- If volume is symmetric / regular → lean Option 4
  (compressed), if Q26 has progressed.

### Task T1.8.C2.2 — Decision

Pick one; document rationale.

### Task T1.8.C2.3 — Refine the design

Write enough to make S1.8.C3 a direct implementation:

- For Option 1: the lazy contradiction detector API.
- For Option 2: the virtual-layer membership predicate.
- For Option 3: the static analysis + suppression hook.
- For Option 4: the compound-node-kind + expansion semantics.

## Acceptance

- Decision recorded.

## Open questions

- **Q-S1.8.C2.A — Multi-option.** Can two options compose
  (lazy + virtual)? Probably; the lazy path can produce only
  ontology-relevant negatives.
- **Q-S1.8.C2.B — Q26 blockage.** Option 4 depends on Q26;
  if Q26 hasn't moved, Option 4 is forced off the table.

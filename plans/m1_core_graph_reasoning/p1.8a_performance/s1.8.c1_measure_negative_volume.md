# S1.8.C1 — Measure negative-fact volume + consumption

**Phase:** P1.8 (improvements), Theme C (negative-fact volume).
**Estimate:** 1-2 days.
**Status:** parked — **measured 2026-06-14: NOT A LEVER.** zebra2's
saturated root is 62/360 = 17% negatives with a ~0s contradiction detector
(`profile_solve.py` / [S1.8.B-idx](s1.8.b_indexes.md) baseline) — not the
>50% that would make lazy materialisation pay. (Entry of Theme C; the
measurement its question asked for is now done.)
**Depends on:** P1.5 + P1.5a ship so the rule set is stable
and the saturation numbers reflect a working engine.
**Blocks:** [S1.8.C2](s1.8.c2_pick_representation.md) — the
representation choice depends on this stage's measurements.

## Context

P1.3 saturation produces lots of negative facts. Zebra.ein's
`(type-exclusivity co-located)` alone derives 120
`(not (co-located A B))` facts; over the full saturation the
REASONING layer holds ~120 not-facts vs ~25 positive
derivations. The volume is correct (each negative is a real
consequence of the rule), but much of it is *load-bearing
only via the contradiction detector* — the negatives are
scanned for matching positives, never queried directly.

This stage **measures** which negatives are actually consumed.
The choice of optimisation in S1.8.C2 (lazy materialisation /
layer-aware filter / goal-driven pruning / compressed
representation) depends on this measurement.

## Tasks

### Task T1.8.C1.1 — Instrumentation

- **a.** For each derived `(not X)` fact, track:
  - When it was derived (depth + sweep number).
  - Whether the contradiction detector ever consumed it.
  - Whether it ever matched a rule premise.
  - Whether it ever activated hypgen filtering.

### Task T1.8.C1.2 — Run on the corpus

- **a.** zebra2.ein (saturation only + full solve).
- **b.** All 11 branching demos.
- **c.** All saturation demos.

For each: total negatives derived; consumed count; unused
count.

### Task T1.8.C1.3 — Analysis

- **a.** Histogram: per puzzle, fraction of negatives
  consumed.
- **b.** Per-rule attribution: which rules generate the most
  unused negatives?
- **c.** Per-layer attribution: ontology-layer negatives vs
  reasoning-layer negatives.

### Task T1.8.C1.4 — Report

Write up findings in this stage file. Recommendation for
S1.8.C2 representation choice based on data:

- If unused-fraction is high (>50%) → lazy materialisation
  is the win.
- If unused-fraction is low but volume is high → compressed
  representation.
- If volume is high *and* most are consumed → maybe nothing to
  do; the negatives are doing their job.

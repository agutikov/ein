# S1.8.B3 — Saturator + contradiction-detector invariants

**Phase:** P1.8 (improvements), Theme B (perf) → COW.
**Estimate:** 1-2 days.
**Status:** parked — **NOT A LEVER** (`fork()` is `0.000s`; see the
[S1.8.B-idx](s1.8.b_indexes.md) baseline). Sequential after S1.8.B2.
**Depends on:** [S1.8.B2](s1.8.b2_engine_lookups.md).
**Blocks:** [S1.8.B4](s1.8.b4_perf_bench.md).

## Context

After overlay-aware lookups (S1.8.B2), the saturator and
contradiction detector should produce *identical* outputs on
a forked KB as they would on a shallow-copied KB. This stage
proves that invariant.

## Tasks

### Task T1.8.B3.1 — Bisection test

For each branching demo:

- **a.** Run saturator on a shallow-copy fork → record
  `Firing` list + `Contradiction` records.
- **b.** Run on COW fork → same.
- **c.** Diff. Must be identical.

### Task T1.8.B3.2 — Hypothesis-loop comparison

Same idea for `solve()` with hypothesis branching:

- **a.** Run zebra2 with shallow-copy fork → record
  `SearchTree`.
- **b.** Run with COW fork → record.
- **c.** Diff search-tree structure; must be identical.

### Task T1.8.B3.3 — Provenance integrity

`Provenance.justified_by` references must resolve correctly
through the fork chain. If a fact in the fork's overlay is
justified by a fact in the parent, the chain walk must
succeed.

### Task T1.8.B3.4 — Edge cases

- **a.** Fact added to fork → contradicts fact in parent → detector fires correctly.
- **b.** Fact derived in fork from premise in parent → provenance walks parent.
- **c.** Hypothesis fork → child fork → grandparent fact looked up.

## Acceptance

- Demos produce identical Firing + Contradiction outputs
  under both fork styles.
- zebra2 SearchTree identical under both.
- Provenance chains resolve correctly.

## Open questions

- **Q-S1.8.B3.A — Determinism.** Iteration order of
  `_facts_by_relation` might differ when overlay walks the
  chain (parent's order then overlay's order) vs flat
  iteration. Confirm the engine doesn't depend on iteration
  order; if it does, normalise.

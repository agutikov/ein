# S1.8.C3 — Refactor saturator + detector + trace renderer

**Phase:** P1.8 (improvements), Theme C (negative-fact volume).
**Estimate:** 3-5 days, depending on the chosen option.
**Status:** parked; terminal stage of Theme C.
**Depends on:** [S1.8.C2](s1.8.c2_pick_representation.md).
**Blocks:** nothing on M1's critical path.

## Context

Implement whichever Option (1-4) S1.8.C2 picked. Touches the
saturator (production side), contradiction detector (consumer
side), and trace renderer (presentation side — the new
representation has to render comprehensibly).

## Tasks

The task list below is **conditional on S1.8.C2's choice**.
Write the appropriate sub-tasks once the choice is made.

### Task T1.8.C3.1 — Refactor saturator

- **Option 1 (lazy)**: rules that would derive `(not X)`
  become no-ops; their conclusions become queryable predicates
  the detector calls.
- **Option 2 (virtual layer)**: type-exclusivity-derived
  negatives stay out of REASONING; the layer-aware lookup
  consults the virtual layer.
- **Option 3 (goal-driven)**: a static analyser pre-suppresses
  unconsumed activator firings.
- **Option 4 (compressed)**: derive compressed forms; expand
  in `_index_fact` or at lookup time.

### Task T1.8.C3.2 — Refactor detector

The contradiction detector's pull-style query path changes per
option. For Options 1 + 2: the detector consults the virtual /
lazy source. For Options 3 + 4: detector unchanged (the
representation change is upstream of it).

### Task T1.8.C3.3 — Refactor trace renderer

- **a.** The trace mentions every derived fact today. Under
  the new representation, what counts as "derived"?
- **b.** Update `(trace …)` IR + DOT rendering to show the
  compressed / virtual / lazy entries comprehensibly.
- **c.** Idea-08 acceptance: the trace still names every
  human-walkthrough firing.

### Task T1.8.C3.4 — Tests

- **a.** Saturation produces ≤ 50% of pre-refactor REASONING-
  layer fact count (target).
- **b.** Contradiction detector still catches every
  contradiction the pre-refactor engine did.
- **c.** Saturation time within 20% of pre-refactor (no
  regression).
- **d.** Trace fidelity (idea-08) preserved.

## Acceptance

- Target metrics met (per the P1.8 README Theme C acceptance):
  - REASONING-layer fact count drops by ≥ 50% with no loss of
    contradiction-detection power.
  - Saturation time within 20% of pre-optimisation.
- All existing tests pass.
- Trace renderer documents the new representation.

## Decision record (2026-06-04)

User-confirmed in conversation. Closes Q30 + T1.8.A1.1–.6 and supersedes the
tentative option-lists in [§ Tasks](#tasks). The first real consumer is the
[`std.macro`](../../../../ein.py/src/ein/stdlib/macro.ein) module shipped by
[S1.5.9](s1.5.9_ein_lang_macros.md) (forall/open), currently inlined per-file
until imports land.

**D1 — Granularity + surface (T1.8.A1.1): three tiers, qualified-by-default.**
Modules are **logical names** (not file paths), written as dotted atoms —
`std.macro`, `std.algebra`, `std.types`.
- `(import std.macro)` → fully-qualified access: `(std.macro.forall …)`.
- `(import std.macro :as m)` → aliased: `(m.forall …)`.
- `(import std.macro :symbols (forall open))` → flat, only listed names: `(forall …)`.

Mirrors Python's `import a.b` / `import a.b as x` / `from a.b import y`. The
default (no kw) is fully-qualified, so it is **collision-proof**; only the
flat `:symbols` tier can ever collide. `:as` + `:symbols` together → error (D3).

**D2 — Namespace + separator (T1.8.A1.2): flat table, `.` is the universal separator.**
One flat symbol table (`kb.relations`/`rules`/`hrules`/`macros`/`names`);
"scoping" is achieved by **name qualification**, not a separate scope object.
`.` becomes a **general atom character** (added to the `SYMBOL` char class) —
in module names *and* qualified refs alike; a qualified name like `m.forall`
is just an atom whose string has dots, and the parser does **no** module
resolution (the loader does, by name). Unified `.` over the Clojure `/`-split:
one separator, logical-name decoupling from file layout. Atoms already admit
`*`/`-`, so this is incremental, not a purity break. No ambiguity: `RANGE`
(`1..5`) is digit-anchored, atoms are letter-anchored; maximal munch makes
`std.macro.forall` one atom.

**D3 — Conflict / error matrix (T1.8.A1.3): strict, everything errors.**
- atom re-definition (same rule/relation/macro name, local or cross-file) → **error**.
- reserved-word re-definition (`absent`/`false`/`and`/… as a defined name) → **error**.
  *Gap to close:* today only the macro loader checks this (`_reserved_macro_names`);
  a shared `_reserved_names()` must also guard `_ingest_rules` + `_ingest_relation`
  (today `(rule absent …)` registers a silently-dead rule — the compiler reads
  `(absent …)` as the NAF primitive, never the rule).
- import not found / unreadable → **error**, carrying the `(import …)` `Loc`.
- import cycle (A→B→A) → **error**.
- `:as` + `:symbols` on one import → **error** (pick one tier).
- a `:symbols` symbol whose body cites a name that is neither imported, local,
  nor reserved → **error** (D7).
No warnings, no silent last-wins; `(override …)` deferred (no M1 puzzle shadows
a library rule).

**D4 — Resolution (T1.8.A1.4): logical name → file, search path, `.ein` implied.**
`std` is the reserved root alias for the **packaged** stdlib (`ein/stdlib/`);
`std.macro` → `<stdlib-root>/macro.ein`, `std.algebra.poset` →
`<stdlib-root>/algebra/poset.ein` (last segment = file, earlier = dirs).
Non-`std` names resolve **file-relative** to the importing file. The name never
carries `.ein` — the resolver appends it. The canonical stdlib moves into the
package with A3/A4; `ein.py/src/ein/stdlib/macro.ein` is the current home (and demo).

**D5 — Loading (T1.8.A1.5): eager, dedup/cache by resolved module name, cycle-checked.**
Eager (saturation needs every rule before it runs). Each module loaded once per
`solve()` (a diamond A→B,A→C,B→D,C→D loads D once). A `currently_loading` set
rejects cycles.

**D6 — Transitive re-qualification (re-export).** Imports resolve bottom-up; an
outer import re-qualifies a module's **entire resolved form-set** under its own
prefix — so `std.algebra`'s internally-imported `forall` re-exports as
`std.algebra.forall`. Transitive visibility = yes; `:private`/no-re-export
deferred.

**D7 — `:symbols` is not auto-closure.** Selective import pulls exactly the
listed names, unrenamed; a referenced-but-unlisted name is an undefined-name
error (D3). Authors list what they use. Whole-file/aliased tiers avoid this
(they pull everything).

**D8 — Architecture: flatten-then-load.** Resolve imports at the **form** level —
parse each module → recursively resolve its imports → emit one ordered form-list
with qualification applied (prefix-rename module-defined names + intra-module
references, **skipping the reserved set**: `primitives.STRUCTURAL` ∪
`predicates.names()` ∪ the declarators) → hand the union to the existing
single-pass `from_ir.load()` **once**. "Merge semantics" = list concatenation;
conflict detection = the loader's existing dup-guards. `(import …)` forms are
consumed by the resolver and never reach `load()` as facts.

**D9 — Dump modes.** *(realized 2026-06-04 — `kb.imports.resolve_and_minimize`
+ `ein ir parse --resolve`.)*
- *faithful* (default): `(import …)` forms round-trip verbatim.
- *resolved+minimized* (`--resolve` / `resolve_and_minimize`): imports spliced
  inline with qualification applied, then **tree-shaken** — a reachability pass
  from the puzzle's own forms (and any imported facts), closing over imported
  declarations' bodies, drops any imported *declaration* nothing references.
  Yields a standalone, minimal `.ein` (e.g. `--resolve examples/zebra2.ein`
  inlines `forall` and drops the unused `open`; identical 43 rules / 73 facts).
  Imported-fact shaking is a future refinement (M1 stdlib ships no facts).

**D10 — DOT rendering.** Route every node-id in `to_dot.py` / `dot_util.py`
through a single quote-always `quote_id()` helper so dotted/qualified atom names
render safely. One-time, forward-compatible.

**Module rename.** The macro library is **`std.macro`** (was `sugar`), file
`ein.py/src/ein/stdlib/macro.ein`. S1.5.9's inlined copies point at it; S1.8.A5
replaces each with `(import std.macro :symbols (forall open))`.

### Worked example (on paper — A3 implements)

`ein.py/src/ein/stdlib/macro.ein` — the `std.macro` module:
```lisp
(macro forall (?b ?G ?B) (absent (and ?G (absent ?B))))
(macro open   (?P)       (and (absent ?P) (absent (not ?P))))
```
A puzzle reaching `forall` three equivalent ways:
```lisp
(import std.macro)                        ; → (rule r () :match (std.macro.forall ?q … …) …)
(import std.macro :as m)                  ; → (rule r () :match (m.forall ?q … …) …)
(import std.macro :symbols (forall open)) ; → (rule r () :match (forall ?q … …) …)
```
Loader: resolve `std.macro` → `<stdlib-root>/macro.ein` → splice its 2 macros
(qualified `std.macro.*` / `m.*` / flat, per tier) ahead of the puzzle's forms →
`load()` once. A `--resolve` dump of the `:symbols` form, with a puzzle that
uses only `forall`, drops `open`.

## Tasks

### Task T1.8.A1.1 — File vs symbol-level imports

Two extreme designs:

- **(a) File-level**: `(import "stdlib/sugar.ein")` includes
  everything from that file. Simple, no name selection.
- **(b) Symbol-level**: `(import "stdlib" :symbols (symmetric
  transitive functional))` selectively pulls names. More
  control, more verbose.

Possible hybrid: `(import "stdlib/sugar.ein")` for whole-file +
`(use stdlib/sugar :symbols (...))` for partial.

Decide based on: M1 puzzle scale (small — file-level wins),
expected stdlib growth (large — symbol-level wins eventually).

### Task T1.8.A1.2 — Namespace scoping

- **(a) Flat namespace**: imports merge into one global
  symbol table. Conflicts → error or warning.
- **(b) Module-scoped**: imported symbols live under
  `module/symbol` (e.g. `stdlib/symmetric`). Verbose but
  clear.
- **(c) Aliasing**: `(import "stdlib" :as std)` then
  `std/symmetric`. Lisp-like.

### Task T1.8.A1.3 — Conflict policy

When a puzzle's local rule clashes with an imported rule of the
same name:

- **(a) Error** — refuse to load.
- **(b) Warn + local wins** — most permissive.
- **(c) Warn + imported wins** — counterintuitive; skip.
- **(d) Explicit `(override …)`** — require the puzzle to flag
  the intent.

### Task T1.8.A1.4 — Path resolution

- **(a) Repo-relative**: `(import "stdlib/sugar.ein")` resolves
  to a fixed path under the engine.
- **(b) File-relative**: relative to the importing file.
- **(c) Environment / install-relative**: discoverable like
  Python's `sys.path`.

For M1, (a) is enough. Future M2 NL-output may want (c).

### Task T1.8.A1.5 — Loading semantics

- **(a) Eager**: imports loaded immediately on parse.
- **(b) Lazy**: imports resolved when needed.
- **(c) Caching**: each file loaded once per `solve()` call.

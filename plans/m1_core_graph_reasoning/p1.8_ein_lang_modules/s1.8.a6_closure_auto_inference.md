## Context

The deferred S1.5.5 work. Two corrections from the 2026-05-22
review (recorded in [P1.8 README Theme A § Standard library](README.md#standard-library)):

- **Parameter-less.** `(rule infer-closure () :match … :assert
  (closed ?R))` — not `(?R)`. The latter would be a param
  list; the engine then expects activator facts and never
  fires. `?R` is a free match var, like `hypothesis-
  contradiction ()`.
- **`functional ⇒ closed` is too weak.** Use **`functional ∧
  total ⇒ closed`** as the sound operational witness.

Two senses of "closed":

- **Strong / semantic**: extension can't grow. Needs cardinality
  predicate the eq/neq-only matcher lacks.
- **Operational**: hypgen needn't branch on `R`. M1's `(closed
  R)` is operational; `functional ∧ total` is the sound
  witness post-S1.5.8.

## Tasks

### Task T1.8.A6.1 — Ship the rule

In the stdlib (per S1.8.A4 layout):

```lisp
(rule infer-closure ()
  :match (and (functional ?R) (total ?R))
  :assert (closed ?R)
  :why "{?R} is functional + total, so its extension is fixed.")
```

Parameter-less. Free `?R` match var.

### Task T1.8.A6.2 — Config flag

- **a.** Add `:enable-auto-closure` to `SolverConfig` (S1.5.4
  T1.5.4.4). Default **off** — explicit `(closed R)` still
  required unless the user opts in.
- **b.** Gate the rule's activation on the flag.

### Task T1.8.A6.3 — Tests

- **a.** Puzzle with `(functional R)` + `(total R)` + flag
  on: `(closed R)` derived; hypgen skips R.
- **b.** Same puzzle with flag off: `(closed R)` not derived;
  hypgen branches on R.
- **c.** Puzzle with `(functional R)` only: flag on or off,
  `(closed R)` not derived (totality required).

### Task T1.8.A6.4 — Document the soundness argument

The "two senses of closed" discussion lives in
[`docs/kernel/inference/`](../../../docs/kernel/inference/) (or
[S1.10.D](../p1.10_kernel_docs/s1.10.d_inference_engine_docs.md)
once it ships): why `functional ∧ total ⇒ closed` is sound for
the operational sense, and what would be needed for the
semantic sense (cardinality predicate).

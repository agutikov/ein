# S1.8.B4 — COW perf benchmark

**Phase:** P1.8 (improvements), Theme B (perf) → COW.
**Estimate:** ½-1 day.
**Status:** parked; terminal stage of COW sub-theme.
**Depends on:** [S1.8.B3](s1.8.b3_saturator_invariants.md).
**Blocks:** nothing; informs whether COW ships as the default
fork mode.

## Context

Confirm the COW overlay actually delivers the perf win. The
target from the parent README:

- **O(1) fork** — no per-fact copy.
- **Lookup overhead ≤ depth × constant** — each lookup walks
  the parent chain; depth grows with hypothesis branching
  depth.

## Tasks

### Task T1.8.B4.1 — Bench harness

A microbenchmark that:

- **a.** Builds a KB with N facts (N ∈ {64, 256, 1024}).
- **b.** Forks once + measures fork time.
- **c.** Forks N times nested + measures total fork time.
- **d.** For each fork depth, measures lookup time.

### Task T1.8.B4.2 — Whole-puzzle bench

- **a.** `bench_solve examples/zebra2.ein` wall time, shallow-
  copy fork vs COW.
- **b.** Branching demos: same comparison.
- **c.** Memory (RSS): COW should also reduce RSS at deep
  forks.

### Task T1.8.B4.3 — Profile + tune

If COW is slower than expected:

- Profile the lookup walk; identify hot spots.
- Maybe cache "last visited parent index" per fork.
- Re-bench.

### Task T1.8.B4.4 — Ship decision

Based on numbers:

- If COW wins clearly → flip default to COW.
- If COW marginally wins → keep shallow-copy default; COW
  opt-in.
- If COW loses (unexpected) → identify why; possibly
  re-architect.

Record decision in the stage file.

## Acceptance

- Bench numbers in the stage file.
- Fork time O(1) confirmed (or explanation if not).
- Lookup overhead measured; verify scaling.
- Ship decision documented.

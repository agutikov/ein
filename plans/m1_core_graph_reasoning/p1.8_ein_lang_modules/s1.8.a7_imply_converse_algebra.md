## Context

Per [P1.8 README Theme A § Standard library](README.md#standard-library):

**`converse` rule** — `(converse R1 R2)` ⟺ `R2 = R1°`:
`(R1 ?a ?b) ⇒ (R2 ?b ?a)`. The user's `imply2-reverse`.
Algebra lemmas:

- `symmetric R ⟺ converse R R`.
- `converse` is symmetric on the pair `(R1, R2)`.
- `right-of` / `left-of` are converse; `next-to` is
  self-converse.

**`imply` family** — one rule per arity (the matcher compiles
fixed-arity Scan/Join, no variadic slot):

- `imply1`: `(R1 ?a) ⇒ (R2 ?a)`.
- `imply2-fwd`: the shipped `implies` (zebra2's
  `(implies right-of next-to)`).
- `imply2-reverse`: = `converse`.

**`imply1` doubles as property→property implication**: a
property fact `(functional is-a)` is a 1-arg fact, so
`(imply functional closed)` *is* `imply1` — no new rule shape.

## Tasks

### Task T1.8.A7.1 — Ship `converse`

```lisp
(rule converse (?R1 ?R2)
  :match (?R1 ?a ?b)
  :assert (?R2 ?b ?a)
  :why "(converse {?R1} {?R2}): mirror {?R1}({?a},{?b}) → {?R2}({?b},{?a}).")
```

Param list `(?R1 ?R2)` — activator-driven; `(converse R1 R2)`
fact activates the rule on the R1×R2 pair.

### Task T1.8.A7.2 — Ship `imply1` + `imply2-fwd` + `imply2-reverse`

- **a.** `imply1`: 1-arg.
- **b.** `imply2-fwd`: 2-arg, forward.
- **c.** `imply2-reverse`: alias of `converse`, kept for
  ergonomic naming.

### Task T1.8.A7.3 — Algebra lemmas

These ship as additional rules / activator-derivers:

- `(symmetric ?R) ⟹ (converse ?R ?R)` and vice versa.
- `(converse ?R1 ?R2) ⟹ (converse ?R2 ?R1)` (symmetry of
  `converse`).

May require reflective rule-implication (S1.8.A9) — defer
implementation of these lemmas until A9 ships if so; ship the
documentation now.

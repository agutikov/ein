content moves.

## Context

After S1.8.A4 ships the library, M1's puzzles (zebra2.ein and
the branching demos) should migrate to use imports instead of
inlining the kernel rules. This stage's scope depends on the
S1.8.A4 decision:

**P1.7c note (2026-06-02).** Post-[S1.7c.4](../p1.7c_block_head_removal/s1.7c.4_migrate_and_drop_shim.md),
zebra2.ein is **already flat** — the kernel rules are top-level `(rule …)`
siblings, not nested in a `(rules …)` block. So "migrate" = delete the
promoted top-level `(rule …)` forms and add an `(import …)`; there is no
wrapper to unwrap.

- **If (a) or (c):** zebra2 → `(import "stdlib/kernel.ein")`
  for the kernel rules; keep `cross-attr-spatial-*` and other
  puzzle-specific rules inline.
- **If (b) inline-only:** this stage is empty.

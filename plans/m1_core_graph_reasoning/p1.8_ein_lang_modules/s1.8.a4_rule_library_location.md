# S1.8.A4 — Rule library location

**Phase:** P1.8 (improvements), Theme A.
**Estimate:** 1 day (decision + ship if chosen).
**Status:** parked; sequential after S1.8.A3 (imports work).
**Depends on:** [S1.8.A3](s1.8.a3_loader_resolve_imports.md).
**Blocks:** [S1.8.A5](s1.8.a5_migration_imports.md) (migration
target depends on this stage's choice).

## Context

[Q30](../open_questions.md#q30--universal-rule-library--import-mechanism)
asks: where do rule definitions live across files?

- **(a) Universal library** — one canonical `rules.ein` (or
  `stdlib/`) that puzzles import.
- **(b) Inline per puzzle** — current zebra.ein convention;
  no library.
- **(c) Hybrid** — universal library for the common rule set;
  puzzles can override / extend.

This stage picks one based on M1's accumulated knowledge of
the rule library + S1.8.A1-A3 import mechanism.

## Tasks

### Task T1.8.A4.1 — Decision

Recommendation criteria:

- Which rules are puzzle-agnostic? (`symmetric`, `transitive`,
  `functional`, `closed`, …) — these belong in a library.
- Which are puzzle-specific? (`type-exclusivity`,
  `cross-attr-spatial-fwd`, …) — these stay inline.

Likely answer: **(c) hybrid**. The universal library houses
the puzzle-agnostic kernel rules; puzzles author their
specific activator facts inline.

### Task T1.8.A4.2 — Ship the library (if a or c)

- **a.** Create `examples/stdlib/` (or wherever paths
  resolve per S1.8.A1.4).
- **b.** Ship the universal rules: `symmetric`, `transitive`,
  `functional`, `closed`, `bijective`, `converse`, `implies`,
  `imply1`, `imply2-fwd`, `imply2-reverse` (= `converse`), …
  Plus the closure rules from S1.8.A6 when shipped.
- **c.** One stdlib file per concern, or one big file? Pick
  per S1.8.A1.1 — file-level vs symbol-level decision
  influences this.

### Task T1.8.A4.3 — Document the library structure

A short `examples/stdlib/README.md` explaining each file +
when to import which.

## Acceptance

- Decision recorded.
- Library shipped (if a or c).
- `examples/stdlib/README.md` explains the layout.
- A puzzle that `(import …)`s the library and only declares
  its activator facts works.

## Open questions

- **Q-S1.8.A4.A — Library versioning.** Library evolves;
  puzzles may pin a version. M1 default: no versioning;
  revisit when M2 NL output starts hitting drift.
- **Q-S1.8.A4.B — Library testing.** Each stdlib rule should
  have at least one test puzzle. Bundle them under
  `tests/stdlib/`.

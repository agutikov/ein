## Tasks

### Task T1.20.J3.1 — The worked embedding script

A single runnable script (in `docs/api/ein.md` or a sibling
`docs/api/example_zebra.md`) that:

- loads [`examples/zebra2.ein`](../../../examples/zebra2.ein) via
  `KnowledgeBase.from_file(...)` (so stdlib `(import …)` resolution is
  exercised — note `from_ir(parse(text))` skips import resolution);
- runs `solve(kb, stop_after=1)`;
- branches on the verdict (`Solution` → print the answer grid via
  `trace.render_answer` / `render_solution_table`; `Contradiction` →
  print the unsat core; `Ambiguity` → report the gap);
- linearises + renders the trace (`linearize` → `render_markdown`).

Derive the call sequence from
[`utils/profile_solve.py`](../../../utils/profile_solve.py) and the
CLI [`cli/solve.py`](../../../ein.py/src/ein/cli/solve.py) (which
already does load → solve → `linearize` → render); keep it minimal and
**actually run it** against the repo before committing the output
block.

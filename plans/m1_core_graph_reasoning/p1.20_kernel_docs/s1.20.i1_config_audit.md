## Context

Theme I ships a measurement-backed page telling a puzzle author which
knobs are *load-bearing*. **Theme I is the one P1.20 theme that is not
doc-only** — its "add missing flags" step (README step 2) touches engine
code (see [I2](s1.20.i2_add_flags.md)). This stage does the design so the

### Live audit (confirmed at `8575bcc`)

**`SolverConfig` fields** ([`inference/config.py`](../../../ein.py/src/ein/inference/config.py))
— and how each behaves as a "feature off" case:

| field | default | "off" case | class |
|-------|---------|-----------|-------|
| `enable_pre_branch_lookahead` | `True` | flip `false` | **perf lever** (one-step `_dies_immediately` prune; the main lever) |
| `enable_lookahead_kill_cache` | `True` | flip `false` | **perf lever** (re-runs lookahead each time when off) |
| `hypgen_scoring` | `"popularity"` | swap `"most-constrained"` | **value-choice** (not on/off) |
| `hypgen_rel_weight` / `hypgen_obj_weight` | `1.0` | vary | value-choice (popularity coefficients) |
| `lattice_order` | `"lex"` | swap `"score-sum"` (needs `hypgen_scoring="popularity"`) | **value-choice** |
| `candidate_order_seed` | `-1` | set ≥ 0 | determinism-probe (not a perf lever) |
| `lattice_order_seed` | `None` | set int | determinism-probe |
| `lattice_sanity_check` | `False` | (default off; turning *on* slows) | release-regression check |
| `print_alive` / `warn_derived_naf` | `False` | — | diagnostic only |

**Correction to the README's step 1** ("flip every True-default flag"):
only **two** fields are True-default booleans (`enable_pre_branch_lookahead`,
`enable_lookahead_kill_cache`). The rest are value-choices, ints, or
default-`False` — they need value-swap rows, not a naive negation.

**Always-on engine features NOT behind any flag** (the real subject of
README step 2 — grepped: their sites have no `config.`/`enable_` guard):

| feature | site | gateable? |
|---------|------|-----------|
| CDCL path-condition no-goods | [`inference/nogoods.py`](../../../ein.py/src/ein/inference/nogoods.py) + `apriori.filter_candidate` | yes — emit/filter behind a flag |

These are hardcoded today, so their impact **cannot be measured** without
either (a) adding a gating flag (I2, code) or (b) a hand-hacked engine
build. Resolving which is **Q-S1.20.I.A**.

**Bench surface** (so I3 has fixed instruments): the runner is
[`ein_pypy.sh solve <file>`](../../../ein_pypy.sh) with `-s/--stats`,

## Execution — findings & decisions (2026-06-17)

### Decision 1 — lever classification (locked; class is a *hypothesis* I3 confirms)

| lever | flag / value (default) | class (hypothesis) | "off" / swap case | new flag? | gate site |
|-------|------------------------|--------------------|-------------------|-----------|-----------|
| pre-branch lookahead | `enable_pre_branch_lookahead` = `True` | **perf** (may exceed budget → effectively broken) | `false` | exists | `config.py` |
| lookahead kill-cache | `enable_lookahead_kill_cache` = `True` | **perf** | `false` | exists | `config.py` |
| CDCL path no-goods | `enable_path_nogoods` = `True` | **perf** | `false` | **new** | `nogoods.emit_nogood` + `apriori.filter_candidate` |
| `__symmetric__` native mirror | `enable_symmetric_mirror` = `True` | **correctness-or-perf** (see Q-S1.20.I2.C) | `false` | **new** | `saturator._next_mirror_firing` / `_enqueue_mirror_sources` / `_has_pending_mirror` |
| singleton-death writeback | `enable_singleton_writeback` = `True` | **perf** | `false` | **new** | `monotonic/_helpers.py` |
| forced-positive promotion | `enable_forced_positive` = `True` | **perf** (possibly correctness) | `false` | **new** | `monotonic/_helpers.py` |
| hypgen scoring | `hypgen_scoring` = `"popularity"` | **perf (value)** — same verdict | swap `"most-constrained"` | exists | `hypgen.py` |
| lattice order | `lattice_order` = `"lex"` | **perf (value)** | swap `"score-sum"` (+ `hypgen_scoring="popularity"`) | exists | `monotonic/` |

Determinism/diagnostic knobs (`candidate_order_seed`, `lattice_order_seed`,
`lattice_sanity_check`, `print_alive`, `warn_derived_naf`) are **out of the
matrix** — they don't disable a feature whose perf/correctness impact a
puzzle author needs (sanity-check only *adds* cost; the seeds probe
determinism; the rest are diagnostics).

### Decision 2 — gating (resolves Q-S1.20.I.A): **flag all four** un-gated features

`enable_path_nogoods`, `enable_symmetric_mirror`, `enable_singleton_writeback`,
`enable_forced_positive` — each `bool = True` (behaviour-preserving at
default). Rationale + feasibility:

- Each is **independently measurable** only behind a flag; demonstrating
  their impact is the matrix's whole purpose.
- **Gateability confirmed**: the no-good logic is two call-sites
  (`emit_nogood` + `filter_candidate`); the three monotonic augmentations
  are centralised in `monotonic/_helpers.py` (single-site each); the
  symmetric mirror is three cohesive `saturator` methods. Low-risk.
- A "breaks when off" result (plausibly the symmetric mirror) is **not a
  failure** — it's a valid *correctness-load-bearing* row, the strongest
  "you need this on" signal.

This is **I2 = engine code** — the one P1.20 code change. It is gated by
this decision but still needs the user's explicit go before I2 runs.

### Decision 3 — matrix spec (locked; handed to I3 + I4)

- **Table A — correctness-load-bearing**: lever | what breaks when off
  (verdict change / non-termination within the outer guard → `∞`).
- **Table B — perf-only**: lever | default solve (s, median of N) | off
  solve (s) | **slowdown ×** | Δ`enterings_total` | Δ`layers_explored` |
  note.
- **Value-choice sub-section**: `hypgen_scoring` (popularity vs
  most-constrained), `lattice_order` (lex vs score-sum) — each default-vs-swap.
- **Per-cell raw metrics** (from `MonotonicStats`): `enterings_total` /
  `_alive` / `_dead_pre` / `_dead_post`, `layers_explored`,
  `forced_positives`, `saturate_count`, `solution_nodes`, `exhausted` +
  median wall-time + RSS.
- **Mode**: **exhaustive** primary (a disabled prune must show the full
  blow-up, not a lucky early stop); optional `stop_after=1` fast-path
  column. **Budget**: ~120 s/cell perf guard; ~600 s outer "broken-if-off"
  sentinel. **PyPy** for wall-clock.

The deliverable doc (`docs/kernel/inference/features.md`) is **not created
here** — it stays an I4 artifact so it ships with real numbers, not a stub.

## Open questions

- **Q-S1.20.I.A — Gate the always-on features, or document them?**
  **RESOLVED (Decision 2): flag all four** (`enable_path_nogoods`,
  `enable_symmetric_mirror`, `enable_singleton_writeback`,
  `enable_forced_positive`), all default `True`. This is the one P1.20
  **engine-code** change ([I2](s1.20.i2_add_flags.md)) — behaviour-
  preserving at default, but still awaits the user's explicit go.
- **Q-S1.20.I.B — Value-choice levers.** **RESOLVED:** measured as
  value-swaps in dedicated sub-rows, not as "off" (Decision 3).
- **Q-S1.20.I2.C — symmetric-mirror "off" semantics** *(surfaced by I1, for
  I2 to decide).* When `enable_symmetric_mirror=false`, does symmetry (a)
  **pure-skip** — not computed at all (→ the marked relation isn't closed,
  so zebra2's `next-to` symmetry breaks → a **correctness-load-bearing /
  broken-if-off** row), or (b) **fall back** to the stdlib `symmetric` rule
  if present (→ a clean **perf** comparison: native mirror vs matcher)?
  The mirror docstring frames itself as a fast-path *over* the rule, so (b)
  is measurable *iff* the rule is compiled when the marker is present.
  **Recommend (a) pure-skip** — simpler, and "breaks when off" is the
  honest "this is required" signal; revisit if a native-vs-rule perf
  delta is specifically wanted. **RESOLVED empirically (I3, 2026-06-17):**
  implemented as pure-skip, but zebra2 imports the stdlib `symmetric` rule,
  which closes symmetry when the native mirror is off — so off is **correct
  and costs nothing measurable** on zebra2 (perf-only, *not* broken-if-off).
  The mirror is a true fast-path over the rule; its benefit shows only where
  the rule's matcher cost dominates.

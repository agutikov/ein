# S1.20.G — `docs/index/` → `docs/lib/` rename

**Phase:** P1.20 (kernel documentation), Theme G.
**Estimate:** ½-1 day, mostly cross-link updates.
**Status:** parked; "should ship *before* M2b paper" per the
parent README.
**Depends on:** nothing structural; can run any time.
**Blocks:** [M2b paper](../../m2b_presentation/README.md) —
"avoid permanent 'see docs/index in the published paper'
footgun".

## Context

The current `docs/index/` is *not* an index — it's the
external-tech library / catalogue (12 thematic files + a
knowledge graph). The name "index" reads as "table of
contents" which mismatches the content. `lib/` (library, in
the sense of "library of references the project draws from")
matches the intent.

Mechanics: pure rename + cross-link sweep.

## Tasks

### Task T1.20.G.1 — Directory rename

- `git mv docs/index docs/lib`.

### Task T1.20.G.2 — Cross-link sweep

- **a.** `grep -rln "docs/index" .` — tabulate hits across
  `docs/`, `plans/`, top-level READMEs, `CLAUDE.md`, code.
- **b.** Per-hit rewrite to `docs/lib`.
- **c.** Knowledge-graph DOT file path references — the SVGs
  + the Cytoscape `index.html` may reference the old name in
  their headers/captions. Update.

### Task T1.20.G.3 — Regenerate the knowledge-graph artefacts

Per `CLAUDE.md` instructions:

```sh
utils/render_knowledge_graph.sh svg all     # 4 SVGs
python3 utils/render_knowledge_graph_cy.py  # elements.js + style.js + index.html
```

Output: SVGs + Cytoscape HTML living under `docs/lib/`.

### Task T1.20.G.4 — Stale path check

- **a.** `grep -rln "docs/index" .` — returns only intentional
  historical refs (commit messages).
- **b.** Confirm the knowledge graph still renders + the
  Cytoscape page still loads.

## Acceptance

- `docs/index/` no longer exists; `docs/lib/` does.
- All cross-references resolve.
- Knowledge graph SVGs + Cytoscape HTML regenerated under the
  new path.
- No reader has to figure out that "docs/lib was formerly
  docs/index".

## Open questions

- **Q-S1.20.G.A — Backward-compat redirect.** Do we want a
  `docs/index/` symlink for the next few months while
  bookmarks update? Probably no — this is a docs-internal
  rename, no external bookmarks expected.

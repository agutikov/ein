## Context

The Ein Zebra Guide is a **learn-Ein-by-example tutorial** — "how to solve
a Zebra puzzle with Ein", from a complete beginner. It is **not** a rule
reference and **does not** explain kernel internals or the inference
process; it shows Ein **from the user's perspective** and links out to the
reference docs ([`ir/01-ein-graph/`](../../../docs/kernel/ir/01-ein-graph/),
[`ir/03-ein-lang/`](../../../docs/kernel/ir/03-ein-lang/),
[`docs/api/`](../../../docs/api/)) and the inference explanation
([`zebra_walkthrough.md`](../../../docs/kernel/inference/zebra_walkthrough.md),
[`architecture_and_algorithms.md`](../../../docs/kernel/inference/architecture_and_algorithms.md)).
It is **complemented by** the walkthrough: the guide teaches incrementally,
the walkthrough is the full solved trace it hands off to (K4).

> **Re-base from the 2026-05-27 K.** The old K opened directly at rules
> (a family-grouped rule catalogue with graph pairs). This re-based K adds
> the from-zero front the catalogue lacked — and that catalogue becomes the
> guide's *later* chapters ([K2](s1.20.k2_first_rules.md)/[K3](s1.20.k3_rule_families.md)),
> not the whole guide.

K1 establishes the guide's spine and teaches the three non-rule
fundamentals, each in the **tri-representation** the guide leans on
throughout: **NL ↔ ein-lang IR ↔ graph**.

### Task T1.20.K1.2 — Chapter: objects & relations (`docs/guide/01_objects_and_relations.md`)

Teach, each in NL ↔ IR ↔ graph, on a **tiny 2–3-house** hand-built
example (not the full puzzle):

- **Objects** — named things (`House-1`, `Norwegian`, `Tea`). NL: "there
  is a house"; IR: an atom; graph: an object node (Levi node with no
  out-arrows). Link the precise model:
  [`01-ein-graph/03_ein_model.md`](../../../docs/kernel/ir/01-ein-graph/03_ein_model.md)
  (atoms vs objects) + the [glossary](../../../docs/kernel/glossary.md).
- **Relations** — how objects connect (`nation-loc`, `next-to`). NL: "the
  Norwegian lives in house 1"; IR: `(relation nation-loc Nation House)`;
  graph: a relation node with outbound arrows.
- **Facts** — a relation applied to objects: `(nation-loc Norwegian House-1)`.
  NL ↔ IR ↔ graph (a Levi hyperedge node). Introduce the three
  [layers](../../../docs/kernel/ir/02-data-model/02_store.md) lightly
  (ontology / fact) — enough to author, not the data-model internals.

### Task T1.20.K1.3 — The tri-representation device

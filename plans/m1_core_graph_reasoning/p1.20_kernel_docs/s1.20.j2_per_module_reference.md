## Context

S1.20.J1 ships the *flow*; this stage ships the *exhaustive
per-symbol reference*. One page per public subpackage, each symbol
documented with **name · signature · params · returns · semantics ·
1-2 line example**, plus an explicit **internal-only** demarcation so
readers don't reach for engine internals as if they were the
contract.

The exact symbols + file:line anchors are in
[S1.20.J1's audit table](s1.20.j1_embedding_contract.md#re-base-audit-2026-06-16--the-live-surface-vs-the-2026-05-27-sketch);
this stage turns that inventory into reference prose. Author against
the source, not memory — the engine surface drifts (Q-S1.20.J.D).

## Tasks

### Task T1.20.J2.1 — `docs/api/ir.md` (parse / dump / AST)

- `parse(text, *, filename=None) -> tuple[SForm, ...]`, `IRParseError`,
  `parse_tree` (escape hatch).
- `dump` / `dump_canonical` / `dump_compact` — the round-trip surface.
- AST node types (`Atom`, `Var`, `Keyword`, `Wildcard`, `String`,
  `Int`, `Range`, `KwPair`, `SForm`, `Loc`, `IRNode`) — a compact
  table, since most embedders treat these as opaque `SForm`s passed
  straight to `load`.
- Source: [`ir/__init__.py`](../../../ein.py/src/ein/ir/__init__.py),
  [`ir/parser.py`](../../../ein.py/src/ein/ir/parser.py).

### Task T1.20.J2.2 — `docs/api/kb.md` (KnowledgeBase + entities)

- Construction: `KnowledgeBase.from_ir(forms, *, base_dir=None)`,
  `.from_file(path)`, and `ein.kb.load(...)`; `KBLoadError`.
- Read surface a downstream user touches: `relations`, `rules`,
  `hrules`, `facts`, `query`, `config`, `classes`, `names`.
- Entity dataclasses: `Fact` (`relation_name`, `args`, `layer`,
  `provenance`), `Relation`, `Rule`, `Layer`
  (ONTOLOGY/FACT/REASONING), `Pattern`, `Query`, `FactView`,
  `EqClasses`.
- Provenance: `Provenance` (+ `from_source`/`from_rule`/`from_hypothesis`)
  and `DerivationDAG` — how to walk a fact's origin.
- Flag mutators (`add_fact`, `add_and_index_fact`, `rebuild_indexes`,
  `fork`) as **loader/engine-internal**, not embedding surface.
- Source: [`kb/__init__.py`](../../../ein.py/src/ein/kb/__init__.py),
  [`kb/store.py`](../../../ein.py/src/ein/kb/store.py),
  [`kb/provenance.py`](../../../ein.py/src/ein/kb/provenance.py).

### Task T1.20.J2.3 — `docs/api/inference.md` (saturate / solve / verdict / config)

- `Saturator(kb, engine=None)`, `.saturate(*, max_steps=None) ->
  Iterator[Firing]`, `.step`, `.is_stalled`, `.contradictions`.
- `Firing(rule, activator, bindings, derived, premises, redundant)`.
- `solve(root_kb, *, stop_after=None, max_set_size=5, config=None,
  dumper=None, max_time=None, max_enterings=None, store_lattice=False,
  on_budget="raise") -> tuple[Verdict | Aborted, MonotonicStats]` —
  document each kwarg, and that the verdict *type* is read from the
  solution count (no `mode=`).
- Verdict union + `Aborted`; goal helpers `Mode`, `is_solved`,
  `goal_bindings`, `query_value`.
- **`SolverConfig` field reference** — one row per knob
  (`enable_pre_branch_lookahead`, `enable_lookahead_kill_cache`,
  `hypgen_scoring` + `hypgen_rel_weight`/`hypgen_obj_weight`,
  `candidate_order_seed`, `lattice_order`/`lattice_order_seed`,
  `lattice_sanity_check`, `print_alive`, `warn_derived_naf`) with
  default + effect. Cross-link **Theme I**'s
  [feature×config matrix](README.md#theme-i--kernel-feature--config-matrix)
  for the *measured* impact (this page is the *definitional* side;
  avoid duplicating the benchmark numbers).
- Source:
  [`inference/saturator.py`](../../../ein.py/src/ein/inference/saturator.py),
  [`inference/monotonic/solver.py`](../../../ein.py/src/ein/inference/monotonic/solver.py),
  [`inference/verdict.py`](../../../ein.py/src/ein/inference/verdict.py),
  [`inference/config.py`](../../../ein.py/src/ein/inference/config.py),
  [`inference/firing.py`](../../../ein.py/src/ein/inference/firing.py).

- Four pages (`ir.md`, `kb.md`, `inference.md`, `trace.md`), each

- Every signature matches the source at the recorded SHA.

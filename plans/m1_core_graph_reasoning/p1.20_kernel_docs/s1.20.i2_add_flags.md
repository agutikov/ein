## Context

> **Not doc-only.** Unlike every other P1.20 stage, this one edits the
> engine. The guardrail: each new flag **defaults to current behaviour**,
> so the shipped solve is byte-identical — the flag only enables the
> *negation* (the "feature off" case) the matrix measures. This keeps the
> change behaviour-preserving (P1.20's "no kernel-semantics change" line
> holds at the default), while making the always-on features measurable.

I1's audit found these features hardcoded (no config guard today):

| feature | flag to add (default = on) | site to gate |
|---------|----------------------------|--------------|
| CDCL path-condition no-goods | `enable_path_nogoods` | [`nogoods.py`](../../../ein.py/src/ein/inference/nogoods.py) emit + `apriori.filter_candidate` |
| `__symmetric__` native mirror | `enable_symmetric_mirror` | [`saturator.py`](../../../ein.py/src/ein/inference/saturator.py) (`_sym_*` / `_mirror_*`) |
| singleton-death writeback | `enable_singleton_writeback` | `monotonic/` augmentation |
| forced-positive promotion | `enable_forced_positive` | `monotonic/` augmentation (`stats.forced_positives`) |

(Final flag names + which subset to add come from I1.)

(`./run_tests.sh`) must stay green — this is the proof the change is
behaviour-preserving.

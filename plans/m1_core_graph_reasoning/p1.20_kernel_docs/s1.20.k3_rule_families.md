## Context

This is the re-based home of the **old K's family catalogue** — but as
*graded teaching chapters*, not a flat reference. The reader graduates from
K2's one-premise rules to the machinery that actually cracks `zebra2`.

A key framing the guide should make explicit (audited from
[`zebra2.ein`](../../../examples/zebra2.ein) at `645b896`): zebra2 is
**two layers of rules** —

- **Reusable stdlib property machinery** (imported, not puzzle-specific):
  `domain-elimination` (`std.elim`), `functional`/`injective` negative
  completion (`std.algebra` / `std.bijection`), `symmetric`/`transitive`
  (`std.algebra`, met in K2). Teach these as "the library does the generic
  logic."
- **zebra2's own spatial rules** (the `(rule …)` blocks in the file):
  `co-located` / `co-located-negative` / `co-located-fanout`,
  `disjunctive-prune-fwd` / `-bwd`, `adjacent-via-fwd` / `-bwd` /
  `-endpoint-fwd` / `-endpoint-bwd` / `-negative`, `derive-adjacent-via*`.
  Teach these as "how this puzzle encodes 'next to' / 'right of'."

## Tasks

### Task T1.20.K3.1 — Chapter `docs/guide/03_rule_families.md` (or a folder — see Q-K1.A)

One section per family, each rule in the **ein-form + NL + graph
before/after** unit from K2, on a **real `zebra2` fragment**:

- **domain-elimination** — "X must be Y because not A, not B, not C, not D"
  (the bijective sole-survivor). The guide's centrepiece — it's the
  measured load-bearing pattern (cross-link
  [`features.md`](../../../docs/kernel/inference/features.md)).
- **disjunctive-prune (fwd/bwd)** — eliminate placements a spatial clue
  forbids.
- **spatial / adjacent-via (+ endpoints)** — `next-to` / `right-of`
  reasoning, including the row-end boundary cases (`-endpoint-*`).
- **negative-completion** — deriving `(not …)` facts from
  `functional`/`injective` + `co-located-negative`; why negative facts
  matter (they feed domain-elimination).

## Acceptance

- Every zebra2 rule family is taught with ein form + NL + compact & Levi

  it does and why — without reading engine internals.

### T1.7b.3.3 вЂ” `_coerce` dispatch table + the `int | None` fix (F-KER-4) вЂ” **bug**

`config.py:205-258`: because `config.py:25` has `from __future__ import
annotations`, `field.type` is always a `str`, so the four `field_type is
bool/int/float/str` checks (`:218,227,240,248`) are **dead**, and any
`Optional` flag (`lattice_order_seed: int | None`) stringifies to
`"int | None"`, matches no arm, and **raises** at load (`:255`). Rewrite:

```python
_COERCERS = {"bool": _coerce_bool, "int": _coerce_int,
             "float": _coerce_float, "str": _coerce_str}
base = field.type.split("|")[0].strip()        # "int | None" -> "int"
fn = _COERCERS.get(base)
if fn is None: raise ValueError(... unsupported type ...)
return fn(field, value)
```

**New regression test:** a `.ein` with `(config :lattice-order-seed 7)` loads
and yields `SolverConfig(lattice_order_seed=7)` (fails before, passes after).

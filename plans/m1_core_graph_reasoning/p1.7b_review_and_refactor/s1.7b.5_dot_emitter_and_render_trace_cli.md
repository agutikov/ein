# S1.7b.5 — Shared DOT emitter + render/trace/cli decomposition

**Phase:** P1.7b (Review & refactor).
**Estimate:** 2–3 days. Independent of S1.7b.2/3/4.
**Depends on:** nothing structural.
**Findings:** [F-RTC-1..10](findings.md#rtc--render--trace--cli--ir),
[F-KB-8/10](findings.md#kb--data-model-kbpy).
**Constraint reminder:** Python tools emit **DOT only** — the DOT→SVG raster
stays a shell-script job. This stage unifies *how DOT strings are built*, not
what's rendered. Snapshot every emitter's output before touching it: the
generated DOT must be **byte-identical** after (the tests already pin much of
it; add golden captures for the rest).

## Goal

Kill the DOT-helper duplication (the single highest-leverage RTC change),
flatten the two deepest-nested parsers, unify the two trace pipelines' parsing,
and split the 154-line CLI parser. All output-preserving.

## Tasks

### T1.7b.5.1 — Promote `render/dot_util.py` to the one DOT-emitter API (F-RTC-1)

`dot_util.py` exists but only `quote`/`value_label`/`fact_label`/shapes moved
there; line construction is re-rolled in six files. Quantified duplication:
escape body 5× byte-identical (`kb/render._q:66` ≡ `dot_util.quote:37`;
`slice._esc:71` ≡ `lattice_dag._esc:168`; inlined in `provenance.py:197`,
`ir/to_dot`), `_multiline` 2×, md5 node-id 3× (`render._fact_node_id:71`,
`slice._node_id:67`, `lattice_dag._cell_id:165`, + `provenance._fact_dot_id:173`
— F-KB-8), the `rankdir=LR; node[fontname="Inter"]` preamble ~6×, and **two
emit idioms** (`ir/to_dot._Builder` class vs everyone's `lines:list[str]`).

Add the minimal real API and route all six renderers through it:

```python
# dot_util.py
def esc(s) -> str
def node_id(prefix, key) -> str            # one md5[:N]
def node(nid, **attrs) -> str              # quotes + formats attrs
def edge(src, dst, **attrs) -> str
def cluster(cid, label, body, **attrs) -> list[str]
def multiline(*parts) -> str
def digraph(name, body, *, rankdir="LR", font="Inter") -> str   # preamble + join
```

Delete `kb/render._q`, both `_esc`, both `_multiline`, all three id-hashers,
the duplicated preambles. Make `ir/to_dot._Builder` a thin wrapper over `node`/
`edge` (or drop it for the functional API the other four use). Pick **one**
attr style (`attrs:list[str]` + join, the majority) — resolves F-RTC-2's
f-string-vs-join inconsistency for free.

### T1.7b.5.4 — Unify the two trace pipelines' parsing (F-RTC-3)

`ir/to_dot.render_trace` (`:308-405`) hand-scans the same `(trace …)` SForm
that `parse_trace_steps` already turns into `TraceStep`s. Both consumers stay
(the `ir dot --trace-view` path and the markdown solve path), but

render `TraceStep`s — deleting `_trace_premises` and both inline KwPair loops,
and guaranteeing the DOT trace view and markdown trace agree on parsing.

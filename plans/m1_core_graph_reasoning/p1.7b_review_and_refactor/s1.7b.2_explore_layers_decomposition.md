# S1.7b.2 — `_explore_layers` decomposition + `Mode` retirement

**Phase:** P1.7b (Review & refactor). **The flagship.**
**Estimate:** 3–5 days (the one stage that needs real care).
**Depends on:** [S1.7b.1](s1.7b.1_dead_code_sweep.md) (removes the dead `Mode`
adjacency it untangles cleaner on a smaller surface). Independent of 3/4/5.
**Findings:** [F-ENG-1/4/5/6/7/8/13/14](findings.md#eng--monotonic-search-engine-inferencemonotonic),
[F-KER-1](findings.md#ker--inference-kernel-inferencepy-excl-monotonic).
**Risk-of-record:** this is the same code P1.7a found *two* soundness traps in
("a correct engine never exhausts a SAT problem to ⊥, never calls a non-model a
model"). The decomposition must be **behaviour-exact** — see the harness below.

## The problem

`_explore_layers` (`solver.py:602-1222`) is **621 lines / 91 branches / nesting
depth 7-9** and fuses the `solve`, `gaps`, and `contradictions` flows through
**~24 `entry ==` discriminators**. It is a category error: one function is
three state machines wearing a trench coat. Symptoms the review pinned:

- `mode = Mode.CONTRADICTIONS if entry=="solve"` (`:675`) — a *deliberately
  wrong* mode passed so `is_solved` returns False on the solve path (F-ENG-4 /
  F-KER-1). Two axes (which-entry × goal-semantics) collapsed into one knob.
- `phase_2_done = True; break` 7× (F-ENG-6) — boolean-as-goto re-breaking the
  layer loop, giving the body ~12 exit points.
- duplicated entry post-amble (F-ENG-5) and unsat-core synthesis ×4 (F-ENG-7).
- `_LatticeLoopState` carries fields no single run uses (F-ENG-8).
- two `# type: ignore[arg-type]` (F-ENG-13) and `assert`-as-control-flow
  (F-ENG-14) papering over the fused typing.

## The target shape — `EntryPolicy` strategy over a thin shared loop

One loop that branches on **nothing**; three small policy objects supply the
per-entry decisions. The reframe is exactly P1.7a's: *the search is invariant;
only the reading of its events differs.*

```python
class EntryPolicy(Protocol):
    accumulator: _Accumulator                  # per-entry (F-ENG-8)
    def solved(self, kb) -> bool: ...           # is_solution_node | is_solved(.,SOLVE)
    def on_root(self, ctx) -> Verdict | None: ...
    def on_dead(self, ctx, c, result) -> _Step: ...
    def on_solution(self, ctx, c, result) -> _Step: ...
    def on_alive(self, ctx, c, result) -> _Step: ...
    def finalise(self, ctx) -> tuple[Verdict, MonotonicStats]: ...

# _explore_layers becomes:
def _explore_layers(root_kb, *, policy: EntryPolicy, max_set_size=5, ...):
    ctx = _setup(root_kb, cfg, dumper, stop_after)          # F-ENG-1 phase 0
    if (v := _phase1_root(ctx, policy)) is not None:         # phase 1
        return v
    _phase2_layers(ctx, policy)                              # phase 2 (no entry==)
    return policy.finalise(ctx)                              # phase 3
```

`solve`/`gaps_solve`/`contradictions_solve` each construct their policy and
call `_explore_layers`. **No `entry` string, no `mode`, no `phase_2_done`** —
the per-candidate handler returns `_Step.CONTINUE | _Step.STOP` and the layer
loop checks it once (F-ENG-6 dissolved). The `is_solved`-neutralising hack
(F-KER-1) dies because `policy.solved(kb)` *is* the right predicate per entry.

The three policies + their accumulators move to a new
`monotonic/policy.py`, dropping `solver.py` from **1360 lines** (the biggest
file in the codebase) to a thin shared-loop + public-entry module — addressing
the "too-long file" axis, not just the function.

## Tasks

### T1.7b.2.1 — Extract the phases (no policy yet, behaviour-exact)

First, **purely mechanical** extraction with the `entry` branches still inside
the new functions — proves the split is behaviour-preserving before semantics
move:

- `_setup(root_kb, cfg, dumper, stop_after) -> _LoopCtx` — `solver.py:662-687`
  (config resolve, `stats`, `t_start`, `lstate`, `shuffle_rng`). `_LoopCtx` is
  a dataclass replacing the ~7 closure-captured locals.
- `_phase1_root(ctx) -> Verdict | None` — `:755-821` (initial saturate, root
  contradiction, root-`solved`, forced-positive cascade, empty-alive).
- `_phase2_layers(ctx) -> None` — the `for layer` shell `:826-1210`, with the
  per-candidate body extracted to `_integrate_commitment(ctx, c, result) ->
  _Step` (`:880-1166`), itself split into `_handle_dead` (`:883-948`),
  `_handle_solution` (`:991-1038`), `_handle_alive` (`:1040-1166`), and a
  `_layer_end_prune` (`:1168-1210`).
- keep the existing nested `_finalise_solve/_finalise_gaps/_finalise_contradictions`
  as-is for now.

Gate: full suite + 3-variant acceptance green after T1 alone.

### T1.7b.2.2 — Introduce `EntryPolicy`; move the `entry ==` branches onto it

Convert the three flows into `SolvePolicy` / `GapsPolicy` /
`ContradictionsPolicy`. Each `_handle_*`/`_phase1_root`/`_finalise_*` branch
that reads `entry` becomes a method on the policy. The shared functions lose
their `entry` parameter and every `if entry == …`. Delete the `entry`
discriminator from `_explore_layers`'s signature.

### T1.7b.2.3 — Retire the `Mode` neutraliser (F-KER-1 / F-ENG-4)

`SolvePolicy.solved = is_solution_node`; `Gaps/Contradictions.solved =
lambda kb: is_solved(kb, Mode.SOLVE)`. Delete `mode = Mode.CONTRADICTIONS …`
(`:675`) and the 10-line comment justifying it. Drop `Mode.GAPS` (never
constructed). `is_solved` stays (gaps/contradictions use it) but loses its
*termination* role on the solve path. `verdict.py` `Mode` shrinks to what's
actually distinguished.

### T1.7b.2.4 — Collapse the duplications (F-ENG-5/6/7)

- `_Step` enum replaces `phase_2_done` (F-ENG-6): handlers `return _Step.STOP`;
  `_phase2_layers` breaks once on it.
- `_lattice_public(expected_type)` (F-ENG-5): the verbatim post-amble of
  `gaps_solve`/`contradictions_solve` (`:1295-1297` / `:1358-1360`) → one
  helper; the public `assert`s become a documented internal-invariant check
  (or `RuntimeError`, F-ENG-14). `solve` keeps its own return (distinct type +
  `proof=None` contract).
- `_union_dead_cores(deads)` + `_source_frontier_core(kb)` (F-ENG-7): the
  4 copies at `:225-228,541-544` and `:441-446,739-742` → 2 helpers.

### T1.7b.2.5 — Split `_LatticeLoopState` (F-ENG-8)

Each policy owns its accumulator: `_SolveAccum(solution_nodes, dead_commitments,
truncated)` vs `_LatticeAccum(solutions, dead_commitments, kb_index,
state_hash_merges, root_was_solved)`. The shared loop never sees a field its
policy can't fill. The two `# type: ignore[arg-type]` (F-ENG-13) vanish because
lattice handling now lives on the lattice policy, typed
`Literal["gaps","contradictions"]`.

### T1.7b.2.6 — (optional) `_BaseStats` + `_TimelineMixin`

If budget allows: factor the 11 shared counters of `MonotonicStats`/
`LatticeStats` into `_BaseStats` (F-ENG-9), and the verbatim
`_emit_timeline`/`summary`/`close` of the two dumpers into a `_TimelineMixin`
(F-ENG-11). Skippable — low metric impact.

  `bench_solve_monotonic_pypy.sh` emit identical `state_hash` dedup counts and
  `solution_nodes` to baseline (capture both before T1). The SAT↛⊥ / UNSAT↛model
  invariant test (P1.7a) must stay green at every task boundary.
- Land T1 (mechanical split) and T2-T5 (semantic move onto policies) as
  **separate commits** so a behaviour regression bisects to the exact step.

# S1.7c.25 — Promote a real shared DOT emitter API

**Phase:** P1.7c, Track B.
**Estimate:** ~4–5 hr. **High leverage / high risk** — six renderers; output
must stay byte-identical. The headline RTC item.
**Depends on:** P1.7b baseline (which already deduped `esc()` / `multiline()`
into `render/dot_util.py`). Pairs with
[S1.7c.26](s1.7c.26_decompose_to_dot.md).
**Findings:** [F-RTC-1](../p1.7b_review_and_refactor/findings.md#rtc--render--trace--cli--ir)
(+ folds in [F-KB-8](../p1.7b_review_and_refactor/findings.md#kb--data-model-kbpy)).

## Goal

`dot_util.py` was meant to be the one DOT home, but only
`quote`/`value_label`/`fact_label`/shapes moved there. **Line construction is
still re-rolled** across six files:

- the `rankdir=LR; node[fontname=…]` preamble ~6×;
- the md5 `(relation,args)→f_<hash>` node-id helper 3× — and they **drift**
  (`kb/render.py:71` uses hash length 10 vs `provenance.py:173` length 12,
  the F-KB-8 bug);
- two parallel emit idioms (`ir/to_dot._Builder` class vs everyone else's
  `lines: list[str]` + `join`).

## Tasks

- Promote a real API into `dot_util.py`: `digraph(...)`, `node(...)`,
  `edge(...)`, `cluster(...)`, `node_id(relation, args)` (one canonical hash
  length — pick one, document it), reusing the existing `esc`.
- Route all six renderers through it: `kb/render.py`, `render/slice.py`,
  `render/lattice_dag.py`, `ir/to_dot.py`, `inference/provenance.py`.

## Acceptance

- **Every renderer's DOT is byte-identical to baseline** (golden-output
  tests across each `ir dot` / render path) — except the deliberate
  node-id-length unification, which is a one-time golden refresh, noted.
- One `node_id` helper; one preamble; one emit idiom.
- Full suite green.

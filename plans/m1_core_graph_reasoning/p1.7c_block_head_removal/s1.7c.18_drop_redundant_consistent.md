# S1.7c.18 — Drop the redundant `consistent()` on the alive solve branch

**Phase:** P1.7c, Track B.
**Estimate:** ~1 hr. **Perf change — must not move verdicts.** Gate on the
3-variant acceptance *and* a bench.
**Depends on:** P1.7b baseline.
**Finding:** [F-ENG-12](../p1.7b_review_and_refactor/findings.md#eng--monotonic-search-engine-inferencemonotonic).

## Goal

On the alive solve branch (`solver.py:~956`) the engine calls
`is_solution_node` (`inference/solution.py:61`), which re-runs `consistent()`
= a **full `ContradictionDetector(kb).detect()`** — on a kb whose
`try_commitment_set` was *already* proved consistent for `kind == "alive"`.
That is one redundant full detect per alive commitment, on the hot path.

## Tasks

- On the alive branch, call `complete()` (`solution.py:51`) **directly**
  instead of `is_solution_node` (which is consistency-check + completeness);
  the consistency half is already known on this branch.
- Keep `is_solution_node` for the non-alive path that genuinely needs the
  re-check.

## Acceptance

- **Identical verdicts + enterings** across the 3-variant gate (this is a
  perf-only change — any verdict drift means the alive-consistency invariant
  did not hold and the change is wrong).
- `bench_solve_monotonic_pypy.sh zebra2` wall-clock ≤ baseline; the
  detect-count drops by the number of alive commitments.

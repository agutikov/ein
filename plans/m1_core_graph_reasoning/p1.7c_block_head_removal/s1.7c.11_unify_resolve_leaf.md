# S1.7c.11 — Unify the two swapped-arg `_resolve`

**Phase:** P1.7c, Track B.
**Estimate:** ~1 hr. Low risk — pure consolidation.
**Depends on:** P1.7b baseline (no behaviour change).
**Finding:** [F-KER-7](../p1.7b_review_and_refactor/findings.md#ker--inference-kernel-inferencepy-excl-monotonic).

## Goal

Two `_resolve` functions live in one package with **swapped argument order**
— `firing.py:62` `_resolve(slot, bindings)` vs `predicates.py:48`
`_resolve(bindings, slot)` — both resolving `Var` / `Atom` / `Int` against a
bindings dict. A copy-paste-between-files footgun: a future edit to one will
read correctly but wire the args backwards in the other.

## Tasks

- Extract one `_resolve_leaf(slot, bindings)` (pick the `(slot, bindings)`
  order — reads as "resolve this slot under these bindings") into a shared
  leaf-resolution module both `firing` and `predicates` already could import.
- Route both call sites through it; delete the two local copies.
- If a full share is awkward (different surrounding context), at minimum
  **align the arg order** and rename so the two cannot be confused.

## Acceptance

- One leaf-resolution implementation; both call sites use the same arg order.
- Firing + predicate tests green; full suite green.

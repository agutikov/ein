# S1.7c.30 — `linearize` dispatch table

> **Status: WON'T-DO — dropped 2026-06-03 (user decision).** A uniform
> `{VerdictType: _from_*}` dispatch table is a poor fit and a byte-identity
> hazard: the per-verdict `linearize` branches are deliberately *asymmetric*
> — the Ambiguity branch omits `full_kb_dot` (even under
> `full_kb_snapshots=True`) and Contradiction omits both diagram kinds — so a
> table that treated the three uniformly would change the emitted trace. There
> are no 3-verdict-kind golden fixtures to gate such a change, and the
> function is not a real pain point (post-P1.7a it builds 4 `Trace()`s, not
> the doc's 5). Net value is low against a real regression risk; the most a
> safe version could do is factor a single guarded `_trace_diagrams` helper
> each branch passes `full_kb_dot=None` to — marginal. Not pursued.

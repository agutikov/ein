# S1.7c.13 — Extract the `_lattice_public` entry post-amble

**Phase:** P1.7c, Track B.
**Estimate:** ~1–2 hr. Low risk — wrapper consolidation, no loop changes.
**Depends on:** P1.7b baseline.
**Findings:** [F-ENG-5](../p1.7b_review_and_refactor/findings.md#eng--monotonic-search-engine-inferencemonotonic)
(+ folds in [F-ENG-14](../p1.7b_review_and_refactor/findings.md#eng--monotonic-search-engine-inferencemonotonic)).

## Goal

`solve` / `gaps_solve` / `contradictions_solve` are thin wrappers; the
gaps/contradictions pair share a **verbatim 4-line post-amble**
(`assert isinstance(verdict, X); assert proof; return verdict, proof.stats`,
`solver.py:1383` and `:1446`) differing only in `X`. F-ENG-14: that public
return-type contract is enforced **only by `assert`** — stripped under
`python -O`, where a contract slip degrades to an opaque `AttributeError`.

## Tasks

- Extract `_lattice_public(entry, expected_type) -> (Verdict, Stats)` that
  runs the loop and performs the typed post-amble once.
- Replace the bare `assert isinstance(...)` with an explicit
  `raise TypeError(...)` so the contract survives `-O`.
- Route both wrappers (and `solve`, to the extent its post-amble overlaps)
  through it.

## Acceptance

- One post-amble; `gaps_solve` / `contradictions_solve` return identical
  verdict + stats to baseline.
- A regression test that a wrong-typed verdict is rejected **under `-O`**
  (`python -O -m pytest …`), not just in debug mode.
- Full suite green; 3-variant gate byte-identical.

**Phase:** P1.7c, Track B (P1.7b refactor-debt tail).

## Goal

`FactId = tuple[str, tuple]` is a domain-agnostic predicate-id, but it is
**defined twice** (`inference/apriori.py:32`, `inference/nogoods.py:51`) and
`back_prop.py` / `commitment.py` only annotate with it — while `solution.py`
imports it *from `apriori`*, coupling the solution predicate to the BFS module
by accident of where the alias happens to live.

## Tasks

- Define `FactId` **once** in a neutral module (recommend a tiny
  `inference/factid.py`, or fold into an existing neutral home such as
  `inference/nogoods.py`'s set-semantics module only if it is not itself a
  BFS-coupled module — prefer a dedicated file).
- Rewrite `apriori.py`, `nogoods.py`, `back_prop.py`, `commitment.py`,
  `solution.py` to import the single alias; delete the duplicate definition.

## Acceptance

- `grep -rn "FactId = tuple" src/` → **1** hit.
- No module imports `FactId` from `apriori`.
- Full suite green; `ruff check .` green.

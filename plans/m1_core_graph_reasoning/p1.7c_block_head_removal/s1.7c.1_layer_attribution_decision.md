# S1.7c.1 — Decide layer attribution for flat forms (the gating design call)

**Phase:** P1.7c, Track A (block-head removal).
**Estimate:** ~½ day — a **design decision + a written rationale**, no code
ships in this stage (it gates S1.7c.2–.5).
**Depends on:** nothing — this is the crux all later Track-A stages build on.
**Resolves:** [README Q1](README.md#open-design-questions-the-reason-its-not-trivial)
(layer attribution) — the one the README flags as "**the crux**".

## The problem

Today the wrapper block heads *carry* the [`Layer`](../../../ein.py/src/ein_bot/kb/entities.py)
of their children: `(ontology …)` → `Layer.ONTOLOGY`, `(facts …)` →
`Layer.FACT`, `(reasoning …)` → `Layer.REASONING` (`from_ir.py:408-412`).
Drop the wrappers and a flat fact has **no layer signal**. Q1 asks where the
signal comes from instead.

## What the layer is actually load-bearing for (investigated 2026-06-02)

The whole premise that "the layer split is load-bearing for the contradiction
detector" is **only half true**, and the half that matters is *not*
ONTOLOGY-vs-FACT:

- **`REASONING` is the only inference-distinguished layer.** The detector's
  pairing logic (`lookahead.py:139,145`) flags `(not g)` / `g` as a
  contradiction **iff the existing fact is `Layer.REASONING`** — *"cross-layer
  is not a contradiction; see the detector's module docstring."* Given
  (authored) facts never pair this way; only engine-derived ones do.
- **No inference path branches on `ONTOLOGY` vs `FACT`.** Grep of
  `inference/` for those two values returns only *producers* and *render*:
  `closed.py:87` **writes** `CLOSED` as ONTOLOGY; `state_dump.py:120` reads
  ONTOLOGY for **dump styling**. The loader itself already treats them as one
  population — `from_ir.py:331`: `layer in (Layer.FACT, Layer.ONTOLOGY)`.
- **`REASONING` is engine-only on input.** Authored `(reasoning …)` is
  "usually empty; the engine dumps it after a solve" (grammar header). So
  authored flat input never needs to *write* REASONING.

**Conclusion:** for *authored* flat forms the only distinction to recover is
ONTOLOGY-vs-FACT, and it is **purely provenance / render** (the `--layers` /
`--colour-by layer` views, `slice.py:200`, `render.py:364`) — not soundness.

## The decision to make (recommended resolution)

Derive the layer from the **provenance annotation already on the form**, which
is the *same* signal `from_ir.py:325-331` uses to pick the `Provenance` kind:

| flat form carries… | layer | rationale |
|---|---|---|
| `:rule` / `:using` | `REASONING` | derived — engine working-memory dump |
| `:source "(N)"` | `FACT` | an explicit, numbered problem statement |
| neither (schema-shaped: `relation`, or an unannotated structural fact) | `ONTOLOGY` | implicit background assumption |

This is fully derivable, needs no new syntax, and keeps the per-fact
`Fact.layer` field (so every existing render/filter path is untouched).

**Alternatives to weigh and record (pick one, justify in writing):**
1. **Derive from provenance** (above) — recommended; zero new surface.
2. **Per-form `:layer ontology|fact` keyword** — explicit, but adds a reserved
   keyword and verbosity to every line the author wants in ONTOLOGY.
3. **Collapse authored input to a single given-layer** (ONTOLOGY≡FACT for
   authored facts; engine still writes REASONING) — simplest, but loses the
   author's explicit/implicit distinction in the renderer.

# S1.7c.3 — Loader (+ dumper): route flat forms by head; back-compat shim

**Phase:** P1.7c, Track A.
**Estimate:** ~1 day.
**Status:** **DONE 2026-06-02** — see [§ Implementation notes](#implementation-notes-2026-06-02).
**Depends on:** [S1.7c.1](s1.7c.1_layer_attribution_decision.md) (the
`_layer_of` rule) + [S1.7c.2](s1.7c.2_flat_form_grammar.md) (the flat
grammar). Pairs with [F-KB-7 / S1.7c.23](s1.7c.23_flatten_from_ir_load.md) —
both rewrite `load`'s dispatch; **do them together**.
**Resolves:** [README Q2](README.md#open-design-questions-the-reason-its-not-trivial)
(migration / deprecation window).

## Goal

Make [`from_ir.load`](../../../ein.py/src/ein/kb/from_ir.py) classify a
**flat** form sequence by head and attribute layer per S1.7c.1 — while
**accepting the old wrapped forms** behind a deprecation shim so the 4
examples + ~40 test fixtures keep loading until S1.7c.4 migrates them.

## Tasks

- **Flat dispatch.** Replace the block-collecting preamble
  (`from_ir.py:366-395`, the six `*_blocks` lists + head `if`-ladder) with a
  per-form classifier: `relation` → relation decl; `rule`/`hrule` → rules
  pass; `query`/`config` → their handlers; `trace` → ignored (as today);
  **anything else → a fact**, with `layer = _layer_of(form)` from S1.7c.1.
  Preserve the existing **three-pass ordering** (relations → rules → facts)
  by bucketing the flat stream into the same passes (relation/rule decls
  resolved before facts), so rule-name + relation resolution still precede
  fact ingestion.
- **Back-compat shim (deprecation window).** When a top-level head is a
  *wrapper* (`ontology`/`facts`/`reasoning`/`rules`), splice its children into
  the flat stream with the block's implied layer (ontology-children keep the
  S1.7c.1 derivation; `facts`→FACT; `reasoning`→REASONING) and emit a
  **deprecation warning** naming the file+loc. One code path, both surface
  forms.
- **Dumper / round-trip (the `load` dual).** Audit every writer that
  *constructs* block-head SForms for output — the `ir` CLI dump path and the
  state/solution dump that emits a `(reasoning …)` block. Switch them to emit
  **flat** forms (derived facts carry `:rule`/`:using`, so they re-classify to
  REASONING on reload — verify the round-trip). Keep emitting wrapped only if
  a consumer still needs it during the window; prefer flat-out immediately.

## Implementation notes (2026-06-02)

- **Two paths, one logic, byte-identical.** Rather than splice wrapper
  children into the flat stream, the shim routes a wrapper-headed form
  through the **existing** per-pass code (`_ingest_ontology` /
  `_ingest_rules` / `_ingest_facts`) — so a still-wrapped file loads
  *exactly* as before (zero behaviour change), while flat forms run a
  parallel bucketing into the same three passes (relations → rules →
  facts). Shared helpers were extracted: `_ingest_relation` (one decl),
  `_ingest_one_fact` (one fact at a given layer); `_ingest_rules` now takes
  an iterable of rule forms. Bucketing preserves source order, so a flat
  rewrite that keeps the block sequence produces the same KB
  **insertion order** as the wrapper — verified byte-identical for
  `zebra.ein` (68 facts + relations + rules) and (in S1.7c.4) the whole
  corpus via the migration `--verify` oracle.
- **`_layer_of`** implements the S1.7c.1 rule: explicit `:layer` wins, else
  `:rule`/`:using`→REASONING, `:source`→FACT, else ONTOLOGY. `:layer` is a
  plain kw-pair (dropped by `_fact_args`, ignored by the provenance arm).
- **Dumper → flat + minimal `:layer`.** `_kb_to_ein_text` emits a flat
  sequence (ONTOLOGY facts first for readability). `_fact_to_sform` now
  emits `:layer` **only** when provenance-derivation wouldn't recover the
  layer — fixing a latent bug where a hypothesis-provenance REASONING fact
  (no `:rule`) lost its layer on reload (the wrapped loader never read the
  old unconditional `:layer reasoning`). Round-trip verified exact across
  ONTOLOGY / FACT(+source) / FACT(no source) / REASONING(+rule) /
  REASONING(hypothesis).
- **Deprecation warning** = one `DeprecationWarning` per wrapper, located
  via a depth-first `_first_loc` (synthetic heads carry no loc; the first
  terminal token does) → e.g. `zebra.ein:66:9: (rules …) … deprecated`.

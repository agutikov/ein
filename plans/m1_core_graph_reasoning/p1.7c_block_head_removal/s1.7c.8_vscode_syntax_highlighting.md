## Why it moved (and why now)

The P1.6 note "owned by P1.6 since it lives next to the DOT renderer work"
was only half the story Рђћ the grammar it highlights is the **surface
syntax**, and P1.7c is the surface-syntax phase. The original keyword list
also still named the block wrappers `facts` / `ontology` / `reasoning` /
`rules`, which **no longer exist** ([S1.7c.4](s1.7c.4_migrate_and_drop_shim.md)).
Building it now means highlighting the *current* (flat) language from the
start, against the single source of truth.

## Goal

A lightweight TextMate / LSP-free grammar definition
(`.tmLanguage.json`, e.g. under `utils/vscode-ein/` or `editors/vscode/`)
that gives `.ein` files structural highlighting in VSCode (and any
TextMate-grammar consumer: Sublime, GitHub `linguist`, `bat`, Рђд). No
language server, no build step.

## What to highlight (flat surface, P1.7c)

- **S-expression structure** Рђћ `( Рђд )` nesting; atom vs `"string literal"`
  vs `42` / `1..*` (RANGE) vs `;`-comment / `#| Рђд |#` block comment.
- **Declarators** (the closed reserved set Рђћ
  [`06_reserved_names.md`](../../../docs/kernel/ir/03-ein-lang/06_reserved_names.md)):
  `relation` ┬и `rule` ┬и `hrule` ┬и `query` ┬и `config` ┬и `trace`
  (+ `macro` once [P1.8 S1.5.9](../p1.8_ein_lang_modules/s1.5.9_ein_lang_macros.md)
  lands). **Highlight these only in head position** (first atom after `(`),
  so a relation/atom that merely *contains* the word isn't miscoloured.
  **Do NOT highlight `ontology` / `facts` / `reasoning` / `rules`** Рђћ they
  are ordinary fact heads now, not keywords.
- **Rule-body / РіЦ primitives:** `not` ┬и `and` ┬и `or` ┬и `neq` ┬и `false` ┬и
  `absent` ┬и `forall` ┬и `open` (the `primitives.STRUCTURAL` / `SUGAR` sets);
  **predicates** `eq` ┬и `neq`.
- **Keywords** (`:keyword` form) Рђћ `:match` ┬и `:assert` ┬и `:why` ┬и
  `:priority` ┬и `:source` ┬и `:rule` ┬и `:using` ┬и `:layer` ┬и `:mode` ┬и
  `:goal` ┬и `:hrules` ┬и `:hypothesis-relations` ┬и Рђд (a single
  `:[a-z][\w-]*` rule is enough; a fixed list is optional polish).
- **Variables** Рђћ `?var` (and `?R*` with the `*` tail).
- **Wildcard** Рђћ bare `_`.
- Optional niceties: distinct scope for the `:layer` *value*
  (`ontology`/`fact`/`reasoning`) and for the `=` head.

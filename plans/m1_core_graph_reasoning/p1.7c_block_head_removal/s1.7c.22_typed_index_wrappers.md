# S1.7c.22 — Typed `_FactIndexes` / `_StaticIndexes` wrappers

**Phase:** P1.7c, Track B.
**Estimate:** ~4 hr. **High risk** — the central kb data structure touched by
`__init__` / `rebuild` / `fork` / `_index_fact` / `snapshot`. Gate hardest.
**Depends on:** P1.7b baseline. **Structurally enables**
[S1.7c.21](s1.7c.21_snapshot_shallow_copy.md) and retro-justifies the
S1.7b.4 `fork` index-sharing / in-place writes.
**Finding:** [F-KB-9](../p1.7b_review_and_refactor/findings.md#kb--data-model-kbpy).

## Goal

Eight loose `_*_by_*` dicts are hand-maintained in **four** places
(`__init__` / `rebuild_indexes` / `fork` / `_index_fact`). `_index_fact`
*correctly* omits the four post-load-immutable indexes — but **nothing
enforces** the load-invariance that makes that omission correct. One wrong
edit silently corrupts index state (exactly the class of bug F-KB-3 was).

## Tasks

- Introduce two wrappers:
  - `_FactIndexes` — the four fact-derived indexes; mutable, with `add(fact)`
    + `copy()` (real copy).
  - `_StaticIndexes` — the four post-load-immutable indexes; `copy()` is
    identity (shared by reference, as `fork` already does).
- Route `__init__` / `rebuild` / `fork` / `_index_fact` / `snapshot` through
  the wrappers; the invariant is now in the **type**, not a comment.

## Acceptance

- Behaviour **identical** across the full 3-variant gate + fork-parity +
  shuffle-invariance + the S1.7b.4 double-index regression tests.
- No public KB API churn (wrappers are internal).
- Full suite green; bench ≤ baseline.

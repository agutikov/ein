# S1.7c.4 — Migrate `examples/` + fixtures to flat forms; drop the shim

**Phase:** P1.7c, Track A.
**Estimate:** ~1 day (mostly mechanical churn + a re-acceptance pass).
**Status:** **DONE 2026-06-02** — see [§ Implementation notes](#implementation-notes-2026-06-02).

## Implementation notes (2026-06-02)

- **Scope was 53 example files, not 4.** The stage's "4 example files" count
  omitted the `examples/**` subtrees (`branching/`, `saturation/`,
  `domain_elim/`, `features/`, `lattice/`, `ein-bugs/`). All 53 wrapped
  examples were migrated; `examples/broken/` hand-rewritten to *flat*
  malformed fixtures (still `rc==1`).
- **The unwrap tool** is [`utils/flatten_ein.py`](../../../../utils/flatten_ein.py)
  (committed, `--help`): a comment-preserving structural text transform (NOT
  parse∘dump — the examples' comments survive) that hoists each wrapper's
  children and injects `:layer` only where the S1.7c.1 derivation would
  disagree with the block layer. Its `--verify` mode is the S1.7c.1/.3
  **oracle** (loads wrapped + flat, diffs facts+layer+provenance+relations+
  rules); all 53 verified byte-identical. It also has a `--py` mode that
  flattens the inline ein fixtures inside Python test files, gated by a
  per-literal parse + KB-equivalence check (so it never corrupts a docstring).
- **~35 behavioral test files** migrated via `--py`; the 3 grammar/AST/render
  test files (`test_ir_parser` / `test_ir_ast` / `test_ir_to_dot`) +
  navigation sites hand-migrated (they assert parse *structure*, so the
  `(form,) = parse("(rules (rule …))"); form.args[0]` idiom became
  `parse("(rule …)"); form`). The renderer source needed flat-awareness too:
  `ir/to_dot.py` and `render/constraints.py` now **regroup** flat forms into
  the layer views their renderers draw, and `cli.py` collects flat `(rule …)`
  forms instead of a `(rules …)` block.
- **Shim + grammar dropped:** `from_ir` lost the wrapper buckets +
  `_warn_deprecated_wrapper`; `grammar.lark` lost the four block productions;
  `ast.py` lost the four wrapper methods. A stray `(facts …)` now loads as a
  fact (`test_former_wrapper_head_is_a_fact`).
- **Acceptance:** full pytest **1031 passed**; the M1 gate
  (`run_tests.sh --acceptance-only`, PyPy) **8 passed** with verdicts intact
  (zebra2-hints → Ambiguity k=2, zebra2-bad → Contradiction k=0, invariants
  Solution k=1 / Contradiction k=0); `ruff` clean; gallery render + `ir dot`
  / `render rules` / `render constraints` smoke-tested on flat input.
**Depends on:** [S1.7c.3](s1.7c.3_loader_flat_routing.md) (the flat loader +
shim must be green first).
**Resolves:** [README Q2](README.md#open-design-questions-the-reason-its-not-trivial)

## Goal

Rewrite every wrapped `.ein` to flat forms, delete the back-compat shim, and
make flat the **only** accepted surface — closing the deprecation window.

## Scope (measured 2026-06-02)

- **4 example files**: `examples/zebra.ein` (7 block heads — the heaviest),
  `zebra2.ein`, `zebra2-minus-15.ein`, `zebra2-hints.ein` (3 each).
- **~40 test fixtures** under `ein.py/tests/` use block heads (inline
  strings + `.ein` files) — the bulk of the churn.
- Any `examples/ein-bugs/` + `examples/broken/` fixtures that use wrappers
  (keep `broken/` still rc==1 — a *flat* malformed file, not a wrapped one).

## Tasks

- **Mechanical unwrap.** A scripted rewrite (a committed
  `utils/` one-shot, per the project's *persistent-script* rule — not an
  inline one-liner): strip the wrapper, hoist children to top level,
  attaching the layer signal S1.7c.1 chose (e.g. ensure each former-`facts`
  child keeps its `:source`; former-`reasoning` children keep `:rule`/
  `:using`). Run it over `examples/` + the `.ein` fixtures; hand-fix the
  inline-string fixtures the script can't reach.
- **Verify equivalence per file** before/after: load both, diff the KB
  (the S1.7c.3 oracle) — the rewrite must be **KB-preserving**, not just
  parse-clean.
- **Drop the shim** + the deprecation warning from `from_ir`; the wrapper
  heads (`ontology`/`facts`/`reasoning`/`rules`) become **unknown top-level
  heads** → they now classify as *facts* (or error, per the closed
  declarator set). Add a test asserting a stray `(facts …)` is no longer a
  recognised wrapper.
- Re-run the **examples gallery** render + the zebra2 trace so the example
  artifacts reflect flat input.

## Acceptance

- No `(ontology …)` / `(facts …)` / `(reasoning …)` / `(rules …)` wrapper
  remains in `examples/` or `ein.py/tests/` (grep → 0, except any
  intentionally-wrapped `broken/` lint fixture, which must still fail).
- The M1 acceptance gate (`run_tests.sh` PyPy 3-variant) is **green with the
  same verdicts + enterings** as before the migration — flat input changes
  syntax, not semantics.
- Full pytest suite green; the unwrap script is committed + `--help`-able.

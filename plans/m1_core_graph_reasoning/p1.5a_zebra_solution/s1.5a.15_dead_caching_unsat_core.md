## Why today's caching is too narrow

`builder.state_index` keys by `state_hash(kb)` over the ENTIRE
fact set (sans bookkeeping). Two branches that reach identical
kb states collapse to one SearchNode — sound, but tight: two
branches reaching the *same dead reason* via different
saturated states are stored as two distinct dead nodes. The
right equivalence class for dead is "shares the unsat-core" —
strictly looser than state-hash equality.

`verdict_at` in `_consume` caches per-call alive / cond-dead
verdicts. Useful but volatile (S1.5a.14 invalidates on bubble).
Not the focus of this stage.

`back_propagate` writes `(not h)` only when the death is
**fully** unconditional — its unsat-core's hypothesis-terminal
set is exactly `{h_self}`. Deaths that rely on *some* ancestor
hypothesis (but not all) get classified as "cond-dead" and
their `(not h)` is **not asserted as a fact**, even though
`(not h)` is provably true at every ancestor at-or-below the
shallowest required hypothesis's depth.

Promoted-dead branches (the parent dies because every child
died — `_promote_verdicts` flips the verdict bottom-up at solve
end) currently don't back-prop their **own** hypothesis at all,
even when the aggregated subtree justifies it.

## Three phases

### Phase 1 — Direct-dead cache by unsat-core

A branch died because `ContradictionDetector(kb).detect()`
found a clash. The unsat-core is the source-frontier of
derivations of both sides of the clash (already computed via
`kb.unsat_core(witness for witness in contradictions)`).

**Cache rule.** Any future kb containing the unsat-core's
source facts (and no neutralising facts — see "soundness
under monotonic adds" below) is dead. Cache entry:
``frozenset((f.relation_name, f.args) for f in unsat_core)``.

**Cache structure.** A side-list on the builder:

```python
@dataclass
class _DeadByCore:
    core: frozenset[tuple[str, tuple]]  # source-frontier fact ids
    representative_nid: BranchId        # the first node that hit it
```

`builder.dead_index: list[_DeadByCore]`. Linear scan on each
`_explore` entry: for each entry, check if the new kb's fact
set is a superset of the cached core. If yes, short-circuit to
the representative.

**Soundness under monotonic adds.** Dead-ness is monotonic:
adding facts can only tighten constraints, never resolve a
contradiction. So "kb contains the core" ⇒ "kb is dead" stays
sound under our engine's append-only saturation. The
S1.5a.14 bubble that writes `(not h)` to ancestors is also
monotonic — doesn't violate this.

**Cost.** O(|dead_index|) per lookup. Acceptable while
|dead_index| is small; needs an index structure (e.g. trie on
sorted core tuples) once it grows past a few hundred entries.

### Phase 2 — Promoted-dead cache

A branch X is promoted dead by `_promote_verdicts` when:
- X.alive ≠ ∅ (else X is a true leaf, not promoted), AND
- Every child of X died.

X's "unsat-core" doesn't exist directly — X's own kb is fine,
no clash. Instead the death is **aggregable**:

```
X's unsat-core := ⋃ child_unsat_cores  (recursively unrolled to
                                          direct-dead leaves)
```

The recursion always grounds out — every promoted-dead subtree
eventually rests on direct-dead leaves with real contradictions.

**Cache key, second component: `alive_fp`.** The aggregated
core alone is not enough — a future kb Y with the same core
but a *larger* alive set has new hypotheses not yet proven
dead. We pair the entry with X's alive-set fingerprint:

```python
alive_fp := frozenset((h.relation_name, h.args) for h in X.alive)
```

Cache hit iff:
- `Y.alive ⊆ X.alive_fp` — Y considers no new hypotheses, AND
- `aggregated_core ⊆ Y.kb` — Y has every fact the subtree's
  dead leaves rested on.

The strict-subset case `Y.alive ⊂ X.alive_fp` is sound — Y
considers fewer extensions, all of which we proved die.

**Where to compute.** In `_promote_verdicts`'s bottom-up walk.
When a node flips from "open" → "dead", aggregate its
children's cores into its own and register in
`builder.dead_index` (Phase 1's side-list, now carrying
optional alive_fp).

### Phase 3 — Per-level back-prop

The S1.5a.14 bubble writes `(not h)` to **every** ancestor when
the death is fully unconditional (unsat-core hypothesis
terminals = {h_self}). But "fully" is the strict case. The
death can be **partially** unconditional — its chain reaches
hypotheses at some ancestor depths but not above. At every
level *at-or-below the shallowest required hypothesis's depth*,
`(not h_self)` is provably true.

**Shallowest-required-depth analysis.** Walk the unsat-core's
hypothesis-kind terminals. Each carries `Provenance.from_hypothesis(branch=N)`
where N is the SearchNode's branch id; map to depth via
`builder` (or a depth-side index built during `_alloc_node`).
Let `D = max depth(h_i)` over non-own hypothesis terminals (or
`-1` if none — the fully-unconditional case).

Then `(not h_self)` is justified at every ancestor at depth
`L ≥ D`. (At `L < D`, the chain's deepest required
hypothesis isn't yet introduced — the justification breaks.)

Today's code:
- Fully unconditional (D = -1): writes to all ancestors. ✓
- Partially unconditional (D ≥ 0): writes nothing — the death
  is just "cond-dead" with the SearchNode allocated. ✗

Fix: write `(not h_self)` to ancestors at depth `≥ D`. Carries
the same monotonicity guarantees as Phase 1's bubble.

**Symmetry with promoted-dead back-prop.** Once Phase 3 lands,
promoted-dead branches should also back-prop their own
hypothesis. Specifically: when `_promote_verdicts` flips X from
open → dead, compute X's aggregated unsat-core; analyse its
shallowest-required-depth; if X's own hypothesis at depth N
isn't required (i.e., D < N), write `(not h_X)` to every
ancestor at depth ≥ D. This is the answer to the user's
question "if the root branch is dead, isn't its hypothesis an
unconditional negative?" — applied at every level, not just
root.

## Tasks

### T1.5a.15.1 — Direct-dead unsat-core cache

- **a.** Add `builder.dead_index: list[_DeadByCore]`.
- **b.** In `_explore`'s contradiction branch (the early-return
  for `contradictions`), after computing `unsat`, register a
  `_DeadByCore(core=frozenset(...), representative_nid=nid)`.
- **c.** At every `_explore` entry (before the state-hash dedup
  check), iterate `builder.dead_index` and short-circuit to the
  representative if any core is a subset of `kb._fact_by_id`'s
  reach (i.e., every core fact is in kb).
- **d.** Benchmark on zebra2-hints (7 hints) — expect tree node
  count to drop noticeably vs the 17 in the S1.5a.14 ship.

### T1.5a.15.2 — Promoted-dead cache via _promote_verdicts

- **a.** Augment `SearchNode` with a `aggregated_unsat_core:
  frozenset[Fact]` field, default empty.
- **b.** In `_promote_verdicts`'s bottom-up walk: when a node
  flips to "dead" (was "open", all children died), set its
  `aggregated_unsat_core` to the union of its children's
  unsat_core (for direct-dead leaves) and aggregated_unsat_core
  (for promoted-dead children).
- **c.** Register `_DeadByCore(core=aggregated_core,
  alive_fp=frozenset(...), representative_nid=nid)` in
  `builder.dead_index`. The alive-fp check happens at lookup.

### T1.5a.15.3 — Per-level back-prop

- **a.** Add a depth side-index: `builder.depth_of[nid] = depth`.
  Populated in `_alloc_node` (already receives parent_id; the
  parent's depth + 1 is the child's depth; or `0` if `parent_id`
  is None).
- **b.** Modify `back_propagate` (or add a sibling
  `back_propagate_per_level`) to accept `shallowest_required:
  int` — the max-depth of non-own hypothesis terminals in the
  unsat-core. Writes `(not h)` to every ancestor at depth
  ≥ `shallowest_required`. Existing fully-unconditional case
  collapses to `shallowest_required = -1`, all ancestors qualify.
- **c.** In `_consume`'s sweep, for **every** dead candidate
  (unconditional or conditional), compute the shallowest
  required depth and call the per-level back-prop. The old
  `is_unconditional_death` becomes a special case of this
  analysis.

### T1.5a.15.4 — Promoted-dead back-prop

After Phase 3's per-level mechanism is in place: in
`_promote_verdicts`, when a node flips open → dead, compute its
own aggregated unsat-core's shallowest-required-depth and
back-prop `(not h_X)` to ancestors at depth ≥ that.

This finally closes the user's "if root is dead, isn't h an
unconditional negative" question — applied recursively at every
level.

## Open questions

- **Q-S1.5a.15.A — Promoted-dead under DAG dedup.** Two
  branches dedup to one SearchNode via `state_index`. Does the
  aggregated_unsat_core still aggregate correctly when the
  same node has multiple parents in the DAG? Probably yes —
  aggregation is over CHILDREN, not parents.
- **Q-S1.5a.15.B — Cache structure scaling.** Linear scan over
  `dead_index` is O(|dead_index| × |kb.facts|). For deep puzzles
  with many distinct dead-cores, this dominates. Indexing
  candidates: trie on sorted (rel_name, args) tuples; or a
  hash-of-frozenset that buckets cores by their smallest member.
- **Q-S1.5a.15.C — Symmetric mirror for partial unconditionals.**
  `back_propagate`'s symmetric-promote handles the
  `(symmetric R)` case for the fully-unconditional path. Should
  the per-level back-prop also mirror to symmetric counterparts?
  Probably yes; same reasoning applies.

# S1.5a.13 — Acceptance: zebra2 solves uniquely

**Phase:** P1.5a (zebra solution + saturator/NAF gaps).
**Estimate:** ½ day — pure measurement + reporting after
everything else lands.
**Status:** terminal stage of P1.5a; the phase ships when this
stage's acceptance holds.
**Depends on:** [S1.5a.1](s1.5a.1_naf_semantic_rearch.md)
(NAF fix), [S1.5a.2](s1.5a.2_hypgen_pre_pruning_recovery.md)

## Context

This stage bundles the three upstream stages' results into a
single phase-acceptance verdict, matching the original
T1.5.8c.7 that was spun out into P1.5a on 2026-05-24.

The three criteria from the parent README:

1. `bench_solve examples/zebra2.ein` returns `Solution` with
   `(house-nation House_? Japanese) ∧ (house-pet House_? Zebra)`
   and `(house-nation House_? Norwegian) ∧ (house-drink House_?
   Water)` bound to specific houses, within a sensible time
   budget (target: < 60 s on a laptop).
2. The trace contains every named rule firing the idea-08
   walkthrough mentions, in structurally equivalent order.
3. The NAF race (or the per-rule workaround for it) is
   documented in [`docs/kernel/inference/`](../../../docs/kernel/inference/).

(2) is owned by S1.5a.12; (1) and (3) are this stage's gate.

## Tasks

### Task T1.5a.13.1 — Measurement

Run `bench_solve examples/zebra2.ein` on the laptop reference
hardware (Linux x86_64, 8-16 GB RAM); record:

- Verdict: `Solution` vs `Ambiguity` vs `Contradiction`.
- Solution bindings (the two zebra-target facts).
- Wall time + max RSS.
- Tree-node count + dead-branch count.
- HypGen stats (raw, per-filter, emitted).

Tabulate vs the original pre-B1 baseline + the post-S1.5a.1
+ S1.5a.2 baselines.

### Task T1.5a.13.2 — Trace + checklist re-run

Bundle:

- `--trace=zebra2.md` + `--trace-dir=zebra2/` (DOT snapshots).
- Re-run the S1.5a.12 checklist against the produced trace.

### Task T1.5a.13.3 — Docs

Update [`docs/kernel/inference/README.md`](../../../docs/kernel/inference/README.md)
(currently a stub) with:

- The NAF semantic invariant (per S1.5a.1's resolution).
- The per-puzzle workaround pattern (if S1.5a.1 ships option 3).
- The hypgen pre-pruning approach (per S1.5a.2's choice).

### Task T1.5a.13.4 — Phase verdict

Write a short ship/no-ship report at the end of this stage
file once measurement closes; if no-ship, list what remains and
queue follow-up stages.

## Acceptance

- **Hard gate:** `bench_solve examples/zebra2.ein` produces
  `Solution` with the expected zebra/water/Norwegian/Japanese
  bindings.
- **Soft gate:** wall time < 60 s on the laptop reference (if
  > 60 s, document why and queue follow-up perf work — could
  be P1.8 Theme B indexes, COW, or P1.5a.6 PyPy).
- All trace fidelity checklist rows ✅.
- `docs/kernel/inference/README.md` has the NAF + hypgen
  patterns documented.
- `pytest` suite green (no regression).

## Context

The B1 refactor (S1.5.8c) replaced the equivalence relation
`co-located : Attribute × Attribute` with specific bijective
`house-*` relations on House × Color / Nationality / Drink /
Cigarette / Pet. The structural payoff is clean typing; the
side-effect was losing the ~120 `(not (co-located A B))`
sibling-exclusive negatives that pruned hypotheses in pre-B1.

Specifically, in pre-B1: `(sibling-exclusive is-a co-located)`
asserted `(not (co-located A B))` for every same-type sibling
pair. The hypgen filter consumed these to drop candidates.

In B1: there's no `co-located`; `(sibling-exclusive is-a
house-color)` etc. would assert *malformed* negatives like
`(not (house-color Red Blue))` — args don't satisfy
house-color's House × Color signature. So no comparable
pruning exists.

Result: 122 root candidates vs ~101 — the 21-candidate delta is
exactly the cross-attribute sibling-exclusive pruning that no
longer fires.

## Approach options

Reproduced + sharpened from the parent README:

### (1) Hybrid encoding with co-located projection

Re-introduce `co-located : Attribute × Attribute` as a derived
equivalence relation, with `house-*` as typed projections via
a `projection` rule. `(sibling-exclusive is-a co-located)`
pre-emptively populates the negatives that hypgen filters on.

- **Pro:** most aligned with original zebra2's proven mechanics
  (the path discussed at length pre-S1.5.8c.7-spinout).
- **Pro:** keeps the bijective typing wins of B1.
- **Con:** adds a derived equivalence layer back into the KB;
  more rules to maintain.

### (2) `linked-induced` + `linked-transitive`

Two-tier reasoning at the linked-fact level.

- **Status:** rejected pre-spinout as functionally equivalent
  to chains the existing rules already cover; doesn't add
  unique pruning. Kept for visibility only.

### (3) Hypgen-side filter on linked facts

Engine extension: have hypgen consult `(linked …)` activator
facts directly when deciding which candidates to emit, rather
than relying on `(not …)` facts to prune.

- **Pro:** lifts the abstraction better — pruning logic in
  hypgen, not in derived negatives.
- **Pro:** no extra rules.
- **Con:** engine work — the hypgen path expands.

### (4) Per-house-* sibling-exclusive analog

Direct rules that derive `(not (house-color H_other V))` when
`(house-color H V)` is committed and `H_other ≠ H`.

- **Status:** functional+total cover SOME of this already; a
  more aggressive version pre-emptively asserts the
  cross-house siblings.
- **Pro:** stays inside B1's relation language; no co-located
  re-introduction.
- **Con:** N-1 negatives per assertion (where N = |House|);
  scales with domain size.

The hybrid (1) is the most aligned with original mechanics. (3)
is engine work but lifts the abstraction better.

## Tasks

### Task T1.5a.2.1 — Pick the approach

A short design pass (½ day) to weigh (1) vs (3) vs (4) against
the actual post-S1.5a.1 baseline. The hypothesis-count gap may
narrow or widen after NAF fix; remeasure before committing.

### Task T1.5a.2.2 — Implement the chosen approach

Sub-tasks depend on choice from T1.5a.2.1.

For **(1) hybrid co-located projection**:
- **a.** Re-declare `(relation co-located Attribute Attribute)`
  and `(equivalence co-located)` activators in zebra2 ontology.
- **b.** Per-house-* `projection` rule:
  `(rule house-color-projects-coloc () :match (house-color ?h ?c)
   :assert (co-located ?h ?c))` — one per bijective relation.
- **c.** `(sibling-exclusive is-a co-located)` activator
  populates same-type negatives.
- **d.** Hypgen filter consumes co-located negatives as before.

For **(3) hypgen-side linked filter**:
- **a.** Extend hypgen to walk `(linked R1 v1 R2 v2)`
  activator facts and skip candidates that would violate the
  link.
- **b.** Counter for visibility: `linked_activator_filtered`
  in the T1.5.4.7 `HypGenStats`.

For **(4) per-house-* sibling-exclusive analog**:
- **a.** One rule per bijective relation:
  `(rule sibling-exclusive-house-color () :match (house-color
   ?h ?v) (instance ?h2 House) (neq ?h ?h2) :assert (not
   (house-color ?h2 ?v)))`.
- **b.** Watch for N-1 fact explosion — measure REASONING
  layer fact count vs pre-implementation.

## Acceptance

- Root hypothesis count on zebra2.ein ≤ 101 (matching pre-B1
  baseline within ±5%).
- No regression in alive-set correctness; the surviving
  alive-set under the new pruning matches the pre-B1 alive-set
  pointwise (modulo the B1 relation renames).
- `pytest tests/inference/test_hypgen_pruning.py` extended
  with at least 3 new tests for the chosen approach.
- bench_solve time budget on zebra2: within 50% of pre-B1 (i.e.
  the perf gap from the missing pruning is mostly closed).

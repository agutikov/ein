## Ship notes 2026-05-27

The module surface below was implemented verbatim with four
small mechanical adjustments for codebase consistency. The
shipped `apriori.py` is the source of truth; the spec block in
"Module surface" stays as the design-time view.

| Adjustment | Why |
|---|---|
| `FactId = tuple[str, tuple]` (plain assignment, no `TypeAlias`) | matches the existing `nogoods.py` style for peer type aliases |
| Parameters renamed `A_prev` → `a_prev`, `S` → `s`, `T` → `t`, `C` → `candidate` | ruff N803/N806 enforce PEP 8 lowercase; mathematical-style uppercase is rejected. Docstrings updated to match. |
| `yield prefix + (s[-1], t[-1])` → `yield (*prefix, s[-1], t[-1])` | ruff RUF005 — tuple-spread literal preferred over concatenation; semantically identical |
| Two extra tests (`test_filter_candidate_empty_nogoods_is_no_op`, `test_layer_1_on_empty_alive`) | edge-case coverage for the no-op short-circuit + empty-input case; doc target of ~10 became 17 |

Type compatibility with `nogoods.py` verified: both modules
declare `FactId = tuple[str, tuple]`, so a `frozenset[FactId]`
nogood clause from `nogoods._nogoods` flows into
`filter_candidate`'s `nogoods=` parameter without conversion
(see Q1.5b.5 — unified refutation semantics).

## Module surface

`ein.py/src/ein_bot/inference/apriori.py`

```python
"""Apriori-style layer-(k+1) candidate generation from
layer-k alive set + current alive hypothesis set + current
nogoods.

Used by both `inference/monotonic/solver.py` and
`inference/lattice/solver.py`. Pure set-arithmetic; no kb
inspection, no saturator invocation, no engine-specific
state. See [P1.5b § frontier](../../plans/m1_core_graph_reasoning/p1.5b_lattice_search/open_questions.md#q15b2--frontier-strategy)
for the resolved design.
"""
from __future__ import annotations

from collections.abc import Iterable
from typing import TypeAlias

# Canonical-ordered tuple of (relation_name, args) fact-id tuples.
# Sort key is (relation_name, args) so the ordering matches
# `_path_ctx`'s shape and the existing `nogoods.py` helpers compose
# without conversion. See Q1.5b.3.
FactId: TypeAlias = tuple[str, tuple]
CanonicalSetId: TypeAlias = tuple[FactId, ...]

def canonicalise(elements: Iterable[FactId]) -> CanonicalSetId:
    """Sort + dedup an iterable of fact-ids into a CanonicalSetId."""
    return tuple(sorted(set(elements)))

def apriori_prefix_join(
    A_prev: Iterable[CanonicalSetId],
) -> Iterable[CanonicalSetId]:
    """Textbook Apriori-gen prefix-join.

    For each pair (S, T) in A_prev × A_prev that agree on their
    first |S|-1 elements with S[-1] < T[-1], emit
    S[:-1] + (S[-1], T[-1]).

    Yields each layer-(k+1) candidate of size |S|+1 exactly
    once. Caller is responsible for filtering against the
    current `_nogoods` + alive set (see `filter_candidate`).
    """
    sorted_prev = sorted(A_prev)
    for i, S in enumerate(sorted_prev):
        prefix = S[:-1]
        for T in sorted_prev[i + 1:]:
            if T[:-1] != prefix:
                break  # sorted — no further matches share the prefix
            if S[-1] < T[-1]:
                yield prefix + (S[-1], T[-1])

def filter_candidate(
    C: CanonicalSetId,
    *,
    alive: frozenset[FactId],
    nogoods: Iterable[frozenset[FactId]],
) -> bool:
    """True iff candidate C should be explored.

    Drops C if:
      - any element of C is not in the current alive set
        (covers single-element negatives back-propagated
        since A_prev was computed);
      - any nogood clause is a subset of C
        (covers multi-element conditional deaths whose
        clauses propagated up from earlier layers).

    The "every (k-1)-subset ∈ A_prev" check is covered by
    `apriori_prefix_join`'s construction — caller does not
    re-verify here.
    """
    if not all(h in alive for h in C):
        return False
    candidate_set = frozenset(C)
    for clause in nogoods:
        if clause.issubset(candidate_set):
            return False
    return True

def generate_layer(
    A_prev: Iterable[CanonicalSetId],
    *,
    alive: frozenset[FactId],
    nogoods: Iterable[frozenset[FactId]],
) -> list[CanonicalSetId]:
    """Convenience: prefix-join + per-candidate filter.

    Returns the surviving layer-(k+1) candidates in canonical
    (sorted) order. Order is deterministic; the engine's
    within-layer scoring (Q1.5b.2.c) re-sorts before iteration.
    """
    return [
        C for C in apriori_prefix_join(A_prev)
        if filter_candidate(C, alive=alive, nogoods=nogoods)
    ]

def layer_1(alive: frozenset[FactId]) -> list[CanonicalSetId]:
    """Layer-1 enumeration is special: every singleton from
    alive. Matches what `apriori_prefix_join` would produce on
    an A_0 = {()} (the empty set), but explicit for clarity."""
    return [(h,) for h in sorted(alive)]
```

### T1.5b.2.2 — Unit tests

New `ein.py/tests/inference/test_apriori.py`:

- **`canonicalise`** — round-trips a frozenset; idempotent;
  sort order matches `(rn, args)`.
- **`apriori_prefix_join`** on small fixtures:
  - `A_prev = [(a,), (b,), (c,)]` → emits `(a,b)`, `(a,c)`,
    `(b,c)` exactly once each.
  - `A_prev = [(a,b), (a,c), (b,c)]` → emits `(a,b,c)` exactly
    once.
  - `A_prev = [(a,b), (c,d)]` → emits nothing (no shared
    prefix).
- **`filter_candidate`** — passes when both checks pass;
  drops on alive miss; drops on nogood subset; drops on either.
- **`generate_layer`** — composition of the above on a 4-elem
  fixture, with one element back-propagated as dead and one
  2-clause nogood active. Verify the surviving set matches
  hand-computation.
- **`layer_1`** — every alive singleton, sorted.

Target ~100 LOC of tests, ~10 distinct cases.

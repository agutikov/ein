## Goal

Ship `contradictions_solve` as the second public entry on
the lattice engine. The implementation is mostly thin
glue:

- Construct `_LatticeLoopState(entry="contradictions")`.
- Call shared `_explore_layers` (from S1.5b.21).
- `state.dispatch_dead(C, result)` records a
  `DeadCommitment` into `state.dead_commitments`.
- `state.dispatch_solution(...)` is a no-op
  (CONTRADICTIONS doesn't track satisfying sets).
- Phase 3 synthesises `Contradiction(unsat_core=⋃ cores,
  proof=…)`.

Under `store_lattice=True`, the state-hash dedup MERGE
fires (S1.5b.22's checkpoint) — distinct dead commitments
with identical post-saturation kb collapse into one
multilabel SetNode.

## Algorithm

### `contradictions_solve` public entry

```python
def contradictions_solve(
    root_kb: KnowledgeBase,
    *,
    max_set_size: int = 5,
    config: SolverConfig | None = None,
    store_lattice: bool = False,
    dumper: LatticeDumper | None = None,
    max_time: float | None = None,
    max_enterings: int | None = None,
) -> tuple[Contradiction, LatticeStats]:
    cfg = config or root_kb.config or SolverConfig()
    root_kb.config = cfg

    state = _LatticeLoopState(entry="contradictions")
    return _explore_layers(
        root_kb, state,
        max_set_size=max_set_size,
        store_lattice=store_lattice,
        dumper=dumper,
        max_time=max_time, max_enterings=max_enterings,
    )
```

### `_LatticeLoopState.dispatch_dead` for contradictions

```python
def dispatch_dead(self, c, result, layer) -> None:
    if self.entry == "contradictions":
        self.dead_commitments.append(DeadCommitment(
            commitment=c,
            unsat_core=result.unsat_core,
            learned_clause=frozenset(c),
            layer=layer,
            kind=result.kind,
        ))
    # gaps: discard (root._nogoods clause is sufficient
    # for downstream pruning).
```

### Verdict synthesis (Phase 3)

```python
if state.entry == "contradictions":
    cores = frozenset().union(
        *(d.unsat_core for d in state.dead_commitments),
    )
    proof = LatticeProof(
        solutions       = (),                    # not tracked
        dead_commitments= tuple(state.dead_commitments),
        kb_index        = state.kb_index if store_lattice else {},
        alive_at_end    = (),                    # not tracked
        learned_nogoods = frozenset(root_kb._nogoods),
        stats           = state.stats,
    )
    return Contradiction(unsat_core=cores, proof=proof, tree=None), state.stats
```

## Edge cases

### Root-level contradiction in Phase 1

If `ContradictionDetector(root_kb).detect()` fires in
Phase 1 under contradictions_solve, return
`Contradiction(unsat_core=detected_witnesses,
proof=LatticeProof.empty())` immediately. There were no
commitments to record; `proof.dead_commitments == ()`.

### Empty dead_commitments + non-empty alive

contradictions_solve on a fully solvable puzzle (no
deaths observed within depth cap):
- `state.dead_commitments == []` → `unsat_core ==
  frozenset()`.
- Verdict is still `Contradiction` per mode contract.
- Caller knows from `len(proof.dead_commitments) == 0`
  that the puzzle isn't contradictory — this is the
  degenerate case the contract acknowledges.

### Goal-satisfying commitments under contradictions_solve

When `is_solved(result.kb)` fires (§3c.ii of
`algorithm_layer_n.md`), `state.dispatch_solution` is a
no-op for the contradictions entry. The cascade still
continues (flat root-merge of unconditional facts can
still trigger downstream effects); we just don't record
the satisfying commitment.

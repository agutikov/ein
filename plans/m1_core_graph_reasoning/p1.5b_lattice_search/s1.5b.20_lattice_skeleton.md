### `bench_lattice.py` (NEW)

```python
#!/usr/bin/env python3
"""Run the unified set-search engine's lattice entries.

--gaps           → gaps_solve
--contradictions → contradictions_solve
--store-lattice  → orthogonal storage flag
"""
import argparse, sys, time
from pathlib import Path

from ein.inference.config import SolverConfig
from ein.inference.monotonic import (
    LatticeDumper, contradictions_solve, gaps_solve,
)
from ein.ir import parse
from ein.kb.store import KnowledgeBase


def main(argv=None):
    ap = argparse.ArgumentParser(...)
    entry = ap.add_mutually_exclusive_group(required=True)
    entry.add_argument("--gaps", action="store_true")
    entry.add_argument("--contradictions", action="store_true")
    ap.add_argument("--store-lattice", action="store_true")
    ap.add_argument("--max-set-size", type=int, default=5)
    ap.add_argument("--dump-states", type=Path, default=None)
    ap.add_argument("--max-time", type=float, default=None)
    ap.add_argument("--max-enterings", type=int, default=None)
    ap.add_argument("puzzle", type=Path)
    args = ap.parse_args(argv)

    kb = KnowledgeBase.from_ir(parse(args.puzzle.read_text()))
    entry_fn = gaps_solve if args.gaps else contradictions_solve
    dumper = (
        LatticeDumper(out_dir=args.dump_states)
        if args.dump_states else None
    )
    try:
        verdict, _stats = entry_fn(
            kb,
            max_set_size=args.max_set_size,
            config=kb.config or SolverConfig(),
            store_lattice=args.store_lattice,
            dumper=dumper,
            max_time=args.max_time,
            max_enterings=args.max_enterings,
        )
    except NotImplementedError as e:
        print(f"** error: {e} **", file=sys.stderr)
        return 2
    # ... print verdict + stats; full output shape lands in S1.5b.21+
```

## Ship notes 2026-05-28

### Course-correction round

First implementation pass landed the stubs in a separate
`inference/lattice/` subfolder under a "two engines" framing.
User direction immediately after:
> *"Why you create new engine in
> ein.py/src/ein/inference/lattice while we decided to
> make unified monotonic-lattice engine?"*

The mistake was a misreading of the earlier
"separate-entries" direction — separate functions, not
separate folders. Course-corrected the same day: deleted
the four wrong files (`inference/lattice/{solver, lattice,
state_dump}.py` + reverted `__init__.py` to 0 bytes), added
the correct files under `inference/monotonic/` per the spec
above, updated imports in `bench_lattice.py` and the test
file. Final state matches this stage's acceptance.

The plan docs (this file + the README + algorithm_layer_n.md
+ stages 21/22/23/29) were rewritten the same day to remove
the "two engines" framing across the board.

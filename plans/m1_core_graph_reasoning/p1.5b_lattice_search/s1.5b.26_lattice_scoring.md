## Goal

Implement the `lattice_order` config option from
[Q1.5b.2.c](open_questions.md#q15b2--frontier-strategy):

- `"lex"` — canonical-tuple order (deterministic,
  uninformed). Useful for regression tests + invariance
  checks.
- `"score-sum"` — per-set score = sum of
  `score_hypothesis(h, kb)` for each element. Default.
  Reuses S1.5a.7's existing per-element scorer.

The frontier `order_candidates` from
`inference/lattice/frontier.py` (currently a lex-only stub from
S1.5b.21) fills out the score-sum branch.

Partial-layer / best-first / cross-layer scoring is
**parked** as the "Phase within the Phase" follow-up
(reserved slots S1.5b.11–.19 or a future P1.5c).

## `frontier.order_candidates` filled

```python
from ein_bot.inference.hypgen import score_hypothesis
from ein_bot.kb.entities import Fact
from ein_bot.kb.store import KnowledgeBase

def order_candidates(
    candidates: list[CanonicalSetId],
    *,
    mode: str = "score-sum",
    kb: KnowledgeBase | None = None,
) -> list[CanonicalSetId]:
    """Within-layer ordering — Q1.5b.2.c.

    "lex"       — canonical-tuple order, deterministic.
    "score-sum" — sum of per-element score_hypothesis values,
                  descending (higher score = explored first).
    """
    if mode == "lex":
        return sorted(candidates)

    if mode == "score-sum":
        if kb is None:
            raise ValueError(
                "score-sum mode requires kb (for "
                "score_hypothesis); pass kb=root_kb",
            )
        def _set_score(C: CanonicalSetId) -> float:
            return sum(
                score_hypothesis(_fact_from_id(kb, fid), kb)
                for fid in C
            )
        # Sort by -score so high-score sets come first;
        # tiebreak by canonical-tuple for determinism.
        return sorted(
            candidates,
            key=lambda C: (-_set_score(C), C),
        )

    raise ValueError(f"unknown lattice_order mode: {mode!r}")

def _fact_from_id(
    kb: KnowledgeBase, fid: tuple[str, tuple],
) -> Fact:
    """Reconstruct a Fact from a (relation_name, args)
    tuple for scoring purposes.

    score_hypothesis takes a Fact, not a FactId — wrap.
    """
    rn, args = fid
    existing = kb._fact_by_id(rn, args)
    if existing is not None:
        return existing
    # Synthetic — the candidate isn't yet in kb (we're
    # scoring the prospective entering).
    return Fact(relation_name=rn, args=args, layer=Layer.REASONING)
```

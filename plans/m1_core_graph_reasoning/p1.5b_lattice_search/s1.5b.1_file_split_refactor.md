# S1.5b.1 — File-split refactor (`inference/{tree/,monotonic/,lattice/}`)

**Phase:** P1.5b (set-indexed search engines).
**Estimate:** S (½–1 day) — mechanical refactor; the risk is
import-path churn, not logic.
**Status:** **shipped 2026-05-27** (`22a4ebd`) — first implementation stage.
697 tests pass (1 skipped: zebra2 d=1 gated on `EIN_RUN_SLOW=1`);
`bench_solve examples/branching/03_*.ein` → identical `Solution`
(h=H5, 2 tree nodes). See [Ship notes](#ship-notes-2026-05-27)
below for triage decisions on modules not in the original split

**Blocks:** every engine-specific stage (S1.5b.2+ for shared
infra, S1.5b.4+ for monotonic, S1.5b.20+ for lattice).

## Ship notes 2026-05-27

| Module | Decision | Reasoning |
|---|---|---|

**Layer violation flagged** (kept; designed for cleanup in
P1.5b): `inference/hypgen.py` and `inference/nogoods.py` both
import upward into `inference.tree.back_prop`
(`back_propagate` from hypgen; `BubbleAbort`,

original design said back_prop is tree-specific "until the
S1.5a.20 channel-isolation rewrite collapses this" — that
rewrite is now native to P1.5b's per-set `integrate`, so the
violation closes when monotonic's outer driver lands
(S1.5b.5+). Until then, the upward import is the mechanical
minimum-change route.

## Goal

Reorganise `ein.py/src/ein_bot/inference/` so the three search
engines have their own namespaces, without changing any
semantic behaviour. The common kernel (modules every engine
imports) stays in `inference/` directly; the existing
tree-search code moves into `inference/tree/`; empty
placeholder directories with `__init__.py` are created for
`inference/monotonic/` and `inference/lattice/`.

Same restructure mirrored under `ein.py/tests/inference/`.

The 670-test suite must stay green after the move; no
docstring or behaviour changes beyond import-path updates.

## What goes where

Apply the **copy-on-modify** rule from
[Q1.5b.1.a](open_questions.md#q15b1--co-existence-layout):
files that need engine-specific divergence today move to the
engine folder; files that don't, stay in common. As DAG /
monotonic implementation progresses (S1.5b.2+), files that
need to diverge get copied at that point.

### Initial split — known-divergent today

Move to `inference/tree/`:

- `solver.py` — `_explore` / `_descend` / `_consume` /
  `_TreeBuilder` / `solve()`. Tree-search orchestrator;
  every other engine has its own.
- `back_prop.py` — ancestor-chain back-prop semantics with
  `_kb_chain_ctx`. The S1.5a.20 channel-isolation rewrite
  will collapse this; until that lands, it's tree-specific.

Stay in `inference/` (common):

- `saturator.py` — pure forward chaining.
- `contradiction.py` — `(X, ¬X)` detection.
- `lookahead.py` — one-step death simulation, kb-only.
- `hypgen.py` — candidate generation, kb-only.
- `firing.py` — derivation records.
- `match.py`, `compile.py`, `engine.py` — rule matching kernel.
- `closed.py`, `config.py` — config + closed-relation emit.
- `nogoods.py` — unified CDCL store (Q1.5b.5).
- `state_dump.py` — base file-system mirror utilities (per
  S1.5a.11 + T1.5a.19.3 enhancements).

### New empty folders

- `inference/monotonic/__init__.py` — empty, just makes it a
  package.
- `inference/lattice/__init__.py` — empty.
- `tests/inference/monotonic/__init__.py` — empty.
- `tests/inference/lattice/__init__.py` — empty.
- `tests/inference/tree/__init__.py` — empty.

### Tests mirror

Tree-specific tests under `tests/inference/` that exercise
`solver.py` or `back_prop.py` move to `tests/inference/tree/`.
Common-kernel tests stay at `tests/inference/`. The
demarcation criterion: a test imports from `ein_bot.inference.solver`
or `ein_bot.inference.back_prop` → it's tree-specific.

## Tasks

### T1.5b.1.1 — Inventory divergent modules

Confirm the split list above by grepping the current
`inference/` directory:

```sh
ls ein.py/src/ein_bot/inference/
# expected: saturator.py, contradiction.py, lookahead.py,
# hypgen.py, firing.py, match.py, compile.py, engine.py,
# closed.py, config.py, nogoods.py, state_dump.py,
# solver.py, back_prop.py + __init__.py + __pycache__
```

If a module surfaces that wasn't in the design discussion
(e.g., something added since 2026-05-25), decide per-file:
tree-specific or common.

### T1.5b.1.2 — Move tree-specific code

```sh
mkdir -p ein.py/src/ein_bot/inference/tree
git mv ein.py/src/ein_bot/inference/solver.py \
       ein.py/src/ein_bot/inference/tree/solver.py
git mv ein.py/src/ein_bot/inference/back_prop.py \
       ein.py/src/ein_bot/inference/tree/back_prop.py
touch ein.py/src/ein_bot/inference/tree/__init__.py
```

### T1.5b.1.3 — Create monotonic / lattice stub packages

```sh
mkdir -p ein.py/src/ein_bot/inference/monotonic
mkdir -p ein.py/src/ein_bot/inference/lattice
touch ein.py/src/ein_bot/inference/monotonic/__init__.py
touch ein.py/src/ein_bot/inference/lattice/__init__.py
```

The `__init__.py` files are empty. Stub modules + bench
scripts land in S1.5b.4 (monotonic) and S1.5b.20 (lattice).

### T1.5b.1.4 — Update imports

Every source file that imports from `ein_bot.inference.solver`
or `ein_bot.inference.back_prop` needs the path updated:

```python
# before
from ein_bot.inference.solver import solve, Solution, ...
from ein_bot.inference.back_prop import back_propagate, ...

# after
from ein_bot.inference.tree.solver import solve, Solution, ...
from ein_bot.inference.tree.back_prop import back_propagate, ...
```

Grep + replace; no other changes. Sites to update:

- `ein.py/demo/bench_solve.py`
- `ein.py/demo/bench_saturate.py` (if it imports solver)
- `ein.py/src/ein_bot/inference/state_dump.py` — if it
  imports back-prop internals (check)
- `ein.py/src/ein_bot/cli.py` (if any)

Avoid creating compat shims like
`inference/solver.py = re-export from tree/`. Updating
callsites is cleaner and forces the explicit-engine-selection
discipline.

### T1.5b.1.5 — Split tests

Tests that import `ein_bot.inference.solver` or
`ein_bot.inference.back_prop` (or test their behaviour
directly) move to `tests/inference/tree/`:

```sh
mkdir -p ein.py/tests/inference/tree
touch ein.py/tests/inference/tree/__init__.py

# move each tree-specific test file
git mv ein.py/tests/inference/test_solver.py \
       ein.py/tests/inference/tree/test_solver.py
# … repeat for every solver/back-prop test
```

Tests that exercise saturator / contradiction / lookahead /
hypgen / etc. only — stay at `tests/inference/`. Update
imports in moved files mirroring T1.5b.1.4.

### T1.5b.1.6 — Verify regression

```sh
python -m pytest ein.py/tests/ -x
# expected: 670 passed
```

Also run the benches manually as a smoke check:

```sh
python ein.py/demo/bench_solve.py examples/branching/03_five_hyps_one_alive.ein
# expected: same verdict + same bindings as before the refactor
```

If any test fails, the failure is an import-path miss or a
module that should have been split differently. Fix
mechanically — no semantic change is permitted in this
stage.

## Acceptance

- `ein.py/src/ein_bot/inference/` has only common modules at
  the top level + the three subfolders `tree/`,
  `monotonic/`, `lattice/`.
- `ein.py/src/ein_bot/inference/tree/` carries `solver.py` +
  `back_prop.py` + `__init__.py`.
- `ein.py/src/ein_bot/inference/monotonic/` and
  `ein.py/src/ein_bot/inference/lattice/` carry only
  `__init__.py` (empty).
- `ein.py/tests/inference/` mirrors the split:
  `tree/test_solver.py` etc. lives under `tree/`; common
  tests stay flat.
- Full 670-test suite green.
- `bench_solve.py examples/branching/03_*.ein` produces an
  identical verdict to the pre-refactor baseline (no behaviour
  change).
- One commit per task, or one squash if the refactor is
  obviously atomic; either way the final commit log is clean
  enough to revert if needed.

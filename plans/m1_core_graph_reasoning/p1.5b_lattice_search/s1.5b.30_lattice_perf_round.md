Extract:

|--------------------------|---------------------------------|-----------|---------|

  monotonic is faster overall only because of early

## Deferred optimisations — evaluate

### Subset-trie lookup for Apriori-gen (Q1.5b.2.b)

The naive prefix-join's `O(|A_{k-1}|² / |prefix-classes|)`
is bounded by `|A_{k-1}|^2`. For zebra2 with `|A_{k-1}|` up
to ~50 at layer 2, that's 2500 join attempts — fine. Above
~200, the trie becomes worthwhile.

**Decision rule:** land only if zebra2 layer-3 enumeration
> 1 s of wall time. Otherwise defer to followup.

### Interned set ids (Q1.5b.2.b + Q1.5b.3)

Replace `CanonicalSetId = tuple[FactId, ...]` with an
intern dict (`int ↔ tuple`). Saves memory (one int vs a
tuple of length k per node) + faster lookup.

**Decision rule:** land only if the lattice's memory peak
on zebra2 > 500 MB. Otherwise defer.

### Commitment-set `try_commitment_set` divergence (Q1.5b.8)

Lattice's `try_commitment_set` currently lives in common
`inference/commitment.py`. If perf measurement shows the
common form is suboptimal for the lattice's per-set
workload (e.g., the lattice could skip the fork-and-copy
when state-hash dedup is about to fire), copy `try_commitment_set` into
`inference/lattice/commitment.py` and let it diverge.

**Decision rule:** land only if `try_commitment_set` dominates the
lattice's runtime (> 50% via cProfile). Otherwise common
stays common.

## Tree-side deprecation tagging

Per the phase README's acceptance #9: tag the tree side as
deprecated.

In `ein.py/src/ein_bot/inference/tree/__init__.py`:

```python
"""Tree-based hypothesis search engine (P1.5 + P1.5a).

DEPRECATED 2026-XX-XX — superseded by:
  - inference/monotonic/ for SOLVE-mode (first solution)
  - inference/lattice/ for full lattice search (all solutions
    + LatticeProof for P1.6).

Removal queued for: post-P1.6/P1.7 green-on-{monotonic,lattice}
acceptance.

The tree engine remains as a reference baseline +
regression harness; the parity tests in
tests/inference/{monotonic,lattice}/ assert verdict equality
against this engine on the branching demos.
"""
import warnings
warnings.warn(
    "ein_bot.inference.tree is deprecated; use "
    "ein_bot.inference.monotonic (SOLVE) or "
    "ein_bot.inference.lattice (full lattice).",
    DeprecationWarning, stacklevel=2,
)
```

Don't break callers — the warning is at import time,
non-fatal. P1.6 + P1.7 will migrate at their own pace.

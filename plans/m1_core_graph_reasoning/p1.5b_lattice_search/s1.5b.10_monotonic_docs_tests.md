### Coverage gap (88%, target was > 90%)

`solver.py` coverage at 85% (~25 uncovered statements). The
gaps are post-S1.5b.9 "root-side fallback" paths the engine
takes only when the fork doesn't satisfy `is_solved` but the
post-merge root does — a narrow semantic window:

- The in-loop unconditional-facts merge + re-saturate +
  root-is_solved chain (lines ~305–343) used to fire for
  zebra2; post-S1.5b.9, fork-side terminates first.
- The forced-positive cascade's internal contradiction +
  is_solved checks fire only when the cascade is the *only*
  termination path — covered by tests that don't directly hit
  this branch.

Both paths remain semantically correct; the engine takes them
when the fixture's structure routes the verdict through root.
The simplest fixture I could synthesise to force the route
(goal satisfied at root but not at fork) is non-trivial — root
is a strict subset of the fork's facts in the standard model,
so "root satisfies + fork doesn't" requires a configuration
where lookahead's `(not h)` writes unlock derivations that the
fork doesn't replicate. Punted as a S1.5b.11+ followup if perf
or audit work needs it.

## Ship notes 2026-05-27

### Signature deviation — `(Verdict, MonotonicStats)` tuple

Spec returned just `Verdict`. The implementation returns
`(Verdict, MonotonicStats)` so the bench script and tests
get direct access to per-run counters without a
side-channel on `root_kb`. User-chosen option after explicit
clarification.

### Spec enhancements (carry into S1.5b.6+)

**Phase 1 ContradictionDetector check.** Spec's Phase 1
only checked `is_solved`; if root saturates to `(false)`
or a `(X, ¬X)` REASONING pair, the backbone would fall
through to Phase 2 where every entering dead-pre's and
the final verdict mis-reports as Ambiguity. Added
`ContradictionDetector(root_kb).detect()` after the
initial saturation; returns `Contradiction` if found.

**Post-merge re-detect.** Same pattern after Phase 2's
re-saturation of root: merged unconditional facts can
derive a root contradiction; checked before re-computing
alive.

### Spec-pseudocode bug — saturator generator must be consumed

`Saturator.saturate(...)` returns `Iterator[Firing]`. The
spec wrote `Saturator(root_kb).saturate()` (and likewise
in `try_commitment_set`'s shipped code, but there it was
caller-driven via `tuple(...)`). The bench result of
`Solution` on `test_trivial_root_only_solve` initially
returned `Ambiguity` because the generator was discarded
unconsumed — Saturator did nothing. Wrap in
`list(...)` (or any consumer) to drive the iteration.
S1.5b.5 uses `_ = list(Saturator(root_kb).saturate())` at
all 3 call sites.

Spec defaulted `max_steps=10_000` at every call site;
shipped code uses `Saturator.saturate`'s native default
(`None`, no cap) per the same argument as S1.5b.3 — M1
ruleset is monotone so saturation terminates; the 10K cap
was an arbitrary safety net.

### Branching/03 acceptance gap — forced-positive promotion not yet

Spec's acceptance criterion:
> `bench_monotonic.py examples/branching/03_five_hyps_one_alive.ein`
> returns `Solution` with the expected bindings.

Shipped behavior: **`Ambiguity`**, not `Solution`. The
gap is structural to the minimal backbone:

1. `try_commitment_set`'s unconditional-fact extraction
   marks any fact whose chain reaches a committed
   hypothesis as conditional.
2. When alive shrinks to a singleton `{h_unique}` (as
   happens at branching/03 after layer 1 — only
   `(co-located White H5)` survives), `h_unique` IS in
   the commitment, so the goal-relevant facts derived
   from it (e.g., `(co-located White H5)` itself) are
   conditional and never merge into root.
3. Tree-side returns `Solution` because its
   `Solution.kb` is the *surviving fork's* kb, which
   includes the committed hypothesis. Monotonic returns
   the root.

**Resolution path**: forced-positive promotion (when
alive shrinks to size 1, merge that hypothesis into root
and re-saturate). The original phase plan named this
S1.5b.25 for the lattice; the monotonic side needs a
twin in the S1.5b.6–.10 range. Tracked as a follow-up
note in this stage's "Limitations".

The shipped test
[`test_branching_03_returns_ambiguity_pending_forced_positives`](../../../ein.py/tests/inference/monotonic/test_monotonic_skeleton.py)
documents this — when forced-positive promotion ships,
the assertion flips from `Ambiguity` to `Solution`.

`monotonic_solve` implementation. After this stage:

  base enough, else Ambiguity at the depth cap).
- The loop's structure matches
  [`algorithm_layer_n.md` § 3a–3d.vii](algorithm_layer_n.md)'s
  monotonic-degenerate subset.

What this stage explicitly does NOT do:
- **No CDCL nogoods yet** — `root._nogoods` exists (from the
  common kb store) but the backbone does not consult it for
  filtering. S1.5b.6 wires that in.
- **No dumper.** `--dump-states` is silently ignored. S1.5b.7
  fills the dumper.
- **No state-hash dedup** — monotonic doesn't have it by
  design.

## Algorithm

```
monotonic_solve(root_kb, max_set_size, config):
    # ── Phase 1 — Initial saturation + alive ──────────────
    Saturator(root_kb).saturate()
    if is_solved(root_kb):
        return Solution(root_kb)

    alive = generate_alive(root_kb)  # via existing hypgen
    if not alive:
        return Contradiction(unsat_core=…)

    A_prev = layer_1(alive)  # [(h,) for h ∈ alive]
    explored_layers: list[list[CanonicalSetId]] = [A_prev]

    # ── Phase 2 — Layer-by-layer iteration ───────────────
    for layer in range(1, max_set_size + 1):
        if layer == 1:
            candidates = A_prev
        else:
            candidates = generate_layer(
                A_prev,
                alive=alive,
                nogoods=frozenset(),  # S1.5b.6 wires real nogoods
            )

        # Within-layer ordering — Q1.5b.2.c default "score-sum".
        # S1.5b.5 ships lex (deterministic); S1.5b.10's docs
        # can flip the default once scoring's wired.
        candidates.sort()  # lex for now

        A_layer: list[CanonicalSetId] = []
        for C in candidates:
            result = try_commitment_set(root_kb, C)

            match result.kind:
                case "dead-pre" | "dead-post":
                    # No nogood emit in the backbone — S1.5b.6.
                    continue
                case "alive":
                    # Merge unconditional facts into root.
                    root_changed = False
                    for f in result.unconditional_facts:
                        if root_kb._fact_by_id(f.relation_name, f.args) is None:
                            stored = root_kb.add_fact(f)
                            root_kb._index_fact(stored)
                            root_changed = True

                    # Re-saturate root + recompute alive if
                    # the merge added anything (Option A
                    # cadence — Q1.5b.2.a).
                    if root_changed:
                        Saturator(root_kb).saturate()
                        alive = generate_alive(root_kb)

                        # Early termination check.
                        if is_solved(root_kb):
                            return Solution(root_kb)

                    A_layer.append(C)

        if not A_layer:
            # Layer empty → nothing to extend → terminate.
            break
        A_prev = A_layer
        explored_layers.append(A_layer)

    # ── Phase 3 — Verdict synthesis ──────────────────────
    if is_solved(root_kb):
        return Solution(root_kb)
    if not alive or layer_exhausted_at_max:
        # Couldn't decide — alive set non-empty but
        # `max_set_size` hit, OR alive empty and goal still
        # not satisfied.
        if not alive:
            return Contradiction(unsat_core=…)
        return Ambiguity(unresolved=A_prev)
    return Ambiguity(unresolved=A_prev)
```

## Module surface

`inference/monotonic/solver.py`:

```python
from __future__ import annotations

from dataclasses import dataclass
from typing import Literal

from ein_bot.inference.apriori import (
    CanonicalSetId, FactId,
    generate_layer, layer_1,
)
from ein_bot.inference.config import SolverConfig
from ein_bot.inference.hypgen import generate_hypotheses
from ein_bot.inference.saturator import Saturator
from ein_bot.inference.commitment import try_commitment_set
from ein_bot.kb.entities import Fact
from ein_bot.kb.store import KnowledgeBase

# ── Verdicts ─────────────────────────────────────────────
# Reuse tree-side verdict shapes to keep downstream (P1.6)
# happy. Importing from `tree.solver` is OK — verdict types
# are pure data, not search machinery.
from ein_bot.inference.tree.solver import (
    Solution, Ambiguity, Contradiction, Verdict, Mode,
)

@dataclass
class MonotonicStats:
    """Cumulative counters for one monotonic_solve run."""
    enterings_total:     int = 0
    enterings_alive:     int = 0
    enterings_dead_pre:  int = 0
    enterings_dead_post: int = 0
    facts_merged:        int = 0
    saturate_count:      int = 0
    layers_explored:     int = 0

def monotonic_solve(
    root_kb: KnowledgeBase,
    *,
    max_set_size: int = 5,
    config: SolverConfig | None = None,
    mode: Mode = Mode.SOLVE,
) -> Verdict:
    """The monotonic engine — see [s1.5b.5 algorithm].

    SOLVE mode only (Q1.5b.7). Other modes either map to
    SOLVE (no-op difference under monotonic) or raise.
    """
    if mode is not Mode.SOLVE:
        raise NotImplementedError(
            "monotonic engine supports SOLVE mode only "
            "(Q1.5b.7); use the lattice engine for GAPS / "
            "CONTRADICTIONS",
        )

    cfg = config or root_kb.config or SolverConfig()
    root_kb.config = cfg

    stats = MonotonicStats()

    # ── Phase 1 — Initial saturation + alive ──────────────
    Saturator(root_kb).saturate(max_steps=10_000)
    stats.saturate_count += 1
    if _is_solved(root_kb, mode):
        return _solution(root_kb)

    alive = _compute_alive(root_kb)
    if not alive:
        return _contradiction(root_kb)

    # Layer 1 candidates.
    A_prev: list[CanonicalSetId] = layer_1(alive)

    # ── Phase 2 — Layer-by-layer iteration ───────────────
    layer_exhausted_at_max = False
    for layer in range(1, max_set_size + 1):
        stats.layers_explored = layer

        if layer == 1:
            candidates = list(A_prev)
        else:
            candidates = generate_layer(
                A_prev,
                alive=alive,
                nogoods=frozenset(),  # S1.5b.6 will pass root_kb._nogoods
            )

        candidates.sort()  # lex; scoring switch lands in monotonic followups

        A_layer: list[CanonicalSetId] = []
        for C in candidates:
            stats.enterings_total += 1
            result = try_commitment_set(root_kb, C)

            if result.kind in ("dead-pre", "dead-post"):
                if result.kind == "dead-pre":
                    stats.enterings_dead_pre += 1
                else:
                    stats.enterings_dead_post += 1
                continue

            # Alive.
            stats.enterings_alive += 1
            root_changed = False
            for f in result.unconditional_facts:
                if root_kb._fact_by_id(f.relation_name, f.args) is None:
                    stored = root_kb.add_fact(f)
                    root_kb._index_fact(stored)
                    stats.facts_merged += 1
                    root_changed = True

            if root_changed:
                Saturator(root_kb).saturate(max_steps=10_000)
                stats.saturate_count += 1
                alive = _compute_alive(root_kb)

                if _is_solved(root_kb, mode):
                    return _solution(root_kb)

                # Refresh candidate filter — the remaining
                # in-layer candidates might now contain
                # elements no longer alive.
                # (For the backbone we just re-filter; later
                # stages can avoid mutating the in-flight
                # list.)

            A_layer.append(C)

        if not A_layer:
            break
        A_prev = A_layer
        if layer == max_set_size:
            layer_exhausted_at_max = True

    # ── Phase 3 — Verdict synthesis ──────────────────────
    if _is_solved(root_kb, mode):
        return _solution(root_kb)
    if not alive:
        return _contradiction(root_kb)
    return _ambiguity(root_kb, A_prev)

# ── Helpers ─────────────────────────────────────────────

def _compute_alive(kb: KnowledgeBase) -> frozenset[FactId]:
    """Build the current alive set as a frozenset of FactIds.

    Reuses the existing `generate_hypotheses` from common
    `hypgen.py` — that's the canonical "which hypotheses are
    still viable" calculation.
    """
    facts = list(generate_hypotheses(kb))
    return frozenset(
        (f.relation_name, f.args) for f in facts
    )

def _is_solved(kb: KnowledgeBase, mode: Mode) -> bool:
    """Wraps the existing `is_solved` from `tree.solver` —
    same goal-check semantics, no search-shape coupling."""
    from ein_bot.inference.tree.solver import is_solved
    return is_solved(kb, mode)

def _solution(kb: KnowledgeBase) -> Verdict:
    return Solution(kb=kb, trace=(), tree=None)

def _contradiction(kb: KnowledgeBase) -> Verdict:
    return Contradiction(unsat_core=frozenset(), tree=None)

def _ambiguity(
    kb: KnowledgeBase,
    unresolved: list[CanonicalSetId],
) -> Verdict:
    # `unresolved` here is the list of alive size-`max_set_size`
    # commitments — the depth-cap frontier. Tree-side's
    # Ambiguity carries SearchNode `unresolved`; we adapt
    # by passing an empty tuple. The lattice's LatticeProof
    # (S1.5b.29) will carry the richer artefact.
    return Ambiguity(
        branches=(Solution(kb=kb, trace=(), tree=None),),
        unresolved=(),
        tree=None,
    )
```

## Limitations (deliberate)

The backbone deliberately skips features that land in
follow-up stages:

| feature                                   | lands in     |
|-------------------------------------------|--------------|
| Nogood emit + filter                      | S1.5b.6      |
| Dumper (per-layer root snapshots)         | S1.5b.7      |
| Acceptance measurement on Zebra           | S1.5b.8      |
| `examples/branching/*` full parity        | S1.5b.9      |
| Docs + canonical tests                    | S1.5b.10     |

The backbone's `Ambiguity` verdict is **partial** —
S1.5b.5's return doesn't expose the dead frontier or the
nogood set because those land in S1.5b.6. P1.6 consumers
should wait for S1.5b.7+ before reading monotonic dumps.

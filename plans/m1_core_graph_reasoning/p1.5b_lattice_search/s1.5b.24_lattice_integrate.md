# S1.5b.24 — F3: multi-parent integrate (per-parent bubble) — **SUPERSEDED 2026-05-28**

> ⛔ **DROPPED.** This stage was killed when monotonic + lattice
> collapsed into a single mode-driven engine. The "multi-parent
> integrate" mechanic was a tree-search reflex; under monotone
> saturation it reduces to **idempotent flat writes** into
> `root._nogoods` (for dead commitments) and `root.facts` (for
> unconditional consequences of alive commitments). The
> unified `monotonic_solve` already performs both — see
> [`monotonic/solver.py:243-353`](../../../ein.py/src/ein/inference/monotonic/solver.py).
>
> **Rationale.** See [`project-set-search-unified` memory] +
> [README § What this phase delivers](README.md#what-this-phase-delivers) +
> [`algorithm_layer_n.md § What this algorithm no longer does`](algorithm_layer_n.md#what-this-algorithm-no-longer-does) +
> the 2026-05-28 conversation. The relevant claims:
> 1. `emit_nogood(P_i, frozenset(C))` ∀ `P_i ∈ ancestors(C)` is
>    N-way idempotent — the clause is the same in every parent
>    and subsumption handles redundant writes. Writing once to
>    `root._nogoods` is equivalent.
> 2. `is_unconditional_at(P_i)` ⇔ `is_unconditional_at(C)`
>    given a correct transitive `_reaches_commitment` walk
>    (the walk terminates at non-rule provenances; if no
>    `h ∈ C` is reached, no `h ∈ P_i ⊆ C` can be reached
>    either). The per-parent check is defensive code with
>    no semantic effect.

> root-write step in the monotonic loop (s1.5b.5 backbone)
> already does the work this stage was scheduled to add.
> Downstream stages that depended on this (S1.5b.25 forced
> positives) are also dropped or fold into the existing
> backbone.
>
> **Do not implement.** Removed from the active stage
> sequence; this file is kept as a historical record so
> cross-references in `s1.5b.20–.23` + README + `open_questions.md`
> still resolve.

## Goal

Replace the backbone's "merge unconditional facts straight
to root" with the **multi-parent integrate** from
[S1.5a.20 § integrate](../p1.5a_zebra_solution/s1.5a.20_branch_isolation_rearch.md#integrate-the-up-channel).

For each alive `SetNode` C of size k:

1. C has k parents P₁…Pₖ in the lattice (the (k-1)-subsets
   of C).
2. For each parent Pᵢ:
   - Call `integrate(Pᵢ.kb_snapshot, child_result)`.
   - Each integrate step decides per-level which proposed
     facts to adopt at Pᵢ's level (per-level
     `is_unconditional_at`).
   - Adopted facts mutate Pᵢ.kb_snapshot + re-saturate Pᵢ.
3. The bubble continues recursively — Pᵢ's own integrate
   may propose negatives/positives that bubble up through
   Pᵢ's parents.
4. **Idempotent under multi-parent fan-out** — the same
   negative reaching Pᵢ from two distinct children produces
   one write (the second is a no-op).

## The integrate step

```python
def integrate(
    parent_node: SetNode,
    child_result: CommitmentSetResult,
) -> tuple[Fact, ...]:
    """Integrate child_result's bubble payload into the
    parent SetNode's kb. Returns the facts adopted at
    parent's level (for the caller to bubble further if
    parent's parents need them).

    Steps (mirrors S1.5a.20's contract):
      1. Adopt unconditional-dead negative.
      2. Adopt proposed_negatives unconditional at parent.
      3. Adopt forced_positives unconditional at parent.
      4. Merge learned_nogoods into parent._nogoods.
      5. Re-saturate parent.kb after the batch.
      6. ContradictionDetector(parent.kb).detect() —
         if hit, parent itself is dead.
      7. Re-prune parent.alive against new _negated_facts.
      8. Return the adopted facts.
    """
    parent_kb = parent_node.kb_snapshot
    adopted: list[Fact] = []

    # 1. unconditional-dead `(not h)` write
    if child_result.kind in ("dead-pre", "dead-post"):
        # Note: this only fires if the parent's own commitment
        # doesn't include the child's hypothesis. For lattice
        # multi-parent, every parent P is C \ {h_i} for some
        # h_i; the missing element IS what the parent doesn't
        # commit. So writing (not h_i) is sound at this parent's
        # level iff the death's unsat_core is unconditional
        # against parent's own commitment.
        child_h_id = _missing_element(parent_node, child_result)
        if _is_unconditional_at(
            child_result.kb, child_result.unsat_core,
            own_commitment=parent_node.canonical_set,
        ):
            not_fact = _make_not(child_h_id)
            if parent_kb._fact_by_id("not", (not_fact.args[0],)) is None:
                stored = parent_kb.add_fact(not_fact)
                parent_kb._index_fact(stored)
                adopted.append(stored)

    # 2. proposed_negatives — per-level unconditional check
    for f in child_result.proposed_negatives if hasattr(child_result, "proposed_negatives") else ():
        if _is_unconditional_at(child_result.kb, frozenset({f}),
                                own_commitment=parent_node.canonical_set):
            if parent_kb._fact_by_id(f.relation_name, f.args) is None:
                stored = parent_kb.add_fact(f)
                parent_kb._index_fact(stored)
                adopted.append(stored)

    # 3. forced_positives
    for f in child_result.unconditional_facts:
        if _is_unconditional_at(child_result.kb, frozenset({f}),
                                own_commitment=parent_node.canonical_set):
            if parent_kb._fact_by_id(f.relation_name, f.args) is None:
                stored = parent_kb.add_fact(f)
                parent_kb._index_fact(stored)
                adopted.append(stored)

    # 4. learned_nogoods merge — idempotent (subsumption).
    for clause in (child_result.learned_nogoods if hasattr(child_result, "learned_nogoods") else ()):
        emit_nogood(parent_kb, clause)

    # 5. Re-saturate.
    if adopted:
        Saturator(parent_kb).saturate()

    # 6. Post-integrate contradiction check (Q2c from S1.5a.19).
    contras = ContradictionDetector(parent_kb).detect()
    if contras:
        parent_node.verdict = "dead"
        parent_node.unsat_core = frozenset(
            parent_kb.unsat_core(c.witness for c in contras),
        )
        # Bubble further: parent itself becomes a dead-child
        # for its own parents' integrate. The caller's
        # iteration handles this.

    # 7. Re-prune alive (Q2a from S1.5a.19).
    parent_node._alive_pruned += _prune_alive(parent_kb)

    return tuple(adopted)
```

`_is_unconditional_at(kb, facts, own_commitment)` is the
existing `reaches_hypothesis`-style walker from
S1.5b.3, with the "is-hypothesis" predicate set to the
own_commitment's element-set.

## Changes to `lattice_solve`

Replace the backbone's `root_changed = True; ... merge to
root` block with:

```python
# Multi-parent integrate — each parent receives the child's
# bubble.
for parent_set in node.parents:
    parent_node = lattice.get_by_set(parent_set)
    if parent_node is None:
        # Root's empty set — handle separately:
        parent_node = lattice.get_by_set(())
    adopted = integrate(parent_node, result)
    # Adopted facts may need to bubble to parent's parents.
    # Lattice's BFS-by-size guarantees parent's parents are
    # at layer (|parent_set| - 1) and already explored, so
    # recursion is bounded.
    if adopted:
        _bubble_upward(lattice, parent_node, adopted)

# Re-saturate root if anything reached it.
# Root has no parents to integrate to; the bubble
# terminates at root.
if root_node.kb_snapshot is not None:
    Saturator(root_node.kb_snapshot).saturate()
    alive = _compute_alive(root_node.kb_snapshot)
```

`_bubble_upward(lattice, node, adopted_facts)` walks up
through `node.parents`, repeating the integrate step on each
grandparent if any of `adopted_facts` is unconditional at
that grandparent's commitment level. Bounded recursion;
under BFS-by-size the depth equals `len(node.canonical_set)`.

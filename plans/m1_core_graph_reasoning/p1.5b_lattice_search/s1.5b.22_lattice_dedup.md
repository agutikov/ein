## Goal

Replace the backbone's naive `Lattice.add(node)` (which
allocates a fresh SetNode every entering) with the
state-hash-aware version from
[`algorithm_layer_n.md` § 3d.iii](algorithm_layer_n.md):

- **Set-equivalence dedup (primary).** If
  `lattice.get_by_set(C)` returns an existing node, skip
  the whole try_set + processing.
- **State-hash dedup (secondary).** After try_set produces
  a fork with state_hash H:
  - If `lattice.get_by_hash(H)` returns an existing node E:
    - Append C to E.labels (multilabel grow).
    - Append C's parents to E.parents (multi-parent union;
      dedup).
    - Skip the rest of the pipeline (verdict, fact merge,
      etc.) — already recorded on E.
  - Else: allocate a fresh SetNode with `labels=(C,)` and
    register it.

## Changes to `Lattice.add`

```python
def add(self, node: SetNode) -> None | SetNode:
    """Register a SetNode OR merge into an existing one if
    state-hash collides.

    Returns:
        None if a fresh SetNode was registered.
        The existing SetNode if dedup-merge happened.
    """
    # Primary dedup — same canonical_set already in store.
    existing_by_set = self.nodes_by_set.get(node.canonical_set)
    if existing_by_set is not None:
        # Defensive — a canonical-set re-arrival should have
        # been caught BEFORE try_set runs. This branch
        # should not fire in practice; log + return.
        return existing_by_set

    # Secondary dedup — state-hash collision.
    existing_by_hash = self.nodes_by_hash.get(node.state_hash)
    if existing_by_hash is not None:
        merged = _merge_labels(existing_by_hash, node)
        # Update the by-set index for the new label.
        self.nodes_by_set[node.canonical_set] = merged
        # by_hash already points to existing_by_hash; merged
        # is the same object after in-place edit.
        layer = len(node.canonical_set)
        while len(self.alive_by_layer) <= layer:
            self.alive_by_layer.append([])
            self.dead_by_layer.append([])
        # The merge doesn't add new alive/dead — it adds a
        # label-pointer. Track separately:
        # `merged_in_by_layer[layer].append(node.canonical_set)`.
        self._record_merge(layer, node.canonical_set, existing_by_hash.canonical_set)
        return merged

    # Fresh node.
    self.nodes_by_set[node.canonical_set] = node
    self.nodes_by_hash[node.state_hash] = node
    layer = len(node.canonical_set)
    while len(self.alive_by_layer) <= layer:
        self.alive_by_layer.append([])
        self.dead_by_layer.append([])
    if node.verdict in ("alive", "solution"):
        self.alive_by_layer[layer].append(node.canonical_set)
    else:
        self.dead_by_layer[layer].append(node.canonical_set)
    return None

def _merge_labels(existing: SetNode, new: SetNode) -> SetNode:
    """In-place: add new.canonical_set to existing.labels;
    union parents.
    """
    existing.labels = existing.labels + (new.canonical_set,)
    new_parents = set(existing.parents) | set(new.parents)
    existing.parents = tuple(sorted(new_parents))
    return existing
```

`Lattice` gains a `_record_merge` helper + a `merged_by_layer`
field for the audit trail.

## Changes to `lattice_solve`

Two additions:

### Pre-fork set-equivalence check

Before each try_set call, check the lattice for a node with
the same canonical_set. If found, that's a re-arrival
(shouldn't happen under Apriori-gen but is defensive):

```python
if lattice.get_by_set(C) is not None:
    # Already explored — Apriori shouldn't produce this,
    # but the check is cheap.
    continue
```

### Post-saturation state-hash check

After try_set returns, before constructing the SetNode,
look up `lattice.get_by_hash(state_hash(result.kb))`:

```python
result = try_set(root_kb, C)
existing = lattice.get_by_hash(state_hash(result.kb))

if existing is not None:
    # Cross-set state convergence. Add C as a label.
    lattice.add(SetNode(
        canonical_set=C,
        state_hash=existing.state_hash,
        # Don't carry a fresh kb_snapshot — share with existing.
        kb_snapshot=existing.kb_snapshot,
        firings=(),  # nothing new beyond existing's firings
        verdict=existing.verdict,
        labels=(C,),
        parents=tuple(_parent_sets(C, A_prev)),
        forced_positives=(),
        learned_nogoods=(),
    ))
    A_layer.append(C)  # still track for next layer's gen
    continue
```

The `lattice.add` call uses the merge-into-existing path
(state_hash matches existing.state_hash).

## Audit instrumentation

For S1.5b.23's dumper, expose the merge events. The
`Lattice` records:

```python
@dataclass
class MergeEvent:
    new_label:      CanonicalSetId
    into_node:      CanonicalSetId          # the existing.canonical_set
    layer:          int
    state_hash:     int

merged_events: list[MergeEvent] = field(default_factory=list)

def _record_merge(self, layer, new_set, into_set):
    self.merged_events.append(MergeEvent(
        new_label=new_set,
        into_node=into_set,
        layer=layer,
        state_hash=self.nodes_by_set[new_set].state_hash,
    ))
```

The dumper (S1.5b.23) iterates `merged_events` for its
timeline + audit-row writes.

  (defensive — shouldn't happen under Apriori-gen but the
  guard fires correctly). Lattice's `nodes_by_set[C]` is
  the same object both times.
- **State-hash collision merge.** Construct two distinct
  canonical_sets `C₁ ≠ C₂` whose try_set produces the
  same state_hash. Verify:
  - The lattice has ONE SetNode with `labels=(C₁, C₂)`
    in arrival order.
  - `parents` is the union of `_parent_sets(C₁) ∪
    _parent_sets(C₂)`.
  - `merged_events` carries one MergeEvent.
- **Verdict preserved.** A dedup-merge against an alive
  existing node keeps the existing's `verdict="alive"`;
  against a dead → "dead"; against a solution →
  "solution".
- **Backbone parity on uniqueness puzzles.** Run
  branching demos that don't trigger state-hash dedup;
  verify lattice's verdict matches monotonic's
  (Solution / bindings unchanged).

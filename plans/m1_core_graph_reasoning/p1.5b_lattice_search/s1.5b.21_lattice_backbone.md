## Goal

Implement `gaps_solve` as a working but unstorage-aware
backbone:

- Exhaustive Apriori-gen layer-by-layer.
- Per-candidate try_commitment_set + flat root-write on
  dead + flat unconditional-merge on alive.
- Collects every satisfying commitment into a local
  `solutions: list[_RawSolution]` (S1.5b.22 promotes to
  `LatticeProof.solutions`).
- Verdict = `Ambiguity(branches=tuple(Solution(kb=s.kb) for
  s in solutions), unresolved=A_N)` always (GAPS contract).
- NO `store_lattice` storage yet (S1.5b.22).
- NO state-hash dedup yet (S1.5b.22 + S1.5b.28).
- NO dumper sections beyond MonotonicDumper-style hooks
  (S1.5b.29 fills the LatticeDumper).
- NO scoring switch (S1.5b.26).

### Why ship gaps_solve before contradictions_solve

Two reasons:

1. **gaps_solve exercises the breakthrough case** — a
   satisfying commitment is the most-tested branch in the
   per-candidate flow (§3c.ii of algorithm_layer_n.md);
   getting it right first makes contradictions_solve a
   simple specialisation.
2. **gaps_solve has the cleaner verdict shape** —
   `Ambiguity(branches=…)` is exactly the existing tree-side
   shape; no new verdict field needed.
   `contradictions_solve`'s `unsat_core` synthesis (⋃ of
   per-dead cores) is a bit more work and benefits from
   the gaps_solve backbone already validating the layer
   loop.

## Algorithm

### `gaps_solve` public entry

```python
def gaps_solve(
    root_kb: KnowledgeBase,
    *,
    max_set_size: int = 5,
    config: SolverConfig | None = None,
    store_lattice: bool = False,        # S1.5b.22 wires this; backbone ignores
    dumper: LatticeDumper | None = None,
    max_time: float | None = None,
    max_enterings: int | None = None,
) -> tuple[Ambiguity, LatticeStats]:
    cfg = config or root_kb.config or SolverConfig()
    root_kb.config = cfg

    state = _LatticeLoopState()        # solutions, dead_commitments, stats, kb_index, ...
    state.entry = "gaps"

    return _explore_layers(
        root_kb, state,
        max_set_size=max_set_size,
        dumper=dumper,
        max_time=max_time, max_enterings=max_enterings,
    )
```

### `_explore_layers` shared helper

Mirrors the monotonic loop structure (S1.5b.5) but
dispatches to entry-specific outcome handlers via the
`state.entry` discriminator. Signature:

```python
def _explore_layers(
    root_kb: KnowledgeBase,
    state: _LatticeLoopState,
    *,
    max_set_size: int,
    dumper: LatticeDumper | None,
    max_time: float | None,
    max_enterings: int | None,
) -> tuple[Verdict, LatticeStats]:
    """Per-candidate flow from algorithm_layer_n.md, with
    outcome dispatch on state.entry ∈ {"gaps", "contradictions"}.

    Phase 1 — initial saturation + alive (same as monotonic).
    Phase 2 — layer-by-layer loop:
        for layer in 1..max_set_size:
            for C in candidates:
                result = try_commitment_set(root, C)
                if dead:
                    flat root-write (emit_nogood, optional negated_fact)
                    state.dispatch_dead(C, result)
                else if alive + is_solved:
                    state.dispatch_solution(C, result.kb)
                else:
                    flat root-merge unconditional facts
                    re-saturate, recompute alive, cascade forced positives
                    if is_solved(root):
                        state.dispatch_solution(C, root)
                a_layer.append(C)
            if a_layer empty: break
    Phase 3 — entry-specific verdict synthesis:
        if state.entry == "gaps":
            return Ambiguity(branches=…, unresolved=A_N, proof=…)
        if state.entry == "contradictions":
            return Contradiction(unsat_core=⋃ cores, proof=…)
    """
```

### `_LatticeLoopState` (backbone shape)

```python
@dataclass
class _LatticeLoopState:
    entry: Literal["gaps", "contradictions"]
    solutions: list[_RawSolution] = field(default_factory=list)
    dead_commitments: list[_RawDead] = field(default_factory=list)  # S1.5b.23 fills for contra
    stats: LatticeStats = field(default_factory=LatticeStats)

    def dispatch_solution(self, c, kb_snapshot) -> None:
        if self.entry == "gaps":
            self.solutions.append(_RawSolution(c, kb_snapshot, ()))
        # contradictions: discard

    def dispatch_dead(self, c, result) -> None:
        if self.entry == "contradictions":
            self.dead_commitments.append(_RawDead(c, result.unsat_core, ...))
        # gaps: discard


@dataclass(frozen=True)
class _RawSolution:
    """Backbone-internal; S1.5b.22 promotes to SolutionRecord."""
    commitment: CanonicalSetId
    kb: KnowledgeBase
    firings: tuple[Firing, ...]


@dataclass(frozen=True)
class _RawDead:
    """Backbone-internal; S1.5b.22 promotes to DeadCommitment."""
    commitment: CanonicalSetId
    unsat_core: frozenset[Fact]
    kind: str
```

The `_Raw*` types are local to backbone testing. S1.5b.22
swaps them for the public `SolutionRecord` / `DeadCommitment`
classes from `lattice.py` and adds the proper kb snapshot.

### KnowledgeBase snapshot

Solutions are stored with `kb=result.kb` (the fork) or
`kb=root_kb` (post-cascade). For the backbone, we use the
live reference — `KnowledgeBase.snapshot()` is implemented
in S1.5b.22 alongside `LatticeProof`. This means backbone
solutions can be aliased / mutated; a backbone test guards
against this by checking values immediately after the
return rather than holding a long-lived reference.

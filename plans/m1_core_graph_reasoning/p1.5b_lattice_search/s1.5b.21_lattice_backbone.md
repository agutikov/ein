## Goal

Two deliverables, in order:

**Part A — extract `_explore_layers`.** Refactor the
existing
[`monotonic_solve`](../../../ein.py/src/ein_bot/inference/monotonic/solver.py)
loop body (the for-layer / for-candidate loop + its Phase 3
verdict synthesis) into a private
`_explore_layers(root_kb, *, entry, ...)` helper inside the
same file. `monotonic_solve` becomes a thin wrapper:

```python
def monotonic_solve(root_kb, *, max_set_size=5, ...):
    return _explore_layers(
        root_kb,
        entry="monotonic",
        max_set_size=max_set_size,
        ...
    )
```

The refactor is **behaviour-preserving** — every existing
monotonic test stays green. Test it by running the full
monotonic test suite + zebra2 acceptance + the branching/*
parity round before adding `gaps_solve`.

**Part B — implement `gaps_solve`.** As a sibling wrapper
that calls `_explore_layers` with `entry="gaps"`. The
helper internally:

- When `entry == "monotonic"` and a fork's
  `is_solved(result.kb)` fires, returns
  `Solution(kb=result.kb)` immediately (existing
  behaviour).
- When `entry == "gaps"` and `is_solved(result.kb)` fires,
  appends to a local `solutions: list[_RawSolution]` and
  **continues** the loop.
- When `entry == "gaps"` and Phase 3 verdict synthesis
  runs, returns
  `Ambiguity(branches=tuple(Solution(kb=s.kb) for s in
  solutions), unresolved=A_N_tuple)` always (GAPS contract).
- The same `entry == "contradictions"` branch is reserved
  but raises until S1.5b.23 wires it.

What this stage explicitly does NOT do:

- NO `LatticeProof` field population — `gaps_solve` uses
  backbone-local `_RawSolution` dataclasses; S1.5b.22
  swaps them for the public `SolutionRecord` /
  `LatticeProof`.
- NO `store_lattice` storage (S1.5b.22).
- NO state-hash dedup (S1.5b.22 + S1.5b.28).
- NO LatticeDumper sections beyond no-op hook calls
  (S1.5b.29 fills the dumper).
- NO scoring switch (S1.5b.26).
- NO `contradictions_solve` body (S1.5b.23).

### Why ship gaps_solve before contradictions_solve

Two reasons:

1. **gaps_solve exercises the breakthrough case** — a
   satisfying commitment is the most-tested branch in the
   per-candidate flow (§3c.ii of algorithm_layer_n.md);
   getting it right first makes contradictions_solve a
   simple specialisation.
2. **gaps_solve has the cleaner verdict shape** —
   `Ambiguity(branches=…)` is exactly the existing tree-side
   shape; no new verdict field needed.
   `contradictions_solve`'s `unsat_core` synthesis (⋃ of
   per-dead cores) is a bit more work and benefits from
   the gaps_solve backbone already validating the layer
   loop.

## Algorithm

All three entries live in
[`inference/monotonic/solver.py`](../../../ein.py/src/ein_bot/inference/monotonic/solver.py).

### `_explore_layers` shared core

Extracted from today's `monotonic_solve` body. Dispatches to
entry-specific outcome handlers via the `entry` discriminator
(``Literal["monotonic", "gaps", "contradictions"]``):

```python
def _explore_layers(
    root_kb: KnowledgeBase,
    *,
    entry: Literal["monotonic", "gaps", "contradictions"],
    max_set_size: int,
    config: SolverConfig | None,
    store_lattice: bool,                    # ignored unless entry != "monotonic"
    dumper: MonotonicDumper | LatticeDumper | None,
    max_time: float | None,
    max_enterings: int | None,
) -> tuple[Verdict, MonotonicStats | LatticeStats]:
    """Per-candidate flow from algorithm_layer_n.md, with
    outcome dispatch on `entry`.

    Phase 1 — initial saturation + alive (same as today's
        monotonic_solve).
    Phase 2 — layer-by-layer loop:
        for layer in 1..max_set_size:
            for C in candidates:
                result = try_commitment_set(root, C)
                if dead:
                    flat root-write (emit_nogood, optional negated_fact)
                    _dispatch_dead(entry, state, C, result, layer)
                elif is_solved(result.kb):
                    if entry == "monotonic":
                        return Solution(kb=result.kb)    # early-terminate
                    else:
                        _dispatch_solution(entry, state, C, result.kb, layer)
                        # gaps/contra: keep going
                else:
                    flat root-merge unconditional facts
                    re-saturate; recompute alive; cascade forced positives
                    if is_solved(root):
                        if entry == "monotonic":
                            return Solution(kb=root)
                        else:
                            _dispatch_solution(entry, state, C, root, layer)
                a_layer.append(C)
            if a_layer empty: break
    Phase 3 — entry-specific verdict synthesis:
        if entry == "monotonic": existing trichotomy
        if entry == "gaps":     return Ambiguity(branches=…, unresolved=A_N, proof=…)
        if entry == "contradictions":
            return Contradiction(unsat_core=⋃ cores, proof=…)
    """
```

Internal state is held in a simple mutable struct local to
`_explore_layers` — no need for a per-state object since
the helper owns everything:

```python
solutions:        list[_RawSolution]   = []     # filled when entry == "gaps"
dead_commitments: list[_RawDead]       = []     # filled when entry == "contradictions"
stats:            MonotonicStats | LatticeStats = ...
```

`_dispatch_solution` and `_dispatch_dead` are private helpers
that no-op for the wrong entry; the loop calls them
unconditionally and they short-circuit.

### `_RawSolution` / `_RawDead` (backbone-internal)

```python
@dataclass(frozen=True)
class _RawSolution:
    """Backbone-internal; S1.5b.22 promotes to SolutionRecord."""
    commitment: CanonicalSetId
    kb: KnowledgeBase
    firings: tuple[Firing, ...]
    layer: int


@dataclass(frozen=True)
class _RawDead:
    """Backbone-internal; S1.5b.22 promotes to DeadCommitment."""
    commitment: CanonicalSetId
    unsat_core: frozenset[Fact]
    kind: str
    layer: int
```

S1.5b.22 swaps these for the public `SolutionRecord` /
`DeadCommitment` from `lattice.py` and adds the proper kb
snapshot via `KnowledgeBase.snapshot()`.

### `gaps_solve` public entry (thin wrapper)

```python
def gaps_solve(
    root_kb: KnowledgeBase,
    *,
    max_set_size: int = 5,
    config: SolverConfig | None = None,
    store_lattice: bool = False,        # S1.5b.22 wires this
    dumper: LatticeDumper | None = None,
    max_time: float | None = None,
    max_enterings: int | None = None,
) -> tuple[Ambiguity, LatticeStats]:
    verdict, stats = _explore_layers(
        root_kb,
        entry="gaps",
        max_set_size=max_set_size,
        config=config,
        store_lattice=store_lattice,
        dumper=dumper,
        max_time=max_time,
        max_enterings=max_enterings,
    )
    # Narrow return type — _explore_layers's union return is
    # discriminated by `entry`.
    assert isinstance(verdict, Ambiguity)
    return verdict, stats
```

### `monotonic_solve` after the refactor

```python
def monotonic_solve(root_kb, *, max_set_size=5, config=None,
                   mode=Mode.SOLVE, dumper=None,
                   max_time=None, max_enterings=None,
                   ) -> tuple[Verdict, MonotonicStats]:
    if mode is not Mode.SOLVE:

```

### KnowledgeBase snapshot

Solutions are stored with `kb=result.kb` (the fork) or
`kb=root_kb` (post-cascade). For the backbone, we use the
live reference — `KnowledgeBase.snapshot()` is implemented
in S1.5b.22 alongside `LatticeProof`. This means backbone
solutions can be aliased / mutated; a backbone test guards
against this by checking values immediately after the
return rather than holding a long-lived reference.

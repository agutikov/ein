## Goal

Two distinct deliverables:

### Part A — `LatticeDumper` implementation

Fill `inference/monotonic/state_dump.py` (skeleton from
S1.5b.20). The dump layout mirrors the
[MonotonicDumper](../../../ein.py/src/ein_bot/inference/monotonic/state_dump.py)
shape from S1.5b.7, plus entry-specific sections:

```
<dump-dir>/
├── root.ein                       ← initial root snapshot
├── 00_timeline.jsonl              ← lifecycle events
├── layers/
│   └── layer_<k>/
│       ├── root_post.ein          ← root after layer k
│       └── enterings/
│           └── <C-slug>/
│               ├── result.json
│               └── …
├── solutions/                     ← under gaps_solve when proof.solutions non-empty
│   └── <C-slug>/
│       ├── commitment.json
│       ├── kb.ein                 ← SolutionRecord.kb (full snapshot)
│       └── firings.jsonl
├── dead/                          ← under contradictions_solve when dead_commitments non-empty
│   └── <C-slug>/
│       ├── commitment.json
│       ├── unsat_core.jsonl
│       └── learned_clause.json
├── kb_index/                      ← when store_lattice=True
│   └── <state_hash_hex>/
│       ├── canonical_set.json
│       ├── labels.json            ← single-element list under gaps_solve; multilabel under contradictions_solve merges
│       └── verdict.txt
├── proof_summary.json             ← top-level index
└── summary.json                   ← cumulative stats
```

`LatticeDumper` is one class with all the hooks from the
S1.5b.20 stub now actually writing files. Both
`gaps_solve` and `contradictions_solve` pass the SAME
dumper instance — sections are emitted based on which
`state.entry` is active + which records were appended to
state.

### Part B — P1.6 handoff contract validator

Helper in `inference/monotonic/contract.py`:

```python
def validate_proof_for_explanation(
    verdict: Verdict, proof: LatticeProof,
) -> None:
    """Exercise every LatticeProof field as P1.6's NL
    renderer would. Raises AssertionError on contract
    violation; returns None on success.

    Usable from tests + from P1.6's renderer entry as a
    pre-flight check.
    """
    # 1. The verdict.proof field is wired.
    assert verdict.proof is proof

    # 2. For each solution: walk its kb.derivation_dag for
    #    every goal fact; verify leaves are root-facts,
    #    hypothesis-facts in the commitment, or forced-
    #    positive promotions.
    for sol in proof.solutions:
        goal_facts = sol.kb.query.goal_facts(sol.kb)
        assert goal_facts, f"solution {sol.commitment} doesn't satisfy goal"
        for f in goal_facts:
            dag = sol.kb.derivation_dag(f)
            for leaf in dag.leaves():
                key = (leaf.relation_name, leaf.args)
                assert (
                    leaf.provenance is None
                    or leaf.provenance.kind in ("root", "fact-layer")
                    or key in sol.commitment
                    or (
                        leaf.provenance.kind == "rule"
                        and leaf.provenance.rule == "<forced-positive>"
                    )
                ), f"unwalked leaf {leaf} in solution {sol.commitment}"

    # 3. For each dead commitment: unsat_core non-empty;
    #    learned_clause matches the commitment (no
    #    subsumption surprises at the per-record level).
    for d in proof.dead_commitments:
        assert d.unsat_core, f"empty unsat_core on dead {d.commitment}"
        assert d.learned_clause == frozenset(d.commitment)

    # 4. learned_nogoods subsumes per-record learned_clauses
    #    (every dead's clause is reachable in learned_nogoods,
    #    possibly via a subset).
    for d in proof.dead_commitments:
        clause = frozenset(d.commitment)
        assert any(
            stored <= clause for stored in proof.learned_nogoods
        ), f"dead clause {clause} not represented in learned_nogoods"

    # 5. kb_index, if non-empty (store_lattice=True), has
    #    consistent multilabel:
    for h, node in proof.kb_index.items():
        assert node.state_hash == h
        assert node.canonical_set in node.labels
        if len(node.labels) > 1:
            # Multi-label only under contradictions_solve;
            # gaps_solve must never merge.
            assert verdict_entry(verdict) == "contradictions", (
                f"multi-label SetNode under non-contradictions entry"
            )

    # 6. Stats sanity.
    assert proof.stats.solutions_found == len(proof.solutions)
    total_dead = (proof.stats.enterings_dead_pre
                  + proof.stats.enterings_dead_post)
    assert total_dead >= len(proof.dead_commitments)
```

Used in tests + as a pre-flight check from P1.6's
renderer (once it's implemented).

```json
  "entry": "gaps" | "contradictions",
  "store_lattice": true | false,
  "solutions": [
    {"slug": "...", "layer": 1, "kb_path": "solutions/.../kb.ein"}
  ],
  "dead_commitments": [
    {"slug": "...", "layer": 2, "kind": "dead-post",
     "unsat_core_path": "...", "learned_clause": [...]}
  ],
  "kb_index": [
    {"state_hash_hex": "0x...", "canonical_set": [...],
     "labels": [[...], ...], "verdict": "dead"}
  ],
  "alive_at_end": [[...]],
  "learned_nogoods_count": 17,
  "stats": {...LatticeStats...}
}
```

### T1.5b.29.3 — Solver wiring

Three new dumper calls in `_explore_layers`:

- After `state.dispatch_solution(...)` →
  `dumper.solution_recorded(record, layer)` (gaps_solve
  only — dispatch is no-op for contradictions).
- After `state.dispatch_dead(...)` →
  `dumper.dead_recorded(dead, layer)` (contradictions_solve
  only).
- Replace existing `dumper.summary(verdict, stats)` with
  the sequence: `dumper.proof_summary(proof)` (if proof
  non-None) then `dumper.summary(verdict, stats)`.

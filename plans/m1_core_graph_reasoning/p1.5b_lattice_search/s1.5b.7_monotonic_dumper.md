## Goal

Fill the `MonotonicDumper` stubs from
[S1.5b.4](s1.5b.4_monotonic_skeleton.md) with a minimal
diagnostic dumper. Output layout intentionally simpler than
the lattice's `set/<canonical-slug>/` per-set folders вЂ”
monotonic doesn't store per-set kbs, so there's nothing to
mirror.

After this stage:

```
dump/<puzzle>-<ts>/
   00_root_initial.ein           в†ђ root before any enterings
   00_timeline.jsonl             в†ђ chronological event log
   layers/
       layer_01_pre.ein          в†ђ root.kb at layer 1 start
       layer_01_post.ein         в†ђ root.kb at layer 1 end (after all merges)
       layer_02_pre.ein
       layer_02_post.ein
       ...
   summary.json                  в†ђ final stats + verdict
```

No `set/*/branch_result.json` (those are lattice-only). The
timeline IS the per-entering audit; reading it linearly tells
the full story.

## `MonotonicDumper` surface

`ein.py/src/ein/inference/monotonic/state_dump.py`:

```python
"""MonotonicDumper вЂ” per-layer filesystem snapshots of the
monotonic solve loop.

Mirrors `state_dump.StateDumper`'s shape but with no per-set
storage (monotonic doesn't have SetNodes). The deliverable
is enough on-disk evidence to reconstruct the search by
reading the dump linearly.
"""
from __future__ import annotations

import json
import time
from dataclasses import dataclass, field
from pathlib import Path
from typing import IO, Any

from ein.inference.commitment import CommitmentSetResult
from ein.inference.state_dump import (
    _kb_to_ein_text,  # base helper from common dumper
    _fact_summary,
)
from ein.kb.entities import Fact
from ein.kb.store import KnowledgeBase

@dataclass
class MonotonicDumper:
    """Filesystem-snapshotting hooks attached to a
    `monotonic_solve()` invocation.
    """
    out_dir: Path
    started_at: float = field(default_factory=time.time)
    _timeline_fp: IO[str] | None = field(
        default=None, init=False, repr=False,
    )
    _timeline_seq: int = field(default=0, init=False)

    def __post_init__(self) -> None:
        self.out_dir.mkdir(parents=True, exist_ok=True)
        (self.out_dir / "layers").mkdir(exist_ok=True)
        self._timeline_fp = (
            self.out_dir / "00_timeline.jsonl"
        ).open("w")

    # в”Ђв”Ђ Lifecycle hooks в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ

    def root_initial(self, kb: KnowledgeBase) -> None:
        (self.out_dir / "00_root_initial.ein").write_text(
            _kb_to_ein_text(kb),
        )
        self._emit_timeline(
            "root_initial",
            facts=len(kb.facts),
        )

    def layer_start(self, layer: int, alive_size: int) -> None:
        path = self.out_dir / "layers" / f"layer_{layer:02d}_pre.ein"
        # kb snapshot will be written by `layer_end`; pre is
        # written here, end-of-previous-layer's post.
        self._emit_timeline(
            "layer_start",
            layer=layer,
            alive_size=alive_size,
        )

    def entering(
        self,
        layer: int,
        commitment: tuple,           # CanonicalSetId
        result: CommitmentSetResult,
        facts_merged: int,
        nogood_emitted: bool,
        nogood_subsumed: bool,
    ) -> None:
        """One try_commitment_set outcome вЂ” alive or dead."""
        self._emit_timeline(
            "entering",
            layer=layer,
            commitment=[
                {"relation": rn, "args": [str(a) for a in args]}
                for (rn, args) in commitment
            ],
            kind=result.kind,
            firings=len(result.firings),
            facts_merged=facts_merged,
            unconditional_count=len(result.unconditional_facts),
            unsat_core_size=len(result.unsat_core),
            nogood_emitted=nogood_emitted,
            nogood_subsumed=nogood_subsumed,
        )

    def layer_end(
        self,
        layer: int,
        kb: KnowledgeBase,
        alive_size: int,
        survived_count: int,
    ) -> None:
        (self.out_dir / "layers" / f"layer_{layer:02d}_post.ein").write_text(
            _kb_to_ein_text(kb),
        )
        # Also write pre-of-next-layer = post-of-this-layer
        # for visual continuity (cheap; just a copy).
        next_pre = self.out_dir / "layers" / f"layer_{layer + 1:02d}_pre.ein"
        next_pre.write_text(_kb_to_ein_text(kb))

        self._emit_timeline(
            "layer_end",
            layer=layer,
            facts=len(kb.facts),
            alive_size=alive_size,
            survived_count=survived_count,
        )

    def early_terminate(
        self, layer: int, reason: str,
    ) -> None:
        """The loop returned mid-layer because root satisfied
        the goal (or another early-exit condition fired)."""
        self._emit_timeline(
            "early_terminate",
            layer=layer,
            reason=reason,
        )

    def summary(self, verdict: Any, stats: Any) -> None:
        verdict_kind = type(verdict).__name__
        elapsed = time.time() - self.started_at
        from dataclasses import fields as _fields
        try:
            stats_dict = {
                f.name: getattr(stats, f.name)
                for f in _fields(stats)
            }
        except TypeError:
            stats_dict = {"repr": repr(stats)}

        (self.out_dir / "summary.json").write_text(
            json.dumps({
                "verdict": verdict_kind,
                "elapsed_seconds": round(elapsed, 3),
                "stats": stats_dict,
            }, indent=2, sort_keys=True, default=str),
        )
        self._emit_timeline(
            "summary",
            verdict=verdict_kind,
            elapsed_seconds=round(elapsed, 3),
        )
        if self._timeline_fp is not None:
            self._timeline_fp.close()
            self._timeline_fp = None

    # в”Ђв”Ђ Internals в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ

    def _emit_timeline(self, event: str, **fields: Any) -> None:
        if self._timeline_fp is None:
            return
        rec = {
            "seq": self._timeline_seq,
            "ts_ms": round(
                (time.time() - self.started_at) * 1000, 3,
            ),
            "event": event,
            **fields,
        }
        self._timeline_fp.write(json.dumps(rec) + "\n")
        self._timeline_fp.flush()
        self._timeline_seq += 1
```

Total ~120 LOC.

## Backbone hooks

`monotonic_solve` accepts an optional `dumper: MonotonicDumper | None = None` parameter and calls hooks at:

| backbone site                                | hook                                       |
|----------------------------------------------|--------------------------------------------|
| right after `Saturator(root_kb).saturate()` in Phase 1 | `dumper.root_initial(root_kb)`             |
| at the top of each `for layer in вЂ¦:`         | `dumper.layer_start(layer, len(alive))`    |
| inside the candidate loop, after `try_commitment_set`   | `dumper.entering(layer, C, result, facts_merged, nogood_emitted, nogood_subsumed)` |
| after each layer's `for` loop completes      | `dumper.layer_end(layer, root_kb, len(alive), len(A_layer))` |
| on early termination (is_solved True)        | `dumper.early_terminate(layer, "is_solved")` |
| at function return (any verdict)             | `dumper.summary(verdict, stats)`            |

The hooks are no-ops when `dumper is None`. Existing
backbone tests pass unchanged.

## What this dumper deliberately doesn't carry

Versus the lattice's per-set dumper (S1.5b.23):

- **No `set/<canonical-slug>/` folders.** Monotonic doesn't
  store per-set kbs вЂ” there's nothing to dump per set.
- **No per-set `branch_result.json`.** The `entering` event
  in the timeline holds the equivalent metadata flat.
- **No `subtree_result.json`.** No subtree exists in
  monotonic.
- **No `enterings/s{i}/integrate.json`.** No integrate; the
  "merge unconditional facts" step has no parent-side
  decision to record.

This is by design вЂ” monotonic trades audit fidelity for
simplicity. P1.6's NL renderer (S1.6.4) can still consume
the timeline + per-layer .ein snapshots to render an
explanation, but the lattice's `LatticeProof` is more
self-contained.

## Ship notes вЂ” 2026-05-27

Status: implemented at `6b30a00` (base `527820d` вЂ” the S1.5b.6
backfill).

Deltas from the spec above:

- **`_kb_to_ein_text` import path.** The spec's
  `from ein.inference.state_dump import _kb_to_ein_text` is
  stale вЂ” S1.5b.1's correction (commit `995315b`) moved the
  helper under `inference/tree/state_dump.py`. The monotonic
  dumper imports from there. Cross-engine import is already an
  established pattern (`monotonic/solver.py` imports Verdict
  types from `inference/tree/solver.py`).

- **`_fact_summary` not imported.** The spec listed it in the
  import block but the dumper doesn't actually need it вЂ” every
  per-fact serialisation happens via the inline
  `{"relation": rn, "args": [str(a) for a in args]}` shape on
  the `entering` event's commitment field.

- **`layer_start` writes `layer_NN_pre.ein`.** The spec had only
  `layer_end` writing a `next_pre` (the next layer's pre).
  That left `layer_01_pre.ein` un-written and, on the last
  iteration, produced a stray `layer_(N+1)_pre.ein` for a layer
  that never ran. Owning the pre write in `layer_start` (one
  kb-text render per layer) keeps `pre`/`post` paired and
  matches the documented layout.

- **`layer_start` takes the kb explicitly.** Spec signature was
  `layer_start(layer, alive_size)`; to write the pre snapshot
  the dumper needs the kb. Added `kb: KnowledgeBase` as the
  second positional. The backbone passes `root_kb` at every
  hook call site.

- **`_finish(dumper, verdict, stats)` helper for single-exit
  summary.** Rather than thread `dumper.summary(...)` into every
  return path (~9 sites across Phases 1/2/3 + forced-positive
  cascade), every `return X, stats` was replaced with
  `return _finish(dumper, X, stats)`. Equivalent semantics; one
  place to audit.

- **`root_changed: bool` в†’ `this_merged: int`.** The Phase-2
  alive-entering's merge counter is now a per-entering integer
  (`this_merged`), so the dumper's `entering` event records
  the actual number of facts merged. The truthy check
  `if this_merged:` replaces `if root_changed:`. No behaviour
  change.

- **`early_terminate` fires only on the in-loop is_solved
  hit.** Phase-1 / Phase-3 is_solved + contradiction returns
  go through `_finish` and surface in `summary.json`'s
  `verdict` field вЂ” sufficient for diagnostics. The
  `early_terminate` event is reserved for "Phase-2 mid-loop
  goal satisfied", which is the only non-obvious exit a
  diagnostic reader needs flagged.

Tests added (`tests/inference/monotonic/test_monotonic_dumper.py`,

- `test_run_produces_expected_files` вЂ” the four documented

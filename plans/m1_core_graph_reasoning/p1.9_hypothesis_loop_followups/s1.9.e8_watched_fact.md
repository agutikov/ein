# S1.9.E8 — Watched-fact rule applicability

**Phase:** P1.9, §conflict-driven learning.

**Value:** M.

**Catalog row:** [P1.9 README — Conflict-driven learning](README.md#conflict-driven-learning-satcdcl-inspired)
E8.

## Context

DPLL "watched literals" idiom adapted to ein's rules. For each
rule's `:match` clause, pick 2 watch literals; a rule needs
re-evaluation only when one of its watches changes. Avoids
the current "re-iterate every applicable rule on every
saturation pass" cost.

## Tasks

### Task T1.9.E8.1 — Watch-literal selection

For each rule, pick 2 literals from `:match` to watch.
Heuristic: pick literals whose relations are *least frequent*
in the KB (rare relations change less often → fewer wakeups).

### Task T1.9.E8.2 — Watcher index

`(relation, args) → list[rule]` map of watch literals to rules
that watch them. Updated on fact insertion.

### Task T1.9.E8.3 — Saturator integration

When a fact arrives:

- **a.** Look up watchers.
- **b.** Enqueue only watched rules for re-eval.
- **c.** If a watch literal "moves" (the rule's other watch
  fires first), re-pick its watch.

### Task T1.9.E8.4 — Tests + bench

Compare saturation perf with and without watches.

## Acceptance

- Watch index works; saturator wakes correctly on watched-fact
  arrivals.
- Bench shows ≥ 30% saturation speedup on dense rule sets.

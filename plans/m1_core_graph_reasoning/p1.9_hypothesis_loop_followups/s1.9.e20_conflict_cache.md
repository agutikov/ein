## Context

S1.5.7 caches `(not h)` learnings within a `solve()` call.
E20 extends to *cross-call* persistence: when the same puzzle
runs twice (e.g., M2b benchmark loop, or interactive REPL
testing), the second run reuses the first run's negative
learnings.

Session-scoped; cleared when the puzzle's source file changes.

## Tasks

### Task T1.9.E20.1 — Cache persistence

Serialize negative learnings to a per-puzzle cache file
(`.ein.cache/` sibling, or `~/.cache/ein/<hash>/`).

### Task T1.9.E20.2 — Validity check

Cache entry is valid iff the puzzle's source hash matches.

### Task T1.9.E20.3 — Tests + bench

Run the same puzzle twice; second run is faster.

## Acceptance

- Cache mechanism works; validity check rejects stale.
- Bench: ≥ 50% speedup on re-runs of cached puzzles.

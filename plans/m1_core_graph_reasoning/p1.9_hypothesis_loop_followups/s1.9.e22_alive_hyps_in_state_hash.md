# S1.9.E22 — Alive-hyps in canonical state hash

**Value:** M (correctness).

## Context

Open question: can two distinct paths reach the same post-
saturation KB but carry **different alive hypothesis sets**?

Under M1's monotonic semantics (append-only KB, no fact
retraction, rules don't depend on path history), the
saturated KB IS a pure function of its fact set — so the
alive set at exit should be a pure function of the KB as well,
regardless of how the KB was reached. *If* that's true, the
current state-hash dedup is sound. *If* not, the dedup is
unsound for the [T1.5.4.5/8 alive-inherit](../p1.5_hypothesis_loop/s1.5.4_hypgen_improvements.md)
optimisation and the fix is to include the alive set in the
canonical hash.

## Tasks

### Task T1.9.E22.1 — Analysis

Formally argue:

- Either: KB ⇒ alive-set (pure function), so dedup is sound
  as-is.
- Or: a counter-example puzzle exists; build it.

The argument likely turns on whether rules can fire
conditional on what's *missing* (NAF) at varying points along
a path. With S1.5a.1's fire-time NAF re-eval, this becomes
state-determined; without it, path-dependent.

### Task T1.9.E22.2 — Fix (if needed)

If counter-example: extend `state_hash` to include the alive
set; collapse only paths whose `(state, alive)` pair matches.

### Task T1.9.E22.3 — Document

Either way, record the invariant in
[`docs/kernel/inference/`](../../../docs/kernel/inference/) so
future readers know what's relied upon.

## Acceptance

- Argument or fix recorded.
- T1.5.4.5/8 alive-inherit ships in production-default mode
  with the invariant documented.

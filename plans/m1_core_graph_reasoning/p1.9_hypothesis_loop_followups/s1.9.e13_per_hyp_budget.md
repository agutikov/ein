## Lattice re-grounding (2026-06-15)

The mechanism is applicable — each commitment-set "entering" forks and
saturates (`try_commitment_set`), and a per-entering step budget would bound
that saturation. But two reframings:

- **"Treat as alive (depth-N open leaf)" is DFS-tree language.** The lattice
  has no tree of leaves; the right framing is "mark the commitment
  **undetermined**" (neither solution nor dead).
- **It breaks soundness for the complete search.** Aborting saturation early
  can **miss a contradiction** the full saturation would find — so the engine
  could call a non-model a model, violating the P1.7a property ("never call a
  non-model a model"). So E13 can only run on the fast, explicitly-incomplete
  `solve(stop_after=1)` path, never on the uniqueness-establishing complete
  `solve()`.

Re-scope to a **fast-path-only** knob with an explicit "undetermined /
budget-truncated" verdict (composes with [E17](s1.9.e17_branch_budget.md)'s
`stats.exhausted`). Not a complete-search lever.

## Context

Branch-and-bound on saturation: if a fork's saturator runs
more than K steps without reaching a verdict, abort + treat
the fork as an open-leaf alive verdict.

Cheap puzzles → tight K; hard puzzles → loose K. The user
trades thoroughness for response time.

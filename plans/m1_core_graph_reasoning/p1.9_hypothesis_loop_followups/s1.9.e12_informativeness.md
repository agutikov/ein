# S1.9.E12 — Hypothesis ordering by informativeness

**Phase:** P1.9, §search heuristics.
**Estimate:** M (1-3 days).
**Value:** L (heuristic gain).
**Status:** ❌ **rejected 2026-06-15** (per user, measured) — on zebra2 a

## Rejected (measured) 2026-06-15

`demo/score_hypotheses.py` (the E9/E12 measurement tool) shows why
informativeness ordering can't beat lex on zebra2. Of 56 layer-1 singletons:
4 complete (fire 561), 34 are alive-incomplete (331–489), and **18 are
multi-step deaths that fire 2794** before reaching `(false)`. "Prefer the
largest cascade" therefore fronts the *deads* — first completer at rank **19**
(vs lex 11; full table in [E9 § Rejected](s1.9.e9_lcv.md#rejected-measured-2026-06-15)).
The completers are only distinguishable as the **max among the *alive***, but
separating alive from dead-post needs *full saturation* (these deaths are too
deep for the one-step lookahead). So the discriminating signal is **irreducibly
post-fork** — the precise reason `branch-info` scoring is a `NotImplementedError`
stub in `hypgen.py`. No cheap pre-fork estimate beats lex. Rejected.

## Lattice re-grounding (2026-06-15)

Like [E9](s1.9.e9_lcv.md), this is a **reorderer** — it changes the order in
which layer-1 atoms are tried, not which commitment sets exist. In the
**complete BFS over cardinality** (`architecture_and_algorithms.md` §O7) all of
layer k−1 is processed before layer k, so within-layer order **cannot change
downstream Apriori/nogood pruning** (prior-layer nogoods are already in hand;
same-size sets don't subsume). One might hope a high-pruning-first order
triggers *earlier* conflict learning — but that intuition is DFS-shaped; in the
layer-synchronous BFS it does not hold.

So informativeness ordering is **inert for the uniqueness-establishing
`solve()`** and matters only to the fast `solve(stop_after=1)` find-first path
(its sibling `branch-info` scoring is already a `NotImplementedError` stub in
`hypgen.py`). Fast-path-only.

## Context

CSP value-ordering. Score each candidate by *expected pruning*:

- If the candidate would land alive, how many other alive
  hypotheses does it eliminate via propagation?
- If it would die, how much does back-propagation
  (S1.5.7) gain?

Pick the highest-info candidate first.

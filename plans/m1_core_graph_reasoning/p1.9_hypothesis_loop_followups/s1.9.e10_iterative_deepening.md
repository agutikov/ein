## Context

IDA*-style: don't commit to a `max_depth` up front. Start at
depth 1; if `Ambiguity` with open leaves, increase depth + retry.
Cheap puzzles bottom out shallow; hard puzzles pay the
normal depth cost.

## Tasks

### Task T1.9.E10.1 — Loop wrapper

Wrap `solve()` in an iterative-deepening loop. Each iteration:
clear the search tree (or use a fresh `state_hash`-namespaced
session), bump `max_depth`, retry.

### Task T1.9.E10.2 — Termination condition

When to stop deepening? Either: a verdict ≠ open-leaf
ambiguity, or `max_depth` hits a configured ceiling.

## Inapplicable to the lattice BFS (2026-06-15)

IDA* iterative deepening is a **depth-first** technique: bound DFS depth,
re-raise the bound when the goal isn't found. The engine's search is not DFS —
it is a **complete BFS over commitment-set cardinality** (`solver.py` runs
`for layer in range(1, max_set_size + 1)`: layer 1 = singletons, layer 2 =
pairs, …, with Apriori downward-closure pruning;
`architecture_and_algorithms.md` §O7). That layering **already is** the
deepening this entry asked for:

- "Start shallow, deepen" — size-1 sets are visited before size-2; the search
  terminates early on solution / contradiction / frontier-exhaustion.
- "Cheap puzzles bottom out shallow" — already delivered: a puzzle solved at
  layer 1–2 never reaches `max_set_size`.

There is **no `max_depth` to iteratively re-raise** (the only bound is the
`max_set_size` cap, walked monotonically upward — no ID wrapper). Re-running
with an increasing bound would only **re-do** the lower layers — strictly worse
than the single ascending pass already shipped. **E10 has no applicable form
here.** Closing.

## Context

IDA*-style: don't commit to a `max_depth` up front. Start at
depth 1; if `Ambiguity` with open leaves, increase depth + retry.
Cheap puzzles bottom out shallow; hard puzzles pay the
normal depth cost.

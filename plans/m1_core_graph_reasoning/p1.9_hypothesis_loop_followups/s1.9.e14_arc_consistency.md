## Lattice re-grounding (2026-06-15)

AC-3 as a pre-pass is a **space-shrinker**: pruning trivially-impossible values
**removes alive atoms**, shrinking the cardinality-BFS lattice and so the number
of fork+saturate enterings (the O1+O2 bottleneck) — a complete-search wall-time
lever, not a reorderer.

Note the engine **already has the weakest form**: `lookahead.py` is a one-step
**singleton arc-consistency / forward-checking** pass that kills a candidate
which would die in a single rule firing, before paying for a fork
(`architecture_and_algorithms.md` §O8). E14 generalises that to a **global,
iterated** AC-3 worklist run **once up front** over the relation graph, rather
than one-candidate-at-a-time at branch time. Value scales with puzzle size;
cold on zebra2.

## Context

Classical CSP AC-3 algorithm: iterate "remove value v from
domain of variable A if no value of variable B is consistent
with v". Pre-saturation pass.

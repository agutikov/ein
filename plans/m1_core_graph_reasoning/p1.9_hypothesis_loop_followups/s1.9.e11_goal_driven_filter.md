## Lattice re-grounding (2026-06-15)

Unlike the value-ordering entries ([E9](s1.9.e9_lcv.md) /
[E12](s1.9.e12_informativeness.md)), goal-driven filtering is a
**space-shrinker**, not a reorderer: dropping goal-irrelevant candidates
**removes layer-1 atoms**, which exponentially reduces the commitment sets the
cardinality BFS enumerates (`architecture_and_algorithms.md` §O7). Since **each
entering is a fork + saturation**, fewer atoms ⇒ fewer enterings ⇒ **fewer
saturations** — it attacks the measured bottleneck (O1+O2, ~95% of a solve)
from the *count* side, complementary to P1.8a's *per-saturation* speedup.

So E11 is a real complete-search wall-time lever (the O2 "no magic sets /
goal-direction" gap) and the most valuable of the search-heuristic entries.
Caveat: on zebra2 the search is already tiny (11 enterings), so the value
**scales with puzzle hardness** — activation awaits a puzzle whose alive
frontier is large.

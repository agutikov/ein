# S1.9.E11 — Goal-driven hypothesis filter

**Phase:** P1.9, §search heuristics.
**Estimate:** M (1-3 days).
**Value:** M.
**Status:** ❌ **rejected 2026-06-15** (per user): "we can't filter a hypothesis

**Catalog row:** [P1.9 README — Search heuristics](README.md#search-heuristics)

## Rejected (2026-06-15)

Per the user: **you can't filter a hypothesis without testing it.** Goal-driven
filtering *drops* a candidate on the claim it can't affect the query — but the
engine only learns a candidate's effect by forking + saturating it. Three
findings, all pointing the same way:

- **Unsound as specified.** "Emit only candidates that unify with the goal
  (positive backward chaining)" misses that a goal-*irrelevant* commitment can
  still trigger a **contradiction** pruning a goal-*relevant* branch; dropping it
  loses that pruning → `solve()` could return a wrong goal-answer.
- **Sound variant is cold.** A sound filter needs goal ∪ *contradiction*
  reachability — which on a fully-connected puzzle (every zebra clue links every
  attribute) reaches *everything* and prunes nothing.
- **Semantic cost.** Even where it prunes, it changes `solve()` from "unique
  *complete* model" (P1.7a) to "unique *goal-answer*," reintroducing the
  goal-directedness P1.7a removed.

E11's one genuine use — a deliberate **goal-directed query mode** for the M2 NL
pipeline — is a separate feature decision, not a P1.9 search heuristic.

## Lattice re-grounding (2026-06-15)

Unlike the value-ordering entries ([E9](s1.9.e9_lcv.md) /
[E12](s1.9.e12_informativeness.md)), goal-driven filtering is a
**space-shrinker**, not a reorderer: dropping goal-irrelevant candidates
**removes layer-1 atoms**, which exponentially reduces the commitment sets the
cardinality BFS enumerates (`architecture_and_algorithms.md` §O7). Since **each
entering is a fork + saturation**, fewer atoms ⇒ fewer enterings ⇒ **fewer
saturations** — it attacks the measured bottleneck (O1+O2, ~95% of a solve)
from the *count* side, complementary to P1.8a's *per-saturation* speedup.

So E11 is a real complete-search wall-time lever (the O2 "no magic sets /
goal-direction" gap) and the most valuable of the search-heuristic entries.
Caveat: on zebra2 the search is already tiny (11 enterings), so the value
**scales with puzzle hardness** — activation awaits a puzzle whose alive
frontier is large.

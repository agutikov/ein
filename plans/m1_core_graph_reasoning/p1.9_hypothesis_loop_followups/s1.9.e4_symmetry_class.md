## Superseded by the D/A/B/C decomposition (2026-06-15)

The 2026-06-15 dunder arc (commits `82983ad` Phase 1, `0ff56ff` Phase 2a,
`6c16488` Phase 2b) reorganised the *relation*-symmetry machinery E4 layers
over and, in doing so, re-homed E4's two genuinely-new deliverables. The
Phase 2a commit decomposes the symmetric design space into four pieces:

| piece | what | where it lives now |
|-------|------|--------------------|
| **D** positive mirror | `(symmetric R)` ⇒ `(R b a)` for every `(R a b)` | stdlib `symmetric` rule ([`std.algebra`](../../../ein.py/src/ein_bot/stdlib/algebra.ein)) + opt-in kernel perf-twin `(__symmetric__ R)` ([`saturator.py`](../../../ein.py/src/ein_bot/inference/saturator.py) `SYMMETRIC`) — Phase 2b |
| **A** negative mirror | `(symmetric R)` + `(not (R a b))` ⇒ `(not (R b a))` | stdlib `symmetric-negative` / `-setup` rules (Phase 2a) — the userspace reinstatement of the on-death mirror [S1.7.24](../p1.7_bootstrapping_zebra/s1.7.24_dehardcode_symmetric.md) removed from the kernel |
| **B** canonical-pair generation | emit only one ordering of a symmetric pair at gen-time | **a user hrule** (the `mini_zebra` `guess`-style pattern, [`05_mini_zebra.ein`](../../../examples/branching/05_mini_zebra.ein)) — *not* a kernel activator |
| **C** uniqueness-up-to-symmetry | collapse isomorphic solutions to one | **subsumed by D**: with the positive mirror an orientation-pair lands in the *same* fact set ⇒ same `state_hash` ⇒ deduped |

E4's two "genuinely-new things" (see [§ Lattice re-grounding](#lattice-re-grounding-2026-06-15))
map exactly onto **B** and **C**: E4(a) gen-time pruning = B (a user hrule, by
design, not a `(symmetry-class R T)` activator); E4(b) uniqueness-up-to-symmetry
= C (subsumed by the positive mirror D). So the dedicated activator this stage
specced is **not going to be built** — its work is done elsewhere. This is the
same "shipped differently" shape as the ⛔ **E21** entry (`solve`/`prove` split,
superseded by P1.7a).

**Residual (re-promotable).** D and C only handle relation-*orientation*
symmetry (`(R a b)` vs `(R b a)`). E4's original framing also covers
**object-permutation value symmetry** — the 5 interchangeable Houses — which
D/A/B/C do *not* address and which `state_hash` still does not dedup. The sound
form is lex-leader SBP / SBDD (L-effort) and **no current puzzle exercises it**
(zebra2's houses are position-distinguished). Re-promote from ⛔ only if a
genuinely value-symmetric puzzle appears.

## Lattice re-grounding (2026-06-15)

Value-symmetry breaking is a **space-shrinker**, but note what it does and does
*not* compose with. `canon.state_hash` collapses **path-duplicate** models
(distinct proof paths reaching the *identical* fact set —
`architecture_and_algorithms.md` §O9); it does **not** collapse value-symmetric
models, because two symmetric solutions have *different* fact sets (`(R a v1)`
vs `(R a v2)`) → different hashes. O9 says as much: full symmetry breaking
"would only matter for far larger or highly-symmetric instances." So **today
symmetric solutions are not pruned at all** — each is explored (a full
saturation per symmetric commitment) and surfaces as a *separate* solution
node, so `solve()` reports **ambiguity** for what is one solution up to
symmetry.

E4 would therefore do two genuinely-new things: (a) **prune** symmetric
branches at gen-time → fewer enterings/saturations (the O1+O2 bottleneck); and
(b) **change the verdict to uniqueness-up-to-symmetry** (collapse the N
isomorphic solution nodes to one) — a *semantic* change a puzzle must opt into.
**Soundness is subtle**: a static "emit one representative" is unsound once
facts break the symmetry mid-search, so a correct E4 needs lex-leader
symmetry-breaking constraints (static SBP) or dynamic SBDD — an L-effort
feature. No current puzzle exercises it (zebra2's houses are
**position-distinguished**, not a symmetry class), so it also needs a synthetic
symmetric fixture. Cold on zebra2; activation awaits a genuinely-symmetric
puzzle.

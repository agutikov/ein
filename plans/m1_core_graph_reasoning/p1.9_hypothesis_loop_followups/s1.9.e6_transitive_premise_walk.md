## Context

Walk `Provenance.premises_raw` chain; "unconditional" iff *no*
transitive premise has `kind="hypothesis"`. Safer than the
original T1.5.4.3.a shallow spec (which checked only direct
parents).

S1.5.7 ships the *immediate* use (back-propagation of `(not h)`
on unconditional death). This catalog entry tracks the
*broader applicability*: unsat-core minimisation (E19) +
learned-clause from unsat-core (E7) both need transitive
premise walks.

**Note (2026-06-15; corrected same day).** The transitive walk ships — but
**not** as `back_prop.is_unconditional_death`, which is **dead tree-solver
residue** (0 callers repo-wide, 0 tests; the tree solver was removed in commit
`8d77b02`). The live form is the generic provenance DFS `reaches()`
(`provenance.py:282`) consumed by `commitment._is_unconditional`
(`commitment.py:182`): the **commitment-set lattice** (`inference/monotonic/`)
uses it to extract an alive commitment's unconditional consequences and merge
them to *root* (`solver.py` — "one alive commitment cascades into ~30
unconditional facts that complete zebra2 at root"). So the shallow-vs-transitive
soundness point is live, but its home and use-site are the lattice — **not** the
parent-KB / sibling / ancestor-chain back-prop this stub's Context + Tasks
describe, which is the removed tree solver's vocabulary (stale; rewrite if this
entry ever promotes). The residual this entry still asks for: a **set-returning**
`walk_premises(fact, kb) → set[Fact]` collector for E7/E19 (today `reaches()`
returns `bool` and short-circuits). NB the back-pointer field is `premises_raw`.

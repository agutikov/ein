## Executed (2026-06-15)

`walk_premises(root, resolve, *, keep, visited) -> set[Fact]`
(`ein.py/src/ein/kb/provenance.py`) — the **set-collecting dual** of
`reaches` (which short-circuits and returns `bool`). It walks the whole premise
closure over rule-kind `premises_raw` and accumulates every fact the
`keep(key, fact)` predicate selects; `visited` memoises across roots. Iterative
(explicit stack) so a deep derivation chain can't blow the recursion limit.

API note vs the stub's `walk_premises(fact, kb)`: it takes a `resolve` callback
(not `kb`) — keeping `provenance.py` free of a `store` import, exactly as
`reaches` / `build_derivation_dag` already do — plus a general `keep` predicate
(the stub's "optional predicate filter", generalised so membership ≠ stopping).

**Live consumer (DRY):** `store.unsat_core` now delegates to it (frontier
predicate = un-provenanced or `kind ∈ {source, hypothesis}`), replacing the
`derivation_dag(f).sources` path — the same premise-closure walk without
materialising the edge set, one shared `visited` across the conflicting facts.
Proven byte-identical to the old output on both `examples/ein-bugs/*`
contradictions (`38 == 38`, `5 == 5`). `reaches` stays as-is (its short-circuit
is load-bearing on the `_is_unconditional` hot path); `build_derivation_dag`
stays (it also builds the edge set, for the trace DOT).

Downstream: this is the shared substrate the core minimiser
([E19](s1.9.e19_unsat_core_min.md)) and the P1.6 trace will consume.
[E7](s1.9.e7_learned_clause.md)'s findings determined that no *pruning* consumer
is warranted (the nogoods are already minimal).

## Context

Walk `Provenance.premises_raw` chain; "unconditional" iff *no*
transitive premise has `kind="hypothesis"`. Safer than the
original T1.5.4.3.a shallow spec (which checked only direct
parents).

S1.5.7 ships the *immediate* use (back-propagation of `(not h)`
on unconditional death). This catalog entry tracks the
*broader applicability*: unsat-core minimisation (E19) +
learned-clause from unsat-core (E7) both need transitive
premise walks.

**Note (2026-06-15; corrected same day).** The transitive walk ships — but
**not** as `back_prop.is_unconditional_death`, which is **dead tree-solver
residue** (0 callers repo-wide, 0 tests; the tree solver was removed in commit
`8d77b02`). The live form is the generic provenance DFS `reaches()`
(`provenance.py:282`) consumed by `commitment._is_unconditional`
(`commitment.py:182`): the **commitment-set lattice** (`inference/monotonic/`)
uses it to extract an alive commitment's unconditional consequences and merge
them to *root* (`solver.py` — "one alive commitment cascades into ~30
unconditional facts that complete zebra2 at root"). So the shallow-vs-transitive
soundness point is live, but its home and use-site are the lattice — **not** the
parent-KB / sibling / ancestor-chain back-prop this stub's Context + Tasks
describe, which is the removed tree solver's vocabulary (stale; rewrite if this
entry ever promotes). The residual this entry still asks for: a **set-returning**
`walk_premises(fact, kb) → set[Fact]` collector for E7/E19 (today `reaches()`
returns `bool` and short-circuits). NB the back-pointer field is `premises_raw`.

## Context

Walk `Provenance.premises_raw` chain; "unconditional" iff *no*
transitive premise has `kind="hypothesis"`. Safer than the
original T1.5.4.3.a shallow spec (which checked only direct
parents).

S1.5.7 ships the *immediate* use (back-propagation of `(not h)`
on unconditional death). This catalog entry tracks the
*broader applicability*: unsat-core minimisation (E19) +
learned-clause from unsat-core (E7) both need transitive
premise walks.

**Note (2026-06-15).** The *analysis* already ships — as the boolean predicate
`is_unconditional_death` + the generic provenance DFS `reaches()`
(`back_prop.py:95`, `provenance.py:282`), which walks `Provenance.premises_raw`
and short-circuits at a hypothesis terminal. What this entry still asks for is
the narrower refactor: a **set-returning** `walk_premises(fact, kb) → set[Fact]`
collector (today `reaches()` returns `bool`). NB the back-pointer field is
`premises_raw`, not the field name this stub originally named.

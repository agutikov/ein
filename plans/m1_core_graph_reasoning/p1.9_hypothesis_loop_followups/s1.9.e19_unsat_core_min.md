# S1.9.E19 — Unsat-core minimisation

**Phase:** P1.9, §engineering / UX.

**Status:** 📅 parked.

E19.

## Context

When a contradiction fires, the unsat-core (premises that
produced it) may be larger than the *minimal* set that
implies the contradiction. Shrinking it (delta-debug or
linear-time minimisation) improves trace readability:
"these N facts together contradict" vs "these M ≫ N facts
include the N that matter".

Pure UX (no correctness gain), but composes with E7
(learned-clause from unsat-core).

## Tasks

### Task T1.9.E19.1 — Minimiser

A `minimise_core(core, kb) → set[Fact]` helper. Strategy:
remove each fact one at a time, re-run the contradiction
check; if still contradicts, keep removed. Greedy.

### Task T1.9.E19.2 — Use sites

Apply at contradiction-record time. Cached so re-derivation
isn't needed.

### Task T1.9.E19.3 — Tests

Synthetic puzzle with a large redundant core → minimiser
shrinks correctly.

# S1.9.E18 — Rule-applicability pruning

## Context

When loading a puzzle, scan each rule's `:match` clause for
relation names. If a referenced relation is absent from the
KB (no declarations + no facts + no rule asserts it), the
rule can never fire — drop it from the compilation cache.

Saves the cost of compiling + watching unreachable rules.

## Tasks

### Task T1.9.E18.1 — Static reachability

Per rule: which relations are referenced in `:match`? Per
KB: which relations are present? Pre-compile, intersect.

### Task T1.9.E18.2 — Drop unreachable rules

In `Engine.compile_all`, skip rules whose required-relation
set isn't a subset of present-relation set.

### Task T1.9.E18.3 — Tests + bench

Confirm rules with absent relations don't fire (which they
already don't, but the engine work is saved).

## Acceptance

- Pruning ships; rule count in cache reduces on puzzles that
  import a wider library than they use.

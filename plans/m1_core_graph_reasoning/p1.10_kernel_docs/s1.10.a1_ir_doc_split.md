## Context

Today's `docs/kernel/ir/02-data-model/` mixes "what abstract
shapes the KB carries" (idiomatic level) with "how Python
encodes them" (python-impl level). The user's framing
2026-05-24:

```
ir/01-ein-graph/        — semantics (unchanged)
ir/02-data-model/       — idiomatic: data types, collections, indexes, …
ir/02b-python-impl/     — Python implementation specifics
ir/03-ein-lang/         — surface syntax (unchanged)
```

This stage performs the split; S1.10.A2 backfills missing
content at the idiomatic level (mathematical pseudocode,
algorithm diagrams).

# S1.1.2 ‚Äî Parser + serialiser

**Phase:** P1.1 (IR language).
**Estimate:** 3-4 days.
**Depends on:** S1.1.1 grammar locked.

## Context

A minimal S-expression reader/printer in pure Python (no
dependencies beyond stdlib). The grammar is small enough that a
recursive-descent tokeniser + reader is ~150 LOC. We deliberately
avoid pulling in `sexpdata` or `hy` ‚Äî keeping the parser in-tree
makes the GBNF lift in M2.P2.3 a structural rewrite of one file,
and lets us track source positions for error messages and trace
provenance.

## Tasks

### Task T1.1.2.1 ‚Äî Token + Node types

`src/ein_bot/ir/__init__.py` plus `src/ein_bot/ir/types.py`. Define:

```python
@dataclass(frozen=True)
class Loc:
    file: str
    line: int
    col: int

@dataclass(frozen=True)
class Atom:
    name: str
    loc: Loc | None = None

@dataclass(frozen=True)
class String:
    value: str
    loc: Loc | None = None

@dataclass(frozen=True)
class SForm:
    head: Atom
    args: tuple[IRNode, ...]
    loc: Loc | None = None

IRNode = Atom | String | SForm | int | float
```

Atoms are interned. `Loc` is optional for synthetically-constructed
nodes (rules emitting derived facts).

### Task T1.1.2.2 ‚Äî Tokeniser

`src/ein_bot/ir/lex.py`. One-pass, tracks `(line, col)`. Tokens:
`LPAREN`, `RPAREN`, `ATOM`, `STRING`, `KEYWORD` (atom starting with
`:`), `INT`, `FLOAT`. Whitespace + `;` line comments skipped.
Block comments `#| ‚Ä¶ |#` supported (non-nested for v0).

### Task T1.1.2.3 ‚Äî Reader

`src/ein_bot/ir/parse.py`. Recursive `read_form()`. On `LPAREN`,
read head (must be `ATOM` or `KEYWORD`), then args until matching
`RPAREN`. Trailing reader functions `parse(text)`,
`parse_file(path)`. Both yield a tuple of top-level `SForm`s.

Error messages quote the offending line + carret, e.g.:

```
zebra.ein:14:7: expected ')' after 6 forms, got 'unexpected'
            (rel lives-in Norwegian House_1 unexpected)
                                            ^^^^^^^^^^
```

### Task T1.1.2.4 ‚Äî Printer (canonical form)

`src/ein_bot/ir/dump.py`. `dump(node)` returns the canonical
text ‚Äî two-space indent, one form per line for top-level, atoms
unquoted, strings double-quoted. Two modes:

- `dump_canonical` ‚Äî fully reformats; `parse(dump_canonical(x)) == x`
  ignoring `Loc`.
- `dump_compact` ‚Äî single-line; useful for log messages.

### Task T1.1.2.5 ‚Äî Wire into the existing CLI

Extend `src/ein_bot/cli.py`: `ein-bot ir parse <file>` reads + dumps
canonical; `ein-bot ir lint <file>` parses + checks for
unknown top-level heads.

## Acceptance

- `examples/zebra.ein` parses cleanly.
- `ein-bot ir parse examples/zebra.ein | diff -u - <(ein-bot ir parse <(ein-bot ir parse examples/zebra.ein | cat))` exits 0
  (parse‚àòprint‚àòparse‚àòprint is stable).
- A deliberately broken `examples/broken/*.ein` set of ~5 cases each
  produces a useful error with file:line:col.

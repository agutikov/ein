# S1.1.2 — Parser + serialiser

**Phase:** P1.1 (IR language).
**Estimate:** 3-4 days.
**Depends on:** S1.1.1 grammar locked.

## Context

A typed AST + canonical printer layered on top of the Lark grammar
locked in S1.1.1.

**Decision (S1.1.2 kickoff, 2026-05-18):** keep Lark as the parser.
The original stage plan called for a hand-rolled tokeniser + reader
to stay dependency-free and ease the M2.P2.3 GBNF lift. In practice
the GBNF lift is a Lark-grammar → GBNF translator
(`utils/lark_to_gbnf.py`, see `grammar.lark:39-40`) — the structural
rewrite is *of the grammar file*, which Lark already gives us in a
clean form. The Lark grammar in `grammar.lark` is the source of
truth; a `lark.Transformer` lowers the parse tree to typed
`IRNode`s.

The reader is therefore "Lark + Transformer", not a custom
recursive-descent parser. Source positions ride on Lark tokens
(`propagate_positions=True`).

## Tasks

### Task T1.1.2.1 — Node types

`src/ein_bot/ir/types.py`. Frozen dataclasses for the AST the
Transformer emits. The grammar mints `SYMBOL`, `VAR`, `KEYWORD`,
`WILDCARD`, `INT`, `RANGE`, `STRING` terminals plus generic lists;
each gets a node class.

```python
@dataclass(frozen=True)
class Loc:
    file: str
    line: int
    col: int

@dataclass(frozen=True)
class Atom:    name: str;  loc: Loc | None = None
@dataclass(frozen=True)
class Var:     name: str;  loc: Loc | None = None
@dataclass(frozen=True)
class Keyword: name: str;  loc: Loc | None = None
@dataclass(frozen=True)
class Wildcard: loc: Loc | None = None
@dataclass(frozen=True)
class String:  value: str; loc: Loc | None = None
@dataclass(frozen=True)
class Int:     value: int; loc: Loc | None = None
@dataclass(frozen=True)
class Range:   low: int;   high: int | None;  loc: Loc | None = None  # None ≡ "*"
@dataclass(frozen=True)
class KwPair:  key: Keyword; value: IRNode;   loc: Loc | None = None
@dataclass(frozen=True)
class SForm:
    head: Atom              # may be the literal "=" atom
    args: tuple[IRNode, ...]
    loc: Loc | None = None

IRNode = Atom | Var | Keyword | Wildcard | String | Int | Range | KwPair | SForm
```

`Loc` is optional so rule-derived nodes (no source position) are
representable.

### Task T1.1.2.2 — Transformer (Lark Tree → typed AST)

`src/ein_bot/ir/ast.py`. A `lark.Transformer` walks the parse tree
and emits typed nodes. Each grammar rule maps to one method
(`type_decl`, `relation_decl`, `eq_fact`, `generic_fact`,
`rule_decl`, `query_form`, …). Terminals lift to `Atom` / `Var` /
`Keyword` / `Wildcard` / `Int` / `Range` / `String` with `Loc`
populated from the `Token.line` / `Token.column`.

Top-level: `parse(text, *, filename=None) -> tuple[SForm, ...]`. The
Lark parser is built with `propagate_positions=True` so trees carry
`meta.line` / `meta.column`.

### Task T1.1.2.3 — Error messages

Catch Lark's `UnexpectedInput` subclasses and re-raise as an
`IRParseError` that prints `<file>:<line>:<col>: <message>` with the
offending line and a carret. Lark's `get_context()` already returns
the line and a pointer; we just prepend the location prefix and
the filename passed to `parse`.

### Task T1.1.2.4 — Printer (canonical form)

`src/ein_bot/ir/dump.py`. `dump(node)` returns canonical text — atoms
unquoted, strings double-quoted, kw-pairs as `:kw value`. Two modes:

- `dump_canonical` — pretty multi-line; line-breaks an SForm when
  its compact form exceeds the column budget (default 80).
- `dump_compact` — single-line; useful for log messages.

Idempotency target: `parse(dump_canonical(x)) ≡ x` modulo `Loc`.

### Task T1.1.2.5 — Wire into the existing CLI

Extend `src/ein_bot/cli.py`: `ein-bot ir parse <file>` reads + dumps
canonical; `ein-bot ir lint <file>` parses + reports errors. The
legacy 2021-prototype entry (load conditions → DOT) moves under
`ein-bot legacy …`.

## Acceptance

- `examples/zebra.ein` parses to a `tuple[SForm, ...]`.
- `parse(dump_canonical(parse(text))) == parse(text)` (modulo `Loc`)
  for every passing case in `tests/test_ir_parser.py`, plus a
  byte-stability check `dump(parse(dump(parse(x)))) == dump(parse(x))`.
- A deliberately broken `examples/broken/*.ein` set of ~5 cases each
  raises `IRParseError` with `file:line:col` in the message.
- `ein-bot ir parse <file>` and `ein-bot ir lint <file>` exit 0 on
  `examples/zebra.ein` and non-zero on the broken set.

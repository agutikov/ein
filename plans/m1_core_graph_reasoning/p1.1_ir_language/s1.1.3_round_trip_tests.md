# S1.1.3 — Round-trip + golden tests

**Phase:** P1.1 (IR language).
**Estimate:** 1-2 days.
**Depends on:** S1.1.2 parser/printer.

## Context

The IR is touched by every other phase; locking its behaviour with
tests is what makes the rest of M1 a refactor against a stable
target. Three test layers: unit (per AST node), round-trip
(parse → dump → parse fixpoint), and **golden** (a curated
`examples/*.ein` directory that doubles as documentation and
regression target).

## Tasks

### Task T1.1.3.1 — Unit tests for the lexer

`tests/ir/test_lex.py`. Cover: atoms, keywords, strings,
integers, floats, line comments, block comments, location tracking
on multi-line input, and the explicit error cases (unterminated
string, unterminated block comment, stray `)`). ~20 tests.

### Task T1.1.3.2 — Unit tests for the parser

`tests/ir/test_parse.py`. Cover each top-level form from
S1.1.1 T1.1.1.1; cover nested forms; cover empty `(facts)` and
empty `(rules)`; cover the pattern sub-language (`?vars`, `:where`,
`:match`). Confirm that an empty file parses to `()` (empty tuple).

### Task T1.1.3.3 — Property test: round-trip stability

`tests/ir/test_roundtrip.py`. Use [hypothesis](https://hypothesis.readthedocs.io/)
(add to `[dev]` extras). Strategy generates valid `IRNode`s. Assert
`parse(dump_canonical(x)) == x` modulo `Loc`. Shrink-friendly.

### Task T1.1.3.4 — Golden fixtures

`tests/ir/golden/` holds:

- `zebra_15_conditions.ein` — full Zebra puzzle ontology + facts.
- `triangle_rule.ein` — single rule definition.
- `square_rule.ein` — single rule definition.
- `gaps_query.ein` — `(query :mode gaps ...)`.
- `contradictions_query.ein` — `(query :mode contradictions ...)`.

For each, a sibling `*.golden` file holds the canonical-printed
form. The test pipeline:

```python
def test_golden(name: str):
    src = (GOLDEN / f"{name}.ein").read_text()
    expected = (GOLDEN / f"{name}.golden").read_text()
    assert dump_canonical(parse(src)) == expected
```

Refresh with `pytest -k golden --update-golden` (custom flag in
`tests/conftest.py`).

### Task T1.1.3.5 — Error fixtures

`tests/ir/broken/*.ein` — five hand-crafted broken files, each
paired with a `.expected_error` containing the exact error
message including `file:line:col`. Tests:

```python
@pytest.mark.parametrize("name", BROKEN_FILES)
def test_broken(name):
    with pytest.raises(IRParseError) as ei:
        parse_file(BROKEN / f"{name}.ein")
    assert str(ei.value) == (BROKEN / f"{name}.expected_error").read_text().strip()
```

# S1.1.3 ‚Äî Round-trip + golden tests

**Phase:** P1.1 (IR language).
**Estimate:** 1-2 days.
**Depends on:** S1.1.2 parser/printer.

## Context

The IR is touched by every other phase; locking its behaviour with
tests is what makes the rest of M1 a refactor against a stable
target. Three test layers: unit (per AST node), round-trip
(parse ‚Üí dump ‚Üí parse fixpoint), and **golden** (a curated
`examples/*.ein` directory that doubles as documentation and
regression target).

## Tasks

### Task T1.1.3.1 ‚Äî Unit tests for the lexer

`tests/ir/test_lex.py`. Cover: atoms, keywords, strings,
integers, floats, line comments, block comments, location tracking
on multi-line input, and the explicit error cases (unterminated
string, unterminated block comment, stray `)`). ~20 tests.

### Task T1.1.3.2 ‚Äî Unit tests for the parser

`tests/ir/test_parse.py`. Cover each top-level form from
S1.1.1 T1.1.1.1; cover nested forms; cover empty `(facts)` and
empty `(rules)`; cover the pattern sub-language (`?vars`, `:where`,
`:match`). Confirm that an empty file parses to `()` (empty tuple).

### Task T1.1.3.3 ‚Äî Edge-case round-trip fixtures

Decision (2026-05-18): drop `hypothesis` for now. Property-based
testing pays off when strategies are cheaper than the cases they
generate; for a fixed small grammar with a dump that's already
exercised by the full Zebra fixture, the strategy maintenance cost
outweighs the marginal coverage.

Add ~20 hand-picked oddball inputs to the existing parametric
`test_roundtrip` in `tests/test_ir_ast.py`. Target the gaps a
hypothesis run would have found:

- **String escapes** ‚Äî `"\\n\\t\\\""`, embedded backslashes, empty
  string, Unicode (`"√©¬∑¬ª‚Üí"`).
- **Range edge cases** ‚Äî `0..0`, `0..*`, large numbers (`9999..*`).
- **Deep nesting** ‚Äî `(and (and (and (and ‚Ä¶))))` to a depth of 5+.
- **Variable / wildcard heads** ‚Äî `(?rel ?a ?b)`, `(_ a b)`,
  `(?r ?a (?s ?b ?c))`.
- **Empty bodies** ‚Äî `(facts)`, `(rules)`, `(reasoning)`, `(trace)`,
  `(ontology (type T))`, `(rule x ())`.
- **Kw-pair ordering** ‚Äî same form with kw-pairs in different orders.
- **Equality patterns** ‚Äî `(= ?a ?b)`, `(= (color H1) Red)`,
  `(facts (= a b))`, `(rules (rule e () :match (= ?a ?b) :assert ?a))`.
- **Negation** ‚Äî `(not X)`, `(not (not X))`, `(facts (not (rel a b)))`,
  `(reasoning (not (rel a b) :rule R :using (s1)))`.
- **Mixed kernel primitives** ‚Äî `(and (or X Y) (not Z) :where (neq ?a ?b))`.

If any input fails round-trip, fix the parser / dumper. The fixture
list itself is the documentation of which corners are exercised.

### Task T1.1.3.4 ‚Äî Golden fixtures

`tests/ir/golden/` holds:

- `zebra_15_conditions.ein` ‚Äî full Zebra puzzle ontology + facts.
- `triangle_rule.ein` ‚Äî single rule definition.
- `square_rule.ein` ‚Äî single rule definition.
- `gaps_query.ein` ‚Äî `(query :mode gaps ...)`.
- `contradictions_query.ein` ‚Äî `(query :mode contradictions ...)`.

For each, a sibling `*.golden` file holds the canonical-printed
form. The test pipeline:

```python
def test_golden(name: str):
    src = (GOLDEN / f"{name}.ein").read_text()
    expected = (GOLDEN / f"{name}.golden").read_text()
    assert dump_canonical(parse(src)) == expected
```

Refresh with `pytest -k golden --update-golden` (custom flag in
`tests/conftest.py`).

### Task T1.1.3.5 ‚Äî Error fixtures

`tests/ir/broken/*.ein` ‚Äî five hand-crafted broken files, each
paired with a `.expected_error` containing the exact error
message including `file:line:col`. Tests:

```python
@pytest.mark.parametrize("name", BROKEN_FILES)
def test_broken(name):
    with pytest.raises(IRParseError) as ei:
        parse_file(BROKEN / f"{name}.ein")
    assert str(ei.value) == (BROKEN / f"{name}.expected_error").read_text().strip()
```

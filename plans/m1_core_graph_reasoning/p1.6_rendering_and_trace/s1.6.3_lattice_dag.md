# S1.6.3 — Commitment-lattice / proof-DAG DOT

**Phase:** P1.6 (rendering + trace).
**Estimate:** 2-3 days.
**Depends on:** P1.5b `LatticeProof` (`inference/monotonic/lattice.py`)
+ `lattice_snapshot` (`inference/monotonic/snapshot.py`).
**Replaces:** the obsolete *S1.6.3 — Search tree DOT*. The ordered
`SearchTree` / `SearchNode` structure was removed with the tree solver
2026-05-29; there is no tree to render.

## Context

The engine no longer produces an ordered search tree; it produces a
**set-indexed commitment lattice**. Each visited commitment is a
`frozenset` of hypothesis facts (a `CanonicalSetId`); commitments are
related by subset/cover (Apriori prefix-join) and collapse by
post-saturation `state_hash`. Render this lattice as a DAG — the
direct analog of the old tree view, but a partial order rather than a
tree.

Inputs (see [P1.6 README §Input contract](README.md#input-contract-p15b--what-the-renderer-consumes)):

- `verdict.proof: LatticeProof` — `solutions`, `dead_commitments`,
  `kb_index: dict[int, SetNode]` (under `store_lattice=True`).
- `lattice_snapshot(verdict, root_kb) → LatticeSnapshotV1` — the
  permutation-invariant, `state_hash`-collapsed projection
  (`nodes_by_state_hash = (state_hash, union_labels, verdict_labels)`).
  **Preferred input:** it is order-stable across `lattice_order_seed`,
  so the diagram is reproducible (reuses the S1.5b.31 guarantee).

## Tasks

### Task T1.6.3.1 — Lattice-DAG renderer

`src/ein_bot/render/lattice_dag.py::render_lattice(proof | snapshot) -> str`
emits an inline `dot` block (no SVG):

- `rankdir=TB`, ranked by **layer** (`= commitment-set size`); layer 0
  (root saturation) at top.
- One node per `SetNode` / per `nodes_by_state_hash` entry. Node label
  = the commitment set, or — when several commitments collapsed to one
  `state_hash` — a multilabel summary of `labels` ("13 hint-commitments
  ≡ one state").
- Colour by `verdict`: `alive` (grey), `dead` (red), `solution`
  (green).
- Edges = subset/cover between commitments (the Apriori lattice edges)
  and/or `state_hash`-merge collapses (annotate "≡ N commitments").
- Dead nodes carry their `unsat_core` (from the matching
  `DeadCommitment`) in the tooltip; the `learned_clause` labels the
  back-edge to layer `k-1` (the lifted no-good).

### Task T1.6.3.2 — Full + solution views

Two inline `dot` blocks (no SVG; mirroring the old `tree-full` /
`tree-path` split):

- **lattice-full** — every visited commitment, all layers (needs
  `store_lattice=True` for the full `kb_index`).
- **lattice-solution** — the surviving solution commitment(s) + the
  dead siblings pruned at each layer (the "interesting" sub-DAG the
  trace references). The analog of the old survivor-path view; this is
  the block embedded in the trace's final section (S1.6.4).

### Task T1.6.3.3 — Wire the `trace-view=dag` stub

`ir/to_dot.py`'s `trace-view="c"` (derivation-DAG) is currently a stub
that emits no DOT (labels-and-falls-through). Point it at the
lattice-DAG renderer so `--trace-view=dag` produces real output. (The
`trace-view="b"` aggregate view is separate — out of scope here.)

### Task T1.6.3.4 — Tests

`tests/render/test_lattice_dag.py`:

- A 3-commitment proof (1 solution + 2 dead) renders; verdict colours
  correct.
- A `state_hash`-collapsed multilabel `SetNode` (e.g. from
  `examples/lattice/03_state_hash_collision.ein`) renders as one node
  labelled with all its commitments.
- Rendering from `lattice_snapshot` is invariant under
  `lattice_order_seed` (reuse the S1.5b.31 shuffle harness).

## Acceptance

- `pytest tests/render/test_lattice_dag.py` green.
- `--trace-view=dag` on a zebra2 solve produces a lattice DAG whose
  `solution`-coloured frontier matches `proof.solutions` and whose
  `dead` nodes match `proof.dead_commitments`.
- The solution view is small and readable on zebra2 (the puzzle is
  near-deterministic after the d=0 saturation; few size-1 commitments
  survive to branch).

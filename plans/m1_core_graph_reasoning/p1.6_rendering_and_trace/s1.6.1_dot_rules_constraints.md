# S1.6.1 — DOT for rules + constraints

**Phase:** P1.6 (rendering + trace).
**Estimate:** 2 days.
**Depends on:** P1.1 IR, P1.3 rule registry, P1.4 constraints.

## Context

The renderers in this stage produce DOT diagrams of *static*
artefacts — the rule registry and the constraint set — so the
trace can refer back to a canonical picture of the inference
calculus, not redraw it per step.

Output goes alongside per-step state diagrams (S1.6.2). The bundle
layout:

```
out/
├── rules.svg              one diagram showing the ten rule families
├── rules.<name>.svg       per-rule pattern → conclusion diagram
├── constraints.svg        one diagram of declared constraints + scopes
└── ...                    state snapshots (S1.6.2), search tree (S1.6.3)
```

## Tasks

### Task T1.6.1.1 — Rule pattern → DOT

`src/ein_bot/render/rules.py`. For each rule, render:

- **Top**: pattern (LHS) — premise edges as solid arrows.
- **Bottom**: conclusion (RHS) — derived edge as a dashed arrow.
- **Labels**: variables `?a`, `?b` carry through; relation names
  on edges.
- **Constraints** (`:where` clauses): annotation block.

Single SVG per rule + a *combined* SVG with all ten arranged in a
3×4 grid using `graphviz fdp -Goverlap=prism`.

### Task T1.6.1.2 — Constraint scopes → DOT

`src/ein_bot/render/constraints.py`. Diagram the constraint scopes:

- **Nodes**: the types/relations the constraint quantifies over.
- **Edges**: `:scope` relationships.
- **Annotations**: the predicate (`at-most-one`, `all-different`,
  `exactly-n`, …).

One SVG showing all constraints; useful for sanity-checking that
the engine has the constraints the puzzle requires.

### Task T1.6.1.3 — CLI plumbing

Extend `ein-bot`:

```sh
ein-bot render rules examples/rules.ein --out=out/rules.svg
ein-bot render constraints examples/zebra.ein --out=out/constraints.svg
ein-bot render rule examples/rules.ein --name=triangle-composition --out=out/rule.svg
```

### Task T1.6.1.4 — Tests

`tests/render/test_rules_dot.py`:

- Every rule in `examples/rules.ein` renders without graphviz
  errors (`dot -Tsvg` exits 0).
- The combined `rules.svg` references all ten rule names.
- Constraint diagram references all constraint names.

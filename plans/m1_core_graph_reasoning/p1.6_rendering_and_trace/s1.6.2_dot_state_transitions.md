# S1.6.2 — DOT for state + state transitions

**Phase:** P1.6 (rendering + trace).
**Estimate:** 2-3 days.
**Depends on:** P1.2 graph + S1.6.1 styles.

## Context

A *state diagram* is the graph at a moment during reasoning. A
*state-transition diagram* shows two snapshots side-by-side with
the newly-added edges highlighted — that's how the trace makes
"rule X fired, deriving edge E" visible.

Three layers (ontology / fact / reasoning) drive the colouring:
ontology in grey, fact in black, reasoning edges colour-coded by
rule (using the deterministic `hash_color` already in
`src/ein_bot/rendering.py` from the package skeleton).

## Tasks

### Task T1.6.2.1 — Snapshot DOT

`src/ein_bot/render/state.py::render_state(g, *, layer_filter=None)`:

- Nodes coloured by type (ontology palette).
- Edges coloured by layer + rule. Source-layer edges black; rule
  edges hash-coloured by rule name (per the PoC's existing
  `hash_color`); hypothesis edges in red dashed; rejected edges
  in red dotted with strikethrough labels.
- Provenance tooltip via DOT `URL=…` / `tooltip=…`.

Optional `layer_filter` selects subsets; useful for the trace
("show fact layer only" then "fact + reasoning").

### Task T1.6.2.2 — Transition diff DOT

`render_transition(g_before, g_after) -> str`. Renders the union
graph; edges present only in `g_after` are highlighted with thicker
strokes + boxed labels. Used by the trace renderer to show one
step's effect.

### Task T1.6.2.3 — Subgraph clusters per layer

When all three layers are shown, group by Graphviz `subgraph
cluster_<layer>` so the renderer keeps them visually distinct.
Mirrors the clusters in
[`docs/index/knowledge-graph.dot`](../../../docs/index/knowledge-graph.dot).

### Task T1.6.2.4 — CLI

```sh
ein-bot render state zebra.ein --out=out/state-0000.svg
ein-bot render state zebra.ein --steps --out=out/state-XXXX.svg  ;; one per saturate step
```

The `--steps` form drives the saturator iterator (P1.3 S1.3.3),
saving a snapshot per `Firing`.

### Task T1.6.2.5 — Tests

`tests/render/test_state_dot.py`:

- A 3-edge graph renders without errors.
- Per-step rendering produces N+1 SVG files for N firings.
- Transition diff highlights exactly the new edges.

# S2.6.1 — Benchmark corpus + gold IR/trace

**Phase:** P2.6 (evaluation harness).
**Estimate:** 4-5 days.
**Depends on:** P2.4 + P2.5 decisions.

## Context

We need a small, curated, *human-vetted* corpus. Quality > quantity:
five well-documented puzzles beats fifty noisy ones.

## Tasks

### Task T2.6.1.1 — Pick the five puzzles

Pre-committed to:

1. **Zebra (canonical)** — the 15-condition Einstein puzzle.
2. **Einstein variant** — a known re-statement with the same
   structure but different entities.
3. **Logic grid puzzle A** — small 4×4 (e.g. pets + colours +
   names + cities).
4. **Logic grid puzzle B** — small 5×5 with explicit spatial
   constraints.
5. **Ambiguity stress** — a hand-crafted 4-sentence puzzle with
   one deliberately ambiguous sentence, gold IR includes the
   parse-hypothesis wrapper.

For each: `tests/nl/corpus/<n>/{puzzle.txt, gold.ein, gold-trace.md}`.

### Task T2.6.1.2 — Author the gold IRs

Hand-write each `gold.ein`. These are the authoritative answers
for IR extraction quality.

### Task T2.6.1.3 — Author one gold trace

For Zebra, expand the human walkthrough from
[idea 08 §The target trace](../../../docs/ideas/08-human-style-deductive-trace.md#the-target-trace-paraphrased)
into a complete step-by-step `gold-trace.md`. Annotate each step
with the expected named rule.

For the other four, capture a short "what the trace should at
least include" checklist (not a full walkthrough — that scales
poorly).

### Task T2.6.1.4 — Metric set

Three per-puzzle metrics:

- **IR-F1** (structural F1 of predicted vs gold IR);
- **Answer correctness** (boolean, after running the M1 engine on
  the predicted IR);
- **Trace coverage** (Zebra only — number of gold-trace named
  moves that appear in the generated trace).

## Acceptance

- Five `tests/nl/corpus/<n>/` directories committed.
- Each `gold.ein` parses cleanly under P1.1 and solves cleanly
  under M1.
- Gold-trace for Zebra contains every named rule firing the
  P1.7 acceptance test already enforces.

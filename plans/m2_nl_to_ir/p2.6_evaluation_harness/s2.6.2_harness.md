# S2.6.2 — Harness + CI integration

**Phase:** P2.6 (evaluation harness).
**Estimate:** 3-4 days.
**Depends on:** S2.6.1.

## Context

The harness is *both* a pytest target (run-locally for the dev)
and a script that produces a reproducible report
(`docs/decisions/M2-final.md`).

## Tasks

### Task T2.6.2.1 — `pytest tests/nl/benchmark/`

Test file `test_corpus.py`: parameterised over the five corpus
entries, runs `ein-bot from-text … --trace=…` and asserts the
three metrics from S2.6.1.

The test is `@pytest.mark.integration` — needs `LLAMA_HOST`
reachable. Skipped otherwise (CI default).

### Task T2.6.2.2 — Mock-LLM smoke test

`tests/nl/benchmark/test_mock.py`: uses a recorded `LlmInvocation`
fixture (captured from a real run) as the LLM response.
Confirms the harness itself works without a live model.

Run in CI on every PR.

### Task T2.6.2.3 — Report script

`scripts/m2-benchmark-report.py`:

- Runs the full corpus N seeds × 2 model choices (primary +
  fallback);
- Emits a table of per-puzzle scores;
- Writes `docs/decisions/M2-final.md` with the table + the
  per-puzzle traces archived under `docs/decisions/M2-traces/`.

### Task T2.6.2.4 — Mark M2 done

When the report passes the M2 acceptance criteria, update:

- [`plans/m2_nl_to_ir/README.md`](../README.md) §Acceptance with
  a "Done — YYYY-MM-DD" stanza;
- [`plans/README.md`](../../README.md) status table.

## Acceptance

- The harness runs on the dev workstation against a real
  llama-server and produces a deterministic-up-to-seed report.
- CI runs the mock-LLM version on every PR.
- M2's milestone-level acceptance criteria all satisfied.

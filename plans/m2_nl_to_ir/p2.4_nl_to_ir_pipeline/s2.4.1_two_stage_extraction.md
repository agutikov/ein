# S2.4.1 — Two-stage extraction (ontology + facts)

**Phase:** P2.4 (NL → IR pipeline).
**Estimate:** 4-5 days.
**Depends on:** P2.2 + P2.3.

## Context

A single LLM pass over the puzzle text trying to emit the full IR
is too much load. Split into two passes:

1. **Ontology pass** — read the whole puzzle text; emit an
   `(ontology …)` block declaring types, instances, relations.
   GBNF: `grammars/ontology.gbnf`.
2. **Facts pass** — for each sentence (or sentence group) of the
   puzzle text, emit `(facts …)` fragments parameterised by the
   ontology from pass 1. GBNF: `grammars/facts.gbnf`.

Two passes lets us audit ontology coverage before facts pile up
referencing unknown atoms, and lets us use the small parameterised
facts grammar (P2.3 S2.3.2 T2.3.2.2).

## Tasks

### Task T2.4.1.1 — Prompt templates

`src/ein_bot/nl/prompts/` holds two templates:

- `ontology.tmpl` — *"Read this puzzle text. List every entity
  type, instance, and relation. Output as `(ontology …)` IR."*
- `facts.tmpl` — *"Given this puzzle text and this ontology,
  extract every fact as IR (rel …) forms. Quote the originating
  sentence in `:source`."*

Both prompts include a small few-shot example (Zebra's first three
sentences). Templates are loaded by `string.Template`-style
substitution.

### Task T2.4.1.2 — Ontology pass

`src/ein_bot/nl/ontology_pass.py`:

```python
def extract_ontology(text: str, llm: LlmClient) -> IRNode:
    grammar = load_grammar("ontology")
    resp = llm.complete(LlmRequest(
        messages=[
            {"role": "system", "content": ONTOLOGY_PROMPT},
            {"role": "user", "content": text},
        ],
        grammar=grammar,
        seed=0,                        # deterministic
    ))
    return parse(resp.text)[0]         # single (ontology …) form
```

period boundaries; one LLM call per sentence; concat the results.
Tradeoff: more LLM calls but easier provenance + smaller context.

### Task T2.4.1.4 — IR assembly

`assemble_ir(ontology, facts, query) -> IRNode`. Combine the three
into a single top-level `(facts …)` document the M1 engine can
load.

### Task T2.4.1.5 — Tests

`tests/nl/test_extraction.py` (`@pytest.mark.integration`):

- Zebra text → ontology pass yields a non-empty `(ontology …)`
  with 6 types (House, Color, Nationality, Pet, Drink, Cigarette).
- Zebra text + ontology → facts pass yields ≥ 15 facts (one per
  Zebra condition).
- Each fact has `:source "..."` referencing the originating
  sentence (verifiable against the input text).

## Acceptance

- Both passes run on a real `examples/zebra.txt` and produce IR
  that parses + populates a graph with the same counts as the
  hand-coded `examples/zebra.ein` (modulo provenance metadata).
- Tests skip when llama-server is not running.

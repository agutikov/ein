# S2.4.4 — End-to-end CLI: `ein-bot from-text`

**Phase:** P2.4 (NL → IR pipeline).
**Estimate:** 2-3 days.
**Depends on:** S2.4.1 + S2.4.2 + S2.4.3.

## Context

Ties the three extraction stages together behind a single CLI. Two
modes:

- **direct**: emit IR to stdout (default).
- **chained**: pipe the IR straight into M1's solver, emit the
  trace.

## Tasks

### Task T2.4.4.1 — `ein-bot from-text`

```
ein-bot from-text PATH
  [--ontology=PATH]    use a library ontology (skip ontology pass)
  [--llm-host=...]     llama-server URL (default http://localhost:9080)
  [--gold=PATH]        if set, compare emitted IR against a gold file
  [--trace=PATH]       chain: feed IR into the engine and write the trace
  [--solve|--gaps|--contradictions]   query mode (default solve)
```

### Task T2.4.4.2 — Reproducibility envelope on output

Top of every emitted IR file: a `(meta …)` block recording:

- input file path + sha;
- ontology source (auto / library / file);
- llama-server `LlmInvocation` envelope (P2.2 S2.2.2 T2.2.2.4);
- ein-bot package version + commit.

So a saved IR file can be reproduced from the same inputs months
later.

### Task T2.4.4.3 — Tests

`tests/cli/test_from_text.py`:

- The CLI emits valid IR for `examples/zebra.txt`.
- `--gold=examples/zebra.ein` reports a structural diff (no diff
  on canonical Zebra; reports differences on a mutated version).
- `--trace=...` produces a markdown trace file.

## Acceptance

- The CLI is the single entry point for NL inputs.
- Reproducibility metadata present on every output.
- The chained mode produces the canonical Zebra answer end-to-end.

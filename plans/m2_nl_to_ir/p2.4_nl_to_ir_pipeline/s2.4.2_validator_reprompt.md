# S2.4.2 — Validator + re-prompt

**Phase:** P2.4 (NL → IR pipeline).
**Estimate:** 3-4 days.
**Depends on:** S2.4.1.

## Context

GBNF guarantees *syntactic* well-formedness, not *semantic*
validity. An LLM-emitted fact can still:

- reference an instance not in the ontology;
- assert a relation with the wrong type signature;
- contradict an already-emitted fact (purely structurally, no
  reasoning needed).

The validator catches all three before facts hit the engine. On
failure it *re-prompts* the LLM with the rejected fact + the
reason — a small "try again, but…" loop, capped at 3 attempts.

## Tasks

### Task T2.4.2.1 — Validator

`src/ein_bot/nl/validate.py`:

```python
@dataclass(frozen=True, slots=True)
class ValidationIssue:
    kind:   Literal['unknown-atom', 'type-mismatch', 'structural-contradiction']
    fact:   IRNode
    reason: str
    fix_hint: str | None = None

def validate(facts: list[IRNode], ontology: IRNode) -> list[ValidationIssue]: ...
```

Checks performed:
- **unknown-atom**: every atom in a fact must be declared in the
  ontology (instances + relations).
- **type-mismatch**: a relation's argument types must match the
  ontology's `(relation ...)` declaration.
- **structural-contradiction**: two facts of a `:functional`
  relation with different destinations (e.g. `(rel has-color
  House-1 Red)` and `(rel has-color House-1 Green)` from the same
  pass — almost certainly an LLM hallucination, not a puzzle
  contradiction). Real contradictions surface as a CONTRADICTIONS
  query, not at the validator stage.

### Task T2.4.2.2 — Re-prompt loop

`src/ein_bot/nl/repair.py`:

```python
def extract_facts_with_validation(text, ontology, llm,
                                    max_attempts: int = 3) -> list[IRNode]:
    facts = extract_facts(text, ontology, llm)
    for attempt in range(max_attempts):
        issues = validate(facts, ontology)
        if not issues:
            return facts
        facts = repair_facts(text, ontology, facts, issues, llm)
    raise NlExtractionError(issues)
```

Repair prompt: shows the LLM the rejected facts and the issues,
asks for a corrected version. GBNF still constrains the output.

### Task T2.4.2.3 — Audit trail

Every validation cycle is recorded in the IR's `(meta …)` block
(carried forward by the engine into the trace). Lets a user see
"the LLM emitted X, validator rejected with Y, fixed on attempt 2
to Z." This is *required* for the user's "personal research
project" framing — every NL → IR decision must be auditable.

### Task T2.4.2.4 — Tests

`tests/nl/test_validate.py`:

- Unknown-atom rejection works.
- Type-mismatch rejection works (e.g. `(rel lives-in Red House-1)`
  rejects because `Red` is a Color, not a Nationality).
- Structural contradiction within a pass rejected; same fact set
  loaded post-hoc into the engine in CONTRADICTIONS mode *also*
  works (the validator and the engine handle different layers).

# S2.3.1 — IR-grammar generator

**Phase:** P2.3 (GBNF for IR).
**Estimate:** 3-4 days.
**Depends on:** P1.1 IR spec + P2.2 client.

## Context

GBNF (GGML BNF) is what `llama-server` accepts as the `grammar`
parameter. Hand-writing it for the full IR is fragile — every
change to `docs/ir.md` would require a manual GBNF edit. Instead,
*generate* the GBNF from the IR's parser definitions so the two
stay in lockstep.

## Tasks

### Task T2.3.1.1 — Author the GBNF skeleton

`grammars/ir.gbnf.in`. The non-parametric core: lexical rules
(atom, keyword, integer, string), the recursive `form` rule
(`'(' ' '* head args ' '* ')'`). Approximately:

```
root           ::= top-form (' '* top-form)* ' '*
top-form       ::= '(' ws? head ws? args? ')'
head           ::= atom | keyword
args           ::= arg (ws arg)*
arg            ::= atom | keyword | string | number | top-form
atom           ::= [a-zA-Z_][a-zA-Z0-9_-]*
keyword        ::= ':' [a-zA-Z_][a-zA-Z0-9_-]*
string         ::= '"' ([^"\\] | '\\' .)* '"'
number         ::= '-'? [0-9]+ ('.' [0-9]+)?
ws             ::= [ \t\n]+
```

This is the *kernel* — it accepts any well-formed S-expression.
Task-specific GBNFs (S2.3.2) restrict it to specific top-level
heads.

### Task T2.3.1.2 — Build the generator

`src/ein/ir/to_gbnf.py`:

```python
def to_gbnf(spec: GrammarSpec = DEFAULT) -> str:
    """Emit GBNF for the IR. spec lets callers restrict to specific

```

Read the IR's keyword inventory from `src/ein/ir/parse.py`
(via reflection or a single source-of-truth constants module) so

### Task T2.3.1.3 — Round-trip test

`tests/llm/test_gbnf_roundtrip.py`
(`@pytest.mark.integration`):

- Load `examples/zebra.ein`.
- For each top-level form, send it to llama-server with the
  IR-restricted GBNF and a system prompt of:

  > Echo exactly the IR form you receive, byte-for-byte except
  > you may normalise whitespace.

- Parse the LLM's output with P1.1's parser.
- Assert structural equality with the input.

If the LLM cannot reproduce the IR under its own grammar, that's a
GBNF bug; iterate.

### Task T2.3.1.4 — Performance check

GBNF decoding can be slow if the grammar is overly recursive.
Measure tokens/sec for a 100-token IR emission under the full IR
grammar. Target: ≥ 30 tok/s on the dev workstation. If lower,
break the grammar by top-level head (S2.3.2 does this anyway).

### Task T2.3.1.5 — Document the grammar

`docs/gbnf.md`:

- how to regenerate `grammars/ir.gbnf` from source;
- the relationship between the parser (P1.1) and the grammar
  (this module);
- the known limitations of GBNF (e.g. no semantic predicates).

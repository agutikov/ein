# S2.2.2 — Python HTTP client + GBNF + retries

**Phase:** P2.2 (LLM infra).
**Estimate:** 3-4 days.
**Depends on:** S2.2.1.

## Context

A thin OpenAI-compatible client tailored to llama-server. Two
flavours:

- **Plain chat** — for prompt engineering / inspection.
- **Constrained** — passes a `grammar` field (GBNF text) so the
  model emits only grammatically-valid output.

Keep the dependency surface small: `httpx` for the HTTP, `pydantic`
for response models. No `langchain` / `llama-index` — we don't need
their orchestration and they hurt reproducibility.

## Tasks

### Task T2.2.2.1 — Client skeleton

`src/ein/llm/client.py`:

```python
@dataclass(frozen=True, slots=True)
class LlmRequest:
    messages:   list[dict[str, str]]
    grammar:    str | None = None
    max_tokens: int = 2048
    temperature: float = 0.0           # default deterministic for IR extraction
    seed:       int | None = None

@dataclass(frozen=True, slots=True)
class LlmResponse:
    text:   str
    usage:  dict[str, int]
    model:  str                        # llama-server returns the file name
    seed:   int                        # echoed back
    timing: TimingInfo

class LlmClient:
    def __init__(self, base_url: str, timeout: float = 60.0): ...
    def complete(self, req: LlmRequest) -> LlmResponse: ...
    def complete_stream(self, req: LlmRequest) -> Iterator[str]: ...
```

POST to `/v1/chat/completions`. Honour the `grammar` field as
llama-server expects it (`response_format`-adjacent in current
versions).

### Task T2.2.2.2 — Typed exceptions

```python
class LlmError(Exception): ...
class LlmTimeout(LlmError): ...
class LlmContextOverflow(LlmError): ...    # caught + surfaced
class LlmRateLimit(LlmError): ...           # for hosted endpoints, future
class LlmGrammarReject(LlmError): ...       # GBNF could not produce output
```

Retry policy: 3 attempts with exponential backoff on `LlmTimeout`
and HTTP 5xx; no retry on `LlmGrammarReject` (it's a logic bug, not
a transient).

### Task T2.2.2.4 — Reproducibility envelope

Every `LlmResponse` is bundled with a manifest:

```python
@dataclass(frozen=True, slots=True)
class LlmInvocation:
    model_sha:   str
    server_id:   str        # llama-server's startup uuid (from /props)
    grammar_sha: str | None
    request:     LlmRequest
    response:    LlmResponse
```

Trace records (M1.P1.6) embed this manifest as a `:llm-source`
field so the same NL → IR run can be replayed.

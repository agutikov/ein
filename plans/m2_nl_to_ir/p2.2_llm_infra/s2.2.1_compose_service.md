# S2.2.1 — Compose service + model pin

**Phase:** P2.2 (LLM infra).
**Estimate:** 3-4 days.
**Depends on:** P2.1 S2.1.6.

## Context

Reuse the acva pattern: a Compose file at
`packaging/compose/llm.yml` that runs llama-server with a mounted
model directory + GBNF directory. The model file lives outside the
repo; the Compose file pins by SHA + size; reproducibility lives in
the decision document, not in `git lfs`.

## Tasks

### Task T2.2.1.1 — Compose file

`packaging/compose/llm.yml`. Lifted from
[`acva/packaging/compose/docker-compose.yml`](../../../acva/packaging/compose/docker-compose.yml)
and trimmed to just the LLM service:

```yaml
services:
  llama-server:
    image: ghcr.io/ggerganov/llama.cpp:server-cuda-b<NNNN>@sha256:<digest>
    container_name: ein-llama-server
    ports:
      - "${LLAMA_PORT:-9080}:8080"
    volumes:
      - ${EIN_MODELS_DIR}:/models:ro
      - ${EIN_GRAMMARS_DIR}:/grammars:ro
    environment:
      - LLAMA_ARG_MODEL=/models/${EIN_LLM_MODEL}
      - LLAMA_ARG_CTX_SIZE=8192
      - LLAMA_ARG_THREADS=8
      - LLAMA_ARG_N_GPU_LAYERS=99
    command: ["--host", "0.0.0.0", "--port", "8080",
              "--mlock", "--no-mmap"]
    healthcheck:
      test: ["CMD-SHELL", "curl -f http://localhost:8080/health || exit 1"]
      interval: 10s
      timeout: 5s
      retries: 5
```

### Task T2.2.1.2 — `.env.example`

Documents `EIN_MODELS_DIR`, `EIN_GRAMMARS_DIR`,
`EIN_LLM_MODEL` (filename in `EIN_MODELS_DIR`), `LLAMA_PORT`.

### Task T2.2.1.3 — Model download script

`scripts/get-model.sh <model-url>` downloads + verifies SHA256
against `docs/decisions/M2-model-gbnf.md`. Refuses to overwrite a
verified file. Logs to a `models/manifest.json` so multiple
candidate models can coexist locally without confusion.

### Task T2.2.1.4 — Compose lifecycle docs

`docs/llm-infra.md`:

- start: `docker compose -f packaging/compose/llm.yml up -d`
- logs: `docker compose -f packaging/compose/llm.yml logs -f`
- stop: `docker compose -f packaging/compose/llm.yml down`
- swap model: `EIN_LLM_MODEL=...` in `.env`.

### Task T2.2.1.5 — Smoke test

`tests/llm/test_compose_up.py` (marked `@pytest.mark.integration`):

- Skips if `LLAMA_HOST` env var not set or `/health` not reachable.
- Sends a tiny chat-completion request, expects HTTP 200 + non-empty
  output.

## Acceptance

- Compose file boots locally with a pinned model.
- Health check passes within 30 s on the dev workstation.
- The model SHA-256 in the model directory matches what's pinned in
  `docs/decisions/M2-model-gbnf.md`.

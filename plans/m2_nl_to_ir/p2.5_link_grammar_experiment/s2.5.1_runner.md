### Task T2.5.1.2 — Serialiser to IR-like fragments

`src/ein/nl/link_grammar.py`:

```python
def parse_sentence(text: str) -> LinkGrammarOutput:
    """Run link-grammar; return all linkages + a confidence score."""

def to_fragment(output: LinkGrammarOutput) -> IRNode:
    """Convert head/dependent structure into a `(link-grammar …)` IR
    fragment that the LLM can read as context."""
```

The IR fragment is *informational* — it's not the constraint, it's
context fed into the LLM's prompt. Shape:

```ein
(link-grammar
  :sentence "The Norwegian lives in the first house."
  :linkages-shown 2
  :linkages (
    (linkage 0
      :links ((subj Norwegian lives)
              (det "the" Norwegian)
              (obj House-1 lives)
              (pos House-1 first)))
    (linkage 1 ...)))
```

If link-grammar returns multiple linkages with similar scores,
show top-K (K = 2-3); the LLM's facts-pass prompt is augmented
with this fragment.

### Task T2.5.1.3 — Prompt augmentation

A new prompt template `facts-with-link-grammar.tmpl`:

> Here is the puzzle sentence: …
> The link-grammar parser produced these linkages: …
> Emit the fact(s) for this sentence as IR.

Used only in the A/B comparison (S2.5.2).

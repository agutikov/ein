# S2.1.1 — Survey + pin the open questions

**Phase:** P2.1 (investigations).
**Estimate:** 1-2 days.
**Depends on:** nothing — this is the kickoff of M2.

## Context

A focused literature + prior-art sweep on the NL → constraint
pipeline. The point is *not* to read everything; it's to populate
a one-page "what we know, what we don't, what we'll experiment on"
so the per-question decisions in S2.1.2–S2.1.6 have a substrate.

## Tasks

### Task T2.1.1.1 — Build the reference list

`docs/decisions/M2-references.md`. Group by question:

- **Q8 (ambiguity)**: papers on under-determined NL parsing,
  CCG / link-grammar handling of multiple parses, schema-guided
  extraction.
- **Q9 (ontology)**: schema-extraction LLM papers, AMR coverage on
  logic puzzles.
- **Q23 (model)**: model leaderboards for structured-output
  extraction (specifically GBNF + JSON), reproducible benchmark
  scripts.

Two hours, max — this is a sweep, not a survey paper. Reuse
[`docs/lib/01-llm-constrained-generation.md`](../../../docs/lib/01-llm-constrained-generation.md)
and `09-cognitive-architectures-neurosymbolic.md`.

### Task T2.1.1.2 — Triage open questions

For each of Q7-Q11, Q23-Q24:

- restate the question in one sentence;
- list the candidate answers (≤ 4);
- note what evidence would decide it (a benchmark, a write-up, a
  spike).

Output: `docs/decisions/M2-triage.md`. The S2.1.2-S2.1.6 stages
consume this to know what to investigate.

### Task T2.1.1.3 — Spike candidates

Pick 2-3 spike experiments worth running before deciding:

- Smallest convincing test: prompt one local model with one Zebra
  sentence + a GBNF for IR; see if it emits valid IR.
- Link-grammar on one Zebra sentence; inspect the output.

Each spike ≤ 2 hours; capture results in
`docs/decisions/M2-spikes.md`.

## Acceptance

- Three documents committed: `M2-references.md`, `M2-triage.md`,
  `M2-spikes.md`.
- The remaining S2.1.* stages have clear-cut deliverables, not
  open-ended questions.
